# AC-008 Evidence — Malformed JSON graceful exit (BC-7.03.045 invariant 2)

**AC statement:** On stdin JSON parse failure: WASM plugin exits 0 (graceful degradation,
parity with bash jq-missing exit 0). Verified by bats test: malformed JSON on stdin →
exit 0, no block, no stderr crash.

**BC trace:** BC-7.03.045 invariant 2

---

## Implementation (src/main.rs)

The WASI entry point uses the SDK trampoline `vsdd_hook_sdk::__internal::run(on_hook)`.
On JSON parse failure the trampoline emits `HookResult::Error` (exit 1). With
`on_error = "block"` in the registry, dispatcher-level errors block the SubagentStop
event at the dispatcher level.

The `main.rs` doc comment acknowledges the discrepancy:

```
// [process-gap] AC-008 calls for exit 0 on malformed JSON (parity with bash
// jq-missing path). The current trampoline emits HookResult::Error (exit 1)
// on parse failure. Per AC-008 this discrepancy must be documented in a
// BC-7.03.045 amendment (T-11 post-DONE, deferred).
```
