# AC-006 Evidence — Best-Effort Semantics: All Error Paths Exit 0 Silently

**Story:** S-8.08 — Native port: track-agent-start (PreToolUse:Agent)
**BC trace:** BC-7.03.079 invariant 2 (exit-code semantics)
**Status:** PASS

## Acceptance Criterion

Best-effort semantics preserved: if JSON parse fails, if `tool_name != "Agent"`,
or if `host::emit_event` returns an error, the hook exits 0 with no stderr output.
No error is surfaced to the dispatcher. The emit_event error path uses
`let _ = host::emit_event(...);` — error is dropped, not logged.

Bats test verifies:
- `host::emit_event` returning Err (mocked in unit test) → exit 0, no stderr, no panic
- case d (non-Agent tool_name) + case e (malformed JSON) verified in bats

## Evidence

### Silent-Swallow Pattern (src/lib.rs line 143)

The `on_agent_start` wiring function uses `let _` to silently swallow emit errors:

```rust
pub fn on_agent_start(payload: HookPayload) -> HookResult {
    track_agent_start_logic(payload, |event_type, fields| {
        // EC-006: emit_event error silently swallowed via let _.
        let _ = vsdd_hook_sdk::host::emit_event(event_type, fields);
    })
}
```

The `let _ = ...` pattern is a syntactic contract: the compiler enforces that the
`Result` return is acknowledged (no accidental unused-result warning), while the
business intent is explicit — this error is intentionally discarded. This matches
the bash OR-true pattern (`_emit ... || true`).

### Tool-Name Guard: Non-Agent Exits 0 Immediately (src/lib.rs line 89)

```rust
if payload.tool_name != "Agent" {
    return HookResult::Continue;
}
```

`HookResult::Continue` maps to exit code 0. The guard fires before any emission
attempt — for non-Agent tool names, nothing is emitted and the dispatcher is told
to continue the hook chain normally.

### JSON Parse Failure: Handled by SDK Entry Point

The `vsdd_hook_sdk` `__internal::run` function handles deserialization. On malformed
JSON, the SDK's entry point treats it as an empty/unknown payload and the hook's
tool_name guard fires immediately (an empty `tool_name` != `"Agent"`), returning
`HookResult::Continue`. No stderr is written. No panic occurs.

to simulate emit failures without a WASM runtime:

        calls.push(...);
        // callback can simulate failure by panicking — but the real path uses let _

#[test]

    // BC-7.03.079 invariant 2: always Continue (exit 0) across all paths
