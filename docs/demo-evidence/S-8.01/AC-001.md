# AC-001 Evidence: Native WASM Crate + Registry Migration

**Story:** S-8.01 — Native port: handoff-validator (SubagentStop)
**AC:** AC-001 (traces to BC-7.03.042 postcondition 1; BC-2.02.012 postconditions 1-4)

## Claim

`crates/hook-plugins/handoff-validator/` exists with `Cargo.toml` targeting
`wasm32-wasip1`, implements the `vsdd-hook-sdk` hook interface, and the
registry entry uses `plugin = "hook-plugins/handoff-validator.wasm"` with
`event = "SubagentStop"`, `priority = 910`, `on_error = "block"`. No
`script_path`, `shell_bypass_acknowledged`, or `[hooks.capabilities]` block
remains in the handoff-validator registry entry. HookPayload typed fields
(`agent_type`, `subagent_name`, `last_assistant_message`, `result`) are used
for stdin deserialization.

### Registry entry (AC-001: registry migrated)

`plugins/vsdd-factory/hooks-registry.toml` handoff-validator entry:
```toml
[[hooks]]
name = "handoff-validator"
event = "SubagentStop"
plugin = "hook-plugins/handoff-validator.wasm"
priority = 910
timeout_ms = 5000
on_error = "block"
```

Verified absent from this entry:
- `script_path` — removed (was `hooks/handoff-validator.sh`)
- `shell_bypass_acknowledged` — removed (native WASM does not shell out)
- `[hooks.config]` block — removed
- `[hooks.capabilities.exec_subprocess]` block — removed (no subprocess calls)
- `binary_allow = ["bash", "jq"]` — removed (serde_json is self-contained)

BC-7.03.042 postcondition 1 binding preserved: `event = "SubagentStop"`,
`priority = 910`, `on_error = "block"`.
