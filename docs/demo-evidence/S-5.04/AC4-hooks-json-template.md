## Invariant 1: `once` Key Defensively Omitted

PostToolUseFailure fires per-failure, not once-per-session. The `once` key must be completely absent — not `once: false`, not `once: true` — matching the S-5.03 worktree-hooks pattern. This is a deliberate defensive omission: the key does not exist in the entry at all.

The VP-068 test asserts:
```rust
assert!(
    entry.get("once").is_none(),
    "BC-4.08.002 Invariant 1: PostToolUseFailure entry must NOT have a 'once' key \
     (fires per-failure; defensive omission; key must not exist at all — \
     not 'once: false', not 'once: true')"
);
```
