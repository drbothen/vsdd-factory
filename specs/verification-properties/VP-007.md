# VP-007: Dispatcher Self-Telemetry Is Always-On and Never Panics

## Property Statement

The internal log write path never panics and never propagates errors to the dispatcher's main execution path, regardless of filesystem errors (missing directory, read-only path, path-is-file). The dispatcher-internal-YYYY-MM-DD.jsonl file is attempted for every invocation independent of observability-config.toml.

## Source Contract

- **BC:** BC-1.06.001 — Internal log writes are best-effort; never panic; never propagate
- **Postcondition/Invariant:** DI-007 — Dispatcher self-telemetry is always-on

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust integration test | yes — filesystem error paths | misconfigured log_dir does not crash dispatcher |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_007_write_never_panics() {
    // log_dir points at an existing file (not a directory) — worst case
    let file = tempfile::NamedTempFile::new().unwrap();
    let log = InternalLog::new(file.path());

    // Must not panic, must not return Err
    log.write(&InternalEvent::now(DISPATCHER_STARTED));

    // Dispatcher is still alive (test didn't panic)
    assert!(true);
}
```

## Notes

The always-on invariant has two parts: (a) no panic / no error propagation (covered by test), and (b) written independently of observability-config.toml. Part (b) is an architectural separation concern enforced by code structure (InternalLog is initialized before SinkRegistry). Structural enforcement does not require a test harness.
