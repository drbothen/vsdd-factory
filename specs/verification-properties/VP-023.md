# VP-023: Wire Format Decoders Reject Truncated Input Without Panic

## Property Statement

All length-prefix wire format decoders (decode_fields for emit_event, decode_args for exec_subprocess) return Err or None when the buffer is truncated — i.e., when the declared length exceeds the remaining bytes. No decoder panics, no buffer overread occurs.

## Source Contract

- **BC:** BC-1.05.016 — decode_rejects_truncated_key_length: <4-byte buffer triggers Err
- **Postcondition/Invariant:** No buffer overread; malformed plugin output handled safely

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in emit_event.rs and exec_subprocess.rs | yes — boundary truncation cases | 4 truncation variants |

## Proof Harness Skeleton

```rust
#[kani::proof]
fn verify_vp_023_no_buffer_overread() {
    let len: usize = kani::any();
    let buf: Vec<u8> = kani::any_where(|v: &Vec<u8>| v.len() < len);

    // Decoder must not panic regardless of truncation
    let result = decode_fields(&buf);
    assert!(result.is_err() || result.unwrap().is_empty());
}
```
