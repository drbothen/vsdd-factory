# VP-020: Epoch Timeout Rounds Up and Terminates Infinite Loops

## Property Statement

timeout_ms_to_epochs(ms) always rounds up (div_ceil), so a sub-tick timeout gets at least 1 epoch of grace. A plugin in an infinite loop is interrupted at its epoch deadline. The epoch ticker advances every EPOCH_TICK_MS (10ms) and is shut down cooperatively.

## Source Contract

- **BC:** BC-1.04.003 — timeout_ms_to_epochs rounds up
- **Postcondition/Invariant:** DI-001 (tier completes) — epoch interruption ensures plugins terminate

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in engine.rs | yes — boundary values near epoch tick | div_ceil behavior + ticker shutdown |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_020_epoch_rounds_up() {
    // Sub-tick timeout gets 1 epoch (rounds up from 0.5 ticks)
    assert_eq!(timeout_ms_to_epochs(5, 10), 1);  // 5ms / 10ms = 0.5 → 1

    // Exact multiple
    assert_eq!(timeout_ms_to_epochs(100, 10), 10);

    // Just over boundary
    assert_eq!(timeout_ms_to_epochs(11, 10), 2);
}
```
