# VP-035: File Sink Auto-Creates Missing Parent Directories

## Property Statement

When writing the first event to a file sink whose parent directory does not exist, the sink creates all missing parent directories (equivalent to mkdir -p) before writing. This applies to both the file sink and the internal log.

## Source Contract

- **BC:** BC-3.02.007 — auto_creates_parent_directory: nested non-existent parent dirs are mkdir-p'd on first write
- **Postcondition/Invariant:** DI-007 — always-on telemetry must write even when log dir is missing

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit test with deeply nested tempdir path | yes — 3-level missing hierarchy | fs::create_dir_all behavior |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_035_auto_creates_parent_dirs() {
    let base = tempdir();
    let nested = base.path().join("a/b/c");
    let sink = FileSink::with_path_template(nested.join("events.jsonl"));

    sink.submit(make_event());
    sink.flush();

    // Parent dirs were created
    assert!(nested.exists());
    // Event was written
    assert_eq!(count_lines(nested.join("events.jsonl")), 1);
}
```
