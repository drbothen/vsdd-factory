# VP-030: Sink Shutdown Drains Queued Events Before Closing

## Property Statement

Calling sink.shutdown() drains all pending events from the channel before the worker thread exits. Events submitted before shutdown() are guaranteed to be written. Post-shutdown submit calls are no-ops (events are silently dropped). Shutdown is idempotent.

## Source Contract

- **BC:** BC-3.02.015 — shutdown_drains_queued_events
- **Postcondition/Invariant:** DI-011 — submit is non-blocking; events may be dropped only after shutdown

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_030_shutdown_drains() {
    let sink = FileSink::new(config);

    // Submit events before shutdown
    for _ in 0..10 {
        sink.submit(make_event());
    }

    // Shutdown drains
    sink.shutdown();

    // All pre-shutdown events must be written
    assert_eq!(count_lines(output_file), 10);

    // Post-shutdown submit is no-op
    sink.submit(make_event());
    assert_eq!(count_lines(output_file), 10);  // still 10
}
```

## Notes

The otel-grpc sink also drains via its worker thread (BC-3.03.007). Both sinks must complete the drain before the dispatcher process exits.
