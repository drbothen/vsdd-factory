# VP-049: Generated hooks-registry.toml Round-Trips Through Registry::load

## Property Statement

The hooks-registry.toml generated by scripts/generate-registry-from-hooks-json.sh round-trips through Registry::load without errors. The generation script is idempotent: running it twice on the same input produces bit-identical output.

## Source Contract

- **BC:** BC-1.07.003 — Generated hooks-registry.toml round-trips through Registry::load
- **Postcondition/Invariant:** DI-014 — schema version in generated file is always valid

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| integration | Rust integration test loading production registry | yes — production registry | parse success |

## Proof Harness Skeleton

```rust
// Integration test: crates/factory-dispatcher/tests/loads_legacy_registry.rs
#[test]
fn verify_vp_049_generated_registry_round_trips() {
    let path = "plugins/vsdd-factory/hooks-registry.toml";
    let registry = Registry::load(path).expect("registry must load");

    // Schema version is valid
    assert_eq!(registry.schema_version, 1);

    // All entries have required fields
    for entry in &registry.hooks {
        assert!(!entry.name.is_empty());
        assert!(!entry.event.is_empty());
        assert!(!entry.plugin.is_empty());
    }
}
```
