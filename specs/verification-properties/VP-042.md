# VP-042: SDK HostError Code Mapping Is Stable

## Property Statement

The SDK maps host function return codes to HostError variants: -1 = CapabilityDenied, -2 = Timeout, -3 = OutputTooLarge, -4 = InvalidArgument, other negative = Other(i32). These discriminants are stable and match the dispatcher's codes:: constants. The ffi.rs uses #[link(wasm_import_module = "vsdd")] on wasm32.

## Source Contract

- **BC:** BC-2.02.003 — HostError code mapping: -1=CapabilityDenied, -2=Timeout, -3=OutputTooLarge, -4=InvalidArgument
- **Postcondition/Invariant:** DI-004 — capability denial return codes are consistent across the ABI boundary

## Proof Method

| Method | Tool | Bounded? | Coverage |

| unit-test | Rust unit tests in hook-sdk/host.rs | yes — all 5 error code cases | HostError from_i32 |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_042_host_error_codes_stable() {
    assert_eq!(HostError::from(-1i32), HostError::CapabilityDenied);
    assert_eq!(HostError::from(-2i32), HostError::Timeout);
    assert_eq!(HostError::from(-3i32), HostError::OutputTooLarge);
    assert_eq!(HostError::from(-4i32), HostError::InvalidArgument);
    assert!(matches!(HostError::from(-99i32), HostError::Other(-99)));
}
```
