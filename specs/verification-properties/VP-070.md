# VP-070: `validate-artifact-path` Path-Pattern Matching Is Pure and Deterministic

## Property Statement

The path-pattern matching function in `validate-artifact-path/src/lib.rs` is
**pure and deterministic**: for the same `(path, registry)` input pair, the
function MUST always return the same `PathDecision` (Allow, Warn, Advisory, or
Block) regardless of:

- Call order
- Thread identity
- Wall-clock time
- Any external state not present in the input arguments

Formally:

```
∀ path: &str, registry: &ArtifactRegistry:
    match_path(path, registry) == match_path(path, registry)
```

Additionally, the following safety invariants hold unconditionally:

1. **Non-`.factory/` paths always return Allow (early exit).** The hook has no
   opinion on writes outside `.factory/`.
2. **Empty path always returns Allow.** Empty paths cannot match any `.factory/`
   prefix.
3. **Block decision is never returned for non-`.factory/` paths.** No combination
   of registry entries can cause a non-`.factory/` path to be blocked.
4. **The decision function is total.** It terminates for all inputs and never
   panics.

## Formal Property Statement (kani harness)

```rust
#[cfg(kani)]
mod kani_proofs {
    use super::*;

    /// Determinism: same (path, registry) always yields same decision.
    /// Kani verifies this holds for all inputs up to the bounded string length.
    #[kani::proof]
    #[kani::unwind(16)]
    fn proof_match_path_is_deterministic() {
        // Kani constructs arbitrary bounded inputs
        let path: String = kani::any();
        kani::assume(path.len() <= 64);

        // Use a fixed-size registry for bounded verification
        let registry = ArtifactRegistry::fixture_single_entry_block();

        let decision_1 = match_path(&path, &registry);
        let decision_2 = match_path(&path, &registry);

        kani::assert(
            decision_1 == decision_2,
            "match_path must be deterministic: same inputs must yield same decision"
        );
    }

    /// Non-.factory/ paths must always yield Allow (never Block).
    #[kani::proof]
    #[kani::unwind(16)]
    fn proof_non_factory_path_always_allow() {
        let path: String = kani::any();
        kani::assume(path.len() <= 64);
        // Assume path does NOT start with ".factory/"
        kani::assume(!path.starts_with(".factory/"));

        let registry = ArtifactRegistry::fixture_single_entry_block();
        let decision = match_path(&path, &registry);

        kani::assert(
            matches!(decision, PathDecision::Allow),
            "non-.factory/ path must always yield Allow"
        );
    }

    /// Empty path must yield Allow.
    #[kani::proof]
    fn proof_empty_path_is_allow() {
        let registry = ArtifactRegistry::fixture_single_entry_block();
        let decision = match_path("", &registry);
        kani::assert(
            matches!(decision, PathDecision::Allow),
            "empty path must yield Allow"
        );
    }

    /// Block decision only fires for .factory/ paths with enforcement_level=block.
    #[kani::proof]
    #[kani::unwind(16)]
    fn proof_block_only_on_factory_path_with_block_level() {
        let path: String = kani::any();
        kani::assume(path.len() <= 64);

        let registry = ArtifactRegistry::fixture_advisory_only();
        let decision = match_path(&path, &registry);

        // Registry with only advisory entries must never produce Block
        kani::assert(
            !matches!(decision, PathDecision::Block),
            "advisory-only registry must never produce Block for any path"
        );
    }
}
```
