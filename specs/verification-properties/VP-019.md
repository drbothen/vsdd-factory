# VP-019: Routing Is Deterministic — Same Input Yields Same Plugin Selection

## Property Statement

For a given registry and HookPayload, the output of match_plugins + group_by_priority is deterministic: the same event_name + tool_name always selects the same set of plugins grouped into the same tiers in the same order. No random or time-dependent factors influence routing.

## Source Contract

- **BC:** BC-1.01.005 — Plugin filter requires event match AND (no tool OR tool regex matches)
- **Postcondition/Invariant:** DI-001 (tier ordering) — tiers must be stable for dispatch to be predictable

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in routing.rs | yes — fixed registry + payload | match_plugins + group_by_priority |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_019_routing_is_deterministic() {
    let registry = make_registry_with_known_entries();
    let payload = make_payload("PreToolUse", "Bash");

    let result_1 = route(&registry, &payload);
    let result_2 = route(&registry, &payload);

    assert_eq!(result_1, result_2);
}
```
