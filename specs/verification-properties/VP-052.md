# VP-052: Epoch Ticker Shuts Down Cooperatively and Idempotently

## Property Statement

EpochTicker::shutdown sets the atomic stop flag and joins the ticker thread. The shutdown is cooperative: the ticker thread checks the flag and exits. Calling shutdown multiple times does not panic or deadlock. The epoch ticker advances the epoch every EPOCH_TICK_MS (10ms).

## Source Contract

- **BC:** BC-1.04.002 — Epoch ticker advances epoch every 10ms; cooperative shutdown
- **Postcondition/Invariant:** DI-001 — epoch ticker must run for the duration of plugin execution

#[test]

## Notes

NFR-REL-009 and NFR-REL-010 document cooperative shutdown and idempotence as reliability requirements. The epoch ticker is rebuilt per dispatcher invocation (NFR-PERF-008), so shutdown is always called exactly once per invocation in normal operation. The idempotence test covers the edge case of programmatic double-shutdown.
