# VP-053: Lobster Workflow DAG Is Acyclic — No Circular Dependencies

## Property Statement

Every .lobster workflow file has an acyclic depends_on graph. No step depends on itself or on a step that (transitively) depends on it. Sub-workflow invocations are resolved and checked. All depends_on references point to existing step names within the same workflow.

## Source Contract

- **BC:** BC-5.01.004 — Step ordering is by depends_on topological resolution
- **Postcondition/Invariant:** Workflow DAGs must be acyclic for topological sort to terminate

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| manual | YAML parsing + DFS cycle detection | no — all .lobster files | acyclicity + dangling reference check |

## Proof Harness Skeleton

```python
# validate_workflows.py
import yaml
from pathlib import Path

def check_acyclic(workflow_file):
    with open(workflow_file) as f:
        doc = yaml.safe_load(f)
    steps = {s['name']: s.get('depends_on', []) for s in doc['workflow']['steps']}
    # DFS cycle detection
    for step in steps:
        if has_cycle(step, steps, set()):
            raise ValueError(f"Cycle detected involving {step}")
    # Dangling reference check
    for step, deps in steps.items():
        for dep in deps:
            if dep not in steps:
                raise ValueError(f"Dangling reference: {step} -> {dep}")
```
