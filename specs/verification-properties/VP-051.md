# VP-051: Dispatcher Startup Flow Writes Parseable JSONL with Correct Envelopes

## Property Statement

The 4-event dispatcher startup flow (dispatcher.started, plugin.loaded, plugin.invoked, internal.dispatcher_error) writes exactly 4 parseable JSONL lines. Each line carries the correct event-specific envelope fields. The files are parseable by standard JSON parsers without a schema.

## Source Contract

- **BC:** BC-1.06.009 — startup_flow_writes_parseable_jsonl: 4-event flow round-trips through JSONL
- **Postcondition/Invariant:** DI-007 (always-on) + DI-017 (trace_id on every event)

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| integration | Rust integration test with JSONL parsing | yes — 4-event scenario | all startup events + envelope fields |

## Proof Harness Skeleton

```rust
// Integration test: crates/factory-dispatcher/tests/internal_log_integration.rs
#[test]
fn verify_vp_051_startup_flow_parseable() {
    let log = InternalLog::new(tempdir);
    write_startup_flow(&log, "trace-1");

    let lines = read_jsonl_lines(&log.current_file());
    assert_eq!(lines.len(), 4);

    for line in &lines {
        let v: serde_json::Value = serde_json::from_str(line).unwrap();
        assert!(v.get("dispatcher_trace_id").is_some());
        assert!(v.get("schema_version").is_some());
        assert!(v.get("ts").is_some());
    }
}
```
