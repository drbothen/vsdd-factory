# VP-012: Sink Failure Affects Only That Sink

## Property Statement

A failure in one sink driver (write error, read-only path, full queue) is recorded in that sink's Mutex<Vec<SinkFailure>> and does not propagate to or affect other sinks. Other sinks continue processing events normally.

## Source Contract

- **BC:** BC-3.01.008 — file sink failures recorded into Mutex<Vec<SinkFailure>>
- **Postcondition/Invariant:** DI-012 — A sink failure affects only that sink

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit test with read-only tempdir | yes — filesystem error condition | failure recording without panic + other sinks unaffected |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_012_failure_isolated_per_sink() {
    let bad_sink = FileSink::with_readonly_path();
    let good_sink = FileSink::with_tempdir();
    let registry = SinkRegistry::new(vec![bad_sink, good_sink]);

    registry.submit_all(make_event());
    registry.flush();

    // Bad sink recorded a failure
    assert!(!bad_sink.failures().is_empty());

    // Good sink still wrote the event
    assert_eq!(good_sink.event_count(), 1);
}
```
