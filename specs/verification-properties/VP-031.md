# VP-031: Tag Enrichment Does Not Overwrite Producer Fields

## Property Statement

Static tags configured for a sink (e.g., env=prod, team=factory) are merged onto every written event. A tag whose key conflicts with a reserved producer field (type, ts, ts_epoch, etc.) does NOT overwrite the producer field. Non-conflicting tags are always added.

## Source Contract

- **BC:** BC-3.04.004 — Router::submit applies RoutingFilter before delegating to each sink (wired dispatch)
- **Postcondition/Invariant:** BC-3.04.004 PC3+PC4 — tag enrichment applied before submit; producer fields win on collision

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in sink-file/lib.rs | yes — tag=type conflict + normal tags | enrichment + conflict avoidance |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_031_tags_enriched_no_overwrite() {
    let config = SinkConfig::with_tags([("env", "prod"), ("type", "injected")]);
    let event = make_event_with_type("plugin.invoked");

    let enriched = enrich_with_tags(event, &config.tags);

    // Normal tags added
    assert_eq!(enriched.get("env").and_then(|v| v.as_str()), Some("prod"));

    // Producer field NOT overwritten
    assert_eq!(enriched.get("type").and_then(|v| v.as_str()), Some("plugin.invoked"));
}
```
