# VP-003: block_intent Is Aggregate; Tier Runs to Completion

## Property Statement

Recording block_intent=true during tier execution does not abort the tier mid-flight. All plugins in the tier complete to a terminal state (Ok, Timeout, or Crashed) before the final exit code 2 is emitted. The block_intent flag is set if and only if on_error=block AND the plugin's exit code requests a block.

## Source Contract

- **BC:** BC-1.03.009 — block_intent set only when on_error=block AND plugin asks to block
- **Postcondition/Invariant:** DI-003 — block_intent is an aggregate; the tier runs to completion regardless

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_003_block_intent_does_not_abort_tier() {
    // Setup: multiple plugins in one tier, one with on_error=block requesting block
    // Execute full tier
    let summary = execute_tier(tier_plugins, inputs).await.unwrap();

    // All plugins must have produced a result
    assert_eq!(summary.per_plugin_results.len(), tier_plugins.len());

    // block_intent is set
    assert!(summary.block_intent);

    // But all siblings also have results (tier ran to completion)
    assert!(summary.per_plugin_results.iter().all(|r| r.result.is_terminal()));
}
```
