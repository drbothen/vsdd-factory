# VP-036: Disabled Sink Drops Every Event Without Writing

## Property Statement

A sink configured with enabled=false drops every submitted event and creates no output files. The sink's submit and flush methods return without performing any I/O. This enables operators to stage a sink config before enabling it.

## Source Contract

- **BC:** BC-3.02.012 — disabled_sink_drops_every_event: enabled=false → no file written, no events accepted
- **Postcondition/Invariant:** DI-013 — unknown/disabled sink types are non-fatal

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit test with enabled=false config | yes — N events submitted | no file created |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_036_disabled_sink_no_output() {
    let config = SinkConfig { enabled: false, path_template: "out.jsonl", .. };
    let sink = FileSink::new(config);

    sink.submit(make_event());
    sink.submit(make_event());
    sink.flush();

    // No file created
    assert!(!Path::new("out.jsonl").exists());
}
```
