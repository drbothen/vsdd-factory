# VP-008: Internal Log Filename Derived from Event Timestamp, Not Wall Clock

## Property Statement

The internal log filename dispatcher-internal-YYYY-MM-DD.jsonl derives its date component from event.ts[..10] (the event's own timestamp field), not from the system wall clock at write time. Events with past or future timestamps produce filenames matching those timestamps.

## Source Contract

- **BC:** BC-1.06.002 — Daily rotation by event timestamp produces separate files per UTC date
- **Postcondition/Invariant:** DI-008 — Internal log filenames are derived from event timestamps, not wall clock

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit test with injected timestamp | yes — fixed timestamp 2026-04-24 | date_stamp function |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_008_filename_from_event_timestamp() {
    let log = InternalLog::new(tempdir);
    let past_ts = "2026-04-24T12:00:00Z";

    // Write event with past timestamp
    log.write(&InternalEvent::with_ts(DISPATCHER_STARTED, past_ts));

    // File must be named after the event timestamp, not today
    let expected = format!("dispatcher-internal-{}.jsonl", &past_ts[..10]);
    assert!(log_dir.join(expected).exists());
}
```
