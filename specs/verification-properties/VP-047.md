# VP-047: Validator Hooks Exit 0 (Pass) or 2 (Block) — No Other Codes

## Property Statement

Hook scripts functioning as validators (protect-secrets.sh, destructive-command-guard.sh, factory-branch-guard.sh, brownfield-discipline.sh, red-gate.sh, and similar PreToolUse gates) exit with code 0 to pass or code 2 to block. No validator exits with any other code under normal operation.

## Source Contract

- **BC:** BC-7.01.006 — Validators exit 0 (pass) or 2 (block) — no other codes
- **Postcondition/Invariant:** DI-003 — block_intent is aggregate; validators must produce deterministic exit codes

| Method | Tool | Bounded? | Coverage |

## Proof Harness Skeleton

```bash
# Manual review checklist per validator script:
# 1. List all exit points (grep 'exit [^0]' hooks/protect-secrets.sh)
# 2. Verify each non-zero exit is exactly 'exit 2'
# 3. Verify no 'exit 1' or other codes in normal flow
# 4. Verify error conditions use 'exit 2' (block), not 'exit 1' (error)
```

## Notes

The legacy-bash-adapter maps exit 2 → Block and any other non-zero → Error. An Error exit from a validator still allows the tool call to proceed (on_error=continue is the default). Validators must use exactly exit 2 to block. Any exit code that accidentally sends error (exit 1) would silently pass the tool call through.
