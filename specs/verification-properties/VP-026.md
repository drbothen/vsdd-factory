# VP-026: InternalEvent Serializes Flat with No Null Optional Fields

## Property Statement

InternalEvent fields serialize flat to top-level JSON (no nested `fields` wrapper). Optional fields that are None are absent from the JSON output (not serialized as null). This preserves downstream jq compatibility and OTLP attribute promotion.

## Source Contract

- **BC:** BC-1.06.006 — InternalEvent fields flatten to top-level JSON (no nested fields)
- **Postcondition/Invariant:** BC-1.06.008 — None optional fields skipped (not null)

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in internal_log.rs | yes — minimal event with all optional fields absent | serde flatten + skip_serializing_if |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_026_flat_json_no_nulls() {
    let event = InternalEvent::with_ts(DISPATCHER_STARTED, "2026-04-25T00:00:00Z");
    let json = serde_json::to_string(&event).unwrap();
    let parsed: serde_json::Value = serde_json::from_str(&json).unwrap();

    // Must be flat object — no "fields" key
    assert!(parsed.get("fields").is_none());

    // Optional fields absent when None
    assert!(parsed.get("dispatcher_trace_id").is_none());
    assert!(parsed.get("session_id").is_none());
    assert!(parsed.get("plugin_name").is_none());
}
```
