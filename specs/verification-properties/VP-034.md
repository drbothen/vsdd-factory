# VP-034: OTLP Sink Batch Trigger Thresholds Are Independent

## Property Statement

The OTLP sink flushes a batch when either: (1) batch.size events are queued, OR (2) batch.interval_ms has elapsed since the last flush. The two conditions are independent — either alone can trigger a flush. Producer-side submit is non-blocking (try_send). Send failures drop the gRPC client for self-healing reconnection.

## Source Contract

- **BC:** BC-3.03.001 — Batch trigger thresholds are independent: size (default 100) AND interval_ms (default 5000)
- **Postcondition/Invariant:** DI-011 — submit is non-blocking

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_034_batch_size_triggers_flush() {
    let sink = OtelGrpcSink::with_batch_size(5);

    // Submit batch_size events
    for _ in 0..5 {
        sink.submit(make_event());
    }

    // Flush should have been triggered (batch size reached)
    sink.flush();
    assert!(sent_count >= 5);
}
```
