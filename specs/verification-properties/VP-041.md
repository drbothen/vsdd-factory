# VP-041: SDK Panic Handler Extracts Message for All Payload Types

## Property Statement

The SDK's internal panic handler extracts the panic message string for &str panics and String panics. For unknown panic payload types, it returns the fallback "(no panic message)". No panic in the SDK causes an unhandled abort that bypasses the dispatcher's crash reporting.

## Source Contract

- **BC:** BC-2.05.001 — panic_message_extracts_static_str: panic of &str is extracted
- **Postcondition/Invariant:** DI-002 — plugin crash does not block siblings; crash is reported cleanly

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in hook-sdk/__internal.rs | yes — 3 panic payload types | &str, String, unknown |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_041_panic_handler_all_types() {
    // &str panic
    let msg = extract_panic_message(&"static message");
    assert_eq!(msg, "static message");

    // String panic
    let msg = extract_panic_message(&"owned string".to_string());
    assert_eq!(msg, "owned string");

    // Unknown type fallback
    let msg = extract_panic_message(&42u32);
    assert_eq!(msg, "(no panic message)");
}
```

## Notes

The SDK's panic handler converts panics into PluginResult::Crashed with the extracted message. This ensures all plugin panics (including panics in user-written hook plugins) are captured by the dispatcher's crash isolation mechanism (DI-002).
