# VP-013: Unknown Sink Driver Types Are Non-Fatal

## Property Statement

If observability-config.toml names an unrecognized driver type, the dispatcher emits a warning to stderr, skips that sink, and continues loading all other sink entries. No error is returned; no events are dropped from functioning sinks.

## Source Contract

- **BC:** BC-3.01.002 — Unknown sink type warns to stderr but does not fail config load
- **Postcondition/Invariant:** DI-013 — Unknown sink driver types are non-fatal

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust integration test | yes — one unknown sink + two known | warn-and-skip behavior |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_013_unknown_sink_is_nonfatal() {
    let config = config_with_unknown_sink_type("datadog");
    let registry = SinkRegistry::from_config(config).unwrap();

    // Registry loads successfully despite unknown type
    assert!(registry.is_ok());

    // Known sinks still loaded
    assert_eq!(registry.unwrap().sinks().len(), known_sink_count);
}
```

## Notes

The warning is emitted to stderr (not internal log) because the internal log may not yet be initialized when config is loaded.
