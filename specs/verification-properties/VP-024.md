# VP-024: Plugin Cache Is Keyed by Path and Invalidated by mtime

## Property Statement

PluginCache uses path as the cache key. A cache hit occurs only when the path has been seen before AND the file's mtime is unchanged. A changed mtime triggers recompilation. Missing plugin path returns NotFound; corrupt bytes return Compile error. The cache is thread-safe via Mutex<HashMap>.

## Source Contract

- **BC:** BC-1.09.001 — PluginCache key is path only; invalidation is mtime-driven
- **Postcondition/Invariant:** Plugins updated on disk are always recompiled; no stale module served

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_024_cache_invalidated_on_mtime_change() {
    let cache = PluginCache::new();
    let path = write_wasm_to_temp();

    // First load → compiles
    let module_1 = cache.get_or_compile(&path).unwrap();

    // Touch file (mtime change)
    touch(&path);

    // Second load → recompiles (not the cached module)
    let module_2 = cache.get_or_compile(&path).unwrap();

    // Both succeed; modules are distinct instances
    assert!(module_1.is_valid());
    assert!(module_2.is_valid());
}
```
