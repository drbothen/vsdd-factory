# VP-028: Sink Fan-Out — Every Event Reaches Every Configured Accepting Sink

## Property Statement

SinkRegistry.submit_all sends each event to every enabled sink whose RoutingFilter accepts it. An event accepted by N sinks is delivered to all N sinks. Sinks with allow-list filters receive only matching events. The fan-out is independent of sink type.

## Source Contract

- **BC:** BC-3.05.002 — registry_fans_events_to_file_sinks_with_filter_and_tags
- **Postcondition/Invariant:** DI-011 — submit is non-blocking; DI-012 — failure in one sink does not affect others

## Notes

Fan-out is implemented by SinkRegistry::submit_all iterating over all sinks and calling try_send on each. The non-blocking property (DI-011) means events may be dropped if a sink queue is full; the fan-out guarantee applies only to non-full queues.
