# VP-045: Legacy Bash Adapter Strips plugin_config Before Piping to Bash

## Property Statement

The legacy-bash-adapter re-serializes the payload with plugin_config set to null before piping to bash. The bash script receives a clean payload with no plugin_config leakage. event_name and dispatcher_trace_id are preserved in the re-serialized payload.

## Source Contract

- **BC:** BC-4.01.002 — legacy-bash-adapter strips plugin_config to Null before piping payload to bash
- **Postcondition/Invariant:** DI-016 — config isolation; bash scripts must not receive adapter internals

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_045_plugin_config_stripped() {
    let mut captured_bytes = vec![];
    let payload = make_payload_with_plugin_config(serde_json::json!({
        "script_path": "hooks/test.sh",
        "extra": 1
    }));

    adapter_logic(&payload, &caps, &mut |bytes: &[u8]| {
        captured_bytes = bytes.to_vec();
        Ok((0, vec![], vec![]))
    });

    let piped: serde_json::Value = serde_json::from_slice(&captured_bytes).unwrap();
    assert!(piped["plugin_config"].is_null());
    assert_eq!(piped["event_name"].as_str(), Some("PreToolUse"));
}
```
