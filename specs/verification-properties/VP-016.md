# VP-016: Each Registry Entry Sees Only Its Own plugin_config

## Property Statement

The executor splices only the entry's own config block as plugin_config into the payload clone for that entry. The plugin_config from any other registry entry is never visible to a different plugin instance. Each plugin invocation receives exactly its own config.

## Source Contract

- **BC:** BC-1.03.010 — Per-plugin plugin_config spliced into HookPayload before invocation
- **Postcondition/Invariant:** DI-016 — Each registry entry sees only its own plugin_config

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit test with two entries | yes — two-entry scenario | config isolation per entry |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_016_plugin_config_isolation() {
    // Two entries with different script_path configs
    let entry_a = make_entry("a", config_a);
    let entry_b = make_entry("b", config_b);

    let payload_a = splice_plugin_config(&base_payload, &entry_a);
    let payload_b = splice_plugin_config(&base_payload, &entry_b);

    // Each payload has its own config
    assert_eq!(payload_a.plugin_config, config_a);
    assert_eq!(payload_b.plugin_config, config_b);

    // Cross-contamination is absent
    assert_ne!(payload_a.plugin_config, config_b);
    assert_ne!(payload_b.plugin_config, config_a);
}
```
