# VP-015: Per-Project Activation Required Before Dispatcher Can Run

## Property Statement

hooks.json is gitignored and must be written by the activate skill before the dispatcher can be invoked. Without activation, no hooks.json exists and Claude Code cannot invoke the dispatcher. The activate skill writes a platform-specific hooks.json variant using the correct binary path for the current OS/arch.

## Source Contract

- **BC:** BC-9.01.005 — hooks.json is gitignored; hooks.json.template + per-platform variants are committed (gate artifact)
- **Co-anchor:** BC-9.01.004 — 5-platform CI matrix is the build matrix (gate prerequisite)
- **Postcondition/Invariant:** DI-015 — Per-project activation is required before the dispatcher can run

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| manual | Code review + CI platform test matrix | no | gitignore check + activation flow |

## Proof Harness Skeleton

```
Manual verification checklist:
1. Confirm plugins/vsdd-factory/hooks/.gitignore lists hooks.json
2. Confirm hooks.json.* variants exist for each supported platform
3. Confirm activate skill copies hooks.json.<platform> to hooks.json
4. Confirm hooks.json.<platform> files reference the correct binary path per platform
5. Confirm CI runs activate before any dispatcher test
```
