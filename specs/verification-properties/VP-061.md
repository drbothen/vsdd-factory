# VP-061: Agent Prompt Discipline Rules Are Present in All Three Agent Files

## Property Statement

After delivery of S-7.01, the three agent prompt files (`story-writer.md`, `product-owner.md`, `adversary.md`) collectively contain all required discipline rules: (1) story-writer's Spec-First Gate preventing `status: ready` with empty `behavioral_contracts:`, (2) story-writer's AC↔BC bidirectional trace requirement, (3) product-owner's Capability Anchor Justification requirement with verbatim citation, (4) adversary's Partial-Fix Regression axis checking primary file + sibling files + prose, and (5) adversary's explicit enumeration of all three propagation surfaces. No existing rule in any of the three files has been removed or semantically weakened.

## Source Contract

- **Primary: BC-5.36.001** — Spec-First Gate (story-writer)
- **BC-5.36.002** — AC↔BC bidirectional trace requirement (story-writer)
- **BC-5.36.003** — Capability Anchor Justification presence (product-owner)
- **BC-5.36.004** — Verbatim citation requirement (product-owner)
- **BC-5.36.005** — Partial-Fix Regression axis (adversary)
- **BC-5.36.006** — Three-surface propagation check (adversary)
- **BC-5.36.007** — Atomic delivery of all three files
