# VP-072: Single-Source-of-Truth Invariant — No Skill or Hook Embeds a Duplicate Artifact Path List

## Property Statement

`plugins/vsdd-factory/config/artifact-path-registry.yaml` is the **sole**
canonical list of `.factory/` artifact path patterns in the vsdd-factory
codebase. No skill file, WASM hook source file, or bash hook script may
contain a hardcoded duplicate path list that is also expressible via an entry
in the registry.

Formally, for every `canonical_path_pattern` entry in the registry, the
following invariant holds across the full codebase:

```
∀ pattern P in artifact-path-registry.yaml:
    grep_count(P.literal_prefix, skills/ ++ hooks/) == 0
```

Where `P.literal_prefix` is the fixed path prefix of the pattern before the
first `{placeholder}` — e.g., for `.factory/specs/behavioral-contracts/ss-`
this is `.factory/specs/behavioral-contracts/`.

A "duplicate path list" is any static enumeration of two or more `.factory/`
path patterns that overlaps with registry entries. A single path reference for
a specific artifact (e.g., a skill writing to its own known output) is NOT
considered a duplicate list — only enumerations of multiple artifact types.

## Rationale for this VP

ADR-016 establishes the single-source-of-truth invariant. This VP verifies it
mechanically. Without this VP, a future skill could silently accumulate a
private path list that diverges from the registry — exactly the problem ADR-016
is designed to prevent. The integration test acts as a continuous drift guard.

## Formal Property Statement (bats integration test)

```bash
#!/usr/bin/env bats
# VP-072: Single-source-of-truth invariant.
# Verifies no skill or hook embeds a duplicate artifact path list
# that overlaps with artifact-path-registry.yaml.

REGISTRY="plugins/vsdd-factory/config/artifact-path-registry.yaml"
SKILLS_DIR="plugins/vsdd-factory/skills"
HOOKS_DIR="plugins/vsdd-factory/hooks"
HOOK_SRC_DIR="crates/hook-plugins"

# Extract literal prefixes from registry (prefix before first {placeholder})
@test "VP-072: artifact-path-registry.yaml exists and is parseable" {
    [ -f "$REGISTRY" ]
    # Verify it has at least one artifact entry
    grep -q "artifact_type:" "$REGISTRY"
}

@test "VP-072: no skill file embeds a hardcoded .factory/specs/behavioral-contracts path list" {
    # Detect any skill SKILL.md that contains 2+ lines matching the BC path pattern
    # A single reference is allowed; a list (2+) indicates a duplicate registry
    local bc_pattern=".factory/specs/behavioral-contracts"
    local violations=0
    while IFS= read -r -d '' skill_file; do
        local count
        count=$(grep -c "$bc_pattern" "$skill_file" 2>/dev/null || true)
        if [ "$count" -ge 2 ]; then
            echo "VIOLATION: $skill_file contains $count references to $bc_pattern"
            violations=$((violations + 1))
        fi
    done < <(find "$SKILLS_DIR" -name "SKILL.md" -print0)
    [ "$violations" -eq 0 ]
}

@test "VP-072: no skill file embeds a hardcoded .factory/specs/verification-properties path list" {
    local vp_pattern=".factory/specs/verification-properties"
    local violations=0
    while IFS= read -r -d '' skill_file; do
        local count
        count=$(grep -c "$vp_pattern" "$skill_file" 2>/dev/null || true)
        if [ "$count" -ge 2 ]; then
            echo "VIOLATION: $skill_file contains $count references to $vp_pattern"
            violations=$((violations + 1))
        fi
    done < <(find "$SKILLS_DIR" -name "SKILL.md" -print0)
    [ "$violations" -eq 0 ]
}

@test "VP-072: validate-artifact-path hook source does not embed artifact paths" {
    local hook_src="$HOOK_SRC_DIR/validate-artifact-path/src/lib.rs"
    # The hook must read registry at runtime; it must not contain hardcoded
    # .factory/specs/* path literals (beyond STATE_FILE_PATH constants)
    if [ -f "$hook_src" ]; then
        local bc_count
        bc_count=$(grep -c ".factory/specs/behavioral-contracts" "$hook_src" 2>/dev/null || true)
        local vp_count
        vp_count=$(grep -c ".factory/specs/verification-properties" "$hook_src" 2>/dev/null || true)
        [ "$bc_count" -eq 0 ]
        [ "$vp_count" -eq 0 ]
    fi
}

@test "VP-072: no bash hook embeds a duplicate .factory artifact path list" {
    local factory_pattern=".factory/"
    local violations=0
    while IFS= read -r -d '' hook_file; do
        # Count .factory/ references; a list of 3+ distinct paths is suspicious
        local count
        count=$(grep -c ".factory/" "$hook_file" 2>/dev/null || true)
        if [ "$count" -ge 3 ]; then
            # Check if it references paths that are in the registry
            # (heuristic: if it contains both "specs/" and "cycles/" it's likely a list)
            if grep -q ".factory/specs/" "$hook_file" && grep -q ".factory/cycles/" "$hook_file"; then
                echo "REVIEW NEEDED: $hook_file may contain a path list (specs + cycles refs)"
                # Note: this is a warning, not a hard block; human review required
            fi
        fi
    done < <(find "$HOOKS_DIR" -name "*.sh" -print0)
    # Hard assertion: validate-factory-path-root.sh is the only bash hook
    # allowed to enumerate .factory/ paths (it enforces root-level path validation)
    [ "$violations" -eq 0 ]
}
```

## Expected Outcome

All bats test cases pass. The `validate-artifact-path` WASM hook source
contains no hardcoded `canonical_path_pattern` values — it reads the registry
file via `host::read_file` at runtime. Skill SKILL.md files that reference
`.factory/` paths contain at most one reference per path pattern (specific
artifact references, not lists).

This test is designed to be fast (filesystem scan, no process spawning beyond
`find` and `grep`) and suitable for inclusion in the CI pre-merge gate.

## Limitations

This VP uses heuristic grep-based detection. It cannot distinguish between:
- A skill that references the same path pattern twice for documentation purposes
- A skill that maintains a private duplicate list

Human review is required when the heuristic threshold (2+ occurrences) fires.
The threshold is intentionally conservative to minimize false negatives (missed
duplicates) at the cost of occasional false positives (documentation repetition).

For the WASM hook source, the check is stricter (zero occurrences of known
artifact type path prefixes), because the hook's design intent is explicit: it
must read from the registry file, never from hardcoded literals.
