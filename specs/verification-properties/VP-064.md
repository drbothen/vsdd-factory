# VP-064: facade-mode mutation gate enforcement — wave-gate skill executes cargo mutants and blocks if kill rate < 80%

## Property Statement

For every wave that contains at least one story with `tdd_mode: facade` in its frontmatter:
1. The wave-gate skill MUST execute `cargo mutants -p <crate> --jobs N --timeout 300` for each facade-mode crate
2. The wave-gate skill MUST produce a mutation report at `.factory/logs/mutation-report-<wave-id>-<crate>.md`
3. If kill rate < 80% for any facade crate AND no disposition (A/B/C per BC-6.21.002) is documented for surviving mutants above the 20% allowance, the wave gate MUST fail
4. If `cargo-mutants` is not installed, the wave gate MUST fail with a clear install instruction
5. A wave with ZERO facade stories is not required to run mutation testing

## Source Contract

- **Primary: BC-6.21.001** — wave-gate skill must run cargo mutants for every facade story
- **BC-6.21.002** — kill rate floor 80%; surviving mutants require disposition A, B, or C
- **BC-8.30.002** — facade mode's compensating control is mutation testing; bypass of Red Gate is only valid because this gate exists

## Proof Harness Skeleton

```bash
# Procedural verification checklist executed at wave gate runtime
# Location: plugins/vsdd-factory/skills/wave-gate/SKILL.md implementation

# Step 1: Identify facade stories in wave
FACADE_STORIES=$(grep -rl "tdd_mode: facade" .factory/stories/ | \
  xargs grep -l "wave: <current_wave>")

# Step 2: Skip if no facade stories
if [ -z "$FACADE_STORIES" ]; then
  echo "[VP-064] No facade stories in wave — mutation testing step skipped"
  exit 0
fi

# Step 3: Verify cargo-mutants is installed
if ! cargo mutants --version >/dev/null 2>&1; then
  echo "[VP-064] ERROR: cargo-mutants not found. Install with: cargo install cargo-mutants"
  exit 1
fi

# Step 4: For each facade crate, run mutation testing
for STORY in $FACADE_STORIES; do
  CRATE=$(grep "crate:" "$STORY" | head -1 | awk '{print $2}')
  WAVE_ID=$(grep "wave:" "$STORY" | head -1 | awk '{print $2}')
  REPORT=".factory/logs/mutation-report-${WAVE_ID}-${CRATE}.md"

  echo "[VP-064] Running cargo mutants for crate: $CRATE"
  cargo mutants -p "$CRATE" --jobs "$(nproc)" --timeout 300 \
    --output "$REPORT" 2>&1

  # Step 5: Extract kill rate from report
  TOTAL=$(grep "^mutants tested:" "$REPORT" | awk '{print $3}')
  KILLED=$(grep "^mutants killed:" "$REPORT" | awk '{print $3}')

  if [ "$TOTAL" -eq 0 ]; then
    echo "[VP-064] WARNING: No mutants generated for $CRATE — vacuous pass"
    continue
  fi

  # Integer arithmetic: killed * 100 / total >= 80
  KILL_RATE=$(( KILLED * 100 / TOTAL ))
  echo "[VP-064] Kill rate for $CRATE: ${KILL_RATE}%"

  if [ "$KILL_RATE" -lt 80 ]; then
    echo "[VP-064] BLOCK: Kill rate ${KILL_RATE}% < 80% for $CRATE"
    echo "         Surviving mutants must be dispositioned (A/B/C per BC-6.21.002)"
    echo "         See: $REPORT"
    GATE_FAILED=true
  fi
done

# Step 6: Verify surviving mutant dispositions (manual check)
# Each surviving mutant above the 20% allowance must have:
# A: new test committed that kills it (verify by re-running cargo mutants)
# B: dead-code-equivalent explanation in PR description
# C: explicit waiver with mutant name in PR description

if [ "$GATE_FAILED" = "true" ]; then
  echo "[VP-064] Wave gate blocked — disposition required for surviving mutants"
  exit 1
fi

echo "[VP-064] Mutation gate passed for all facade crates"
```

```markdown
# Manual Verification Checklist (run at wave gate review)

## Facade Story Detection
- [ ] All stories in wave scanned for `tdd_mode: facade`
- [ ] Each facade story's crate name extracted

## cargo-mutants Execution
- [ ] `cargo mutants` executed for each facade crate
- [ ] `--timeout 300` used (not higher — higher indicates flaky tests)
- [ ] Mutation report written to `.factory/logs/mutation-report-<wave-id>-<crate>.md`
- [ ] Report is committed to the wave PR

## Kill Rate Threshold
- [ ] Kill rate computed as: killed * 100 / total (integer arithmetic)
- [ ] Kill rate ≥ 80% for each facade crate

## Surviving Mutant Dispositions (if kill rate < 80%)
- [ ] Each surviving mutant above 20% allowance has a documented disposition
- [ ] Disposition A: new test committed + re-run confirms mutant killed
- [ ] Disposition B: dead-code-equivalent explanation in PR description with execution condition
- [ ] Disposition C: explicit waiver in PR description naming specific mutant (file, line, mutation type)
- [ ] No "blanket waiver" language accepted

## Not-Installed Gate
- [ ] `cargo mutants --version` succeeds before attempting run
- [ ] If not installed: wave gate blocks with install instruction
```

## Notes

VP-064 is a procedural verification property rather than an automated formal proof. The mutation testing itself is the proof of test coverage quality — VP-064 verifies that the pipeline correctly invokes and gates on mutation testing results. The manual checklist approach is appropriate because the wave-gate skill is a Markdown prompt document (not an executable binary), and its "execution" involves an LLM agent following instructions. The proof method is therefore procedural review: at each wave gate execution, verify the checklist is followed by auditing the mutation report and PR description for disposition evidence. The first live wave gate execution with a facade story constitutes the initial proof.
