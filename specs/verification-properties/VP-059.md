# VP-059: ID Monotonicity — Allocated ADR-NNN is Strictly Greater Than All Existing IDs

## Property Statement

For any successful invocation of `create-adr` that allocates a new ADR ID (auto-allocation path): the allocated ID `ADR-N` satisfies `N > max(all existing ADR numbers in filesystem ∪ ARCH-INDEX)`. There is no invocation that produces a duplicate, a lower, or an equal ID relative to any existing ADR-NNN already recorded in either source.

## Source Contract

- **BC:** BC-6.20.001 — Auto-allocation: proposes `ADR-<max+1>`
- **BC:** BC-6.20.002 — Duplicate `--id` override refused
- **BC:** BC-6.20.003 — Filesystem/ARCH-INDEX mismatch blocks before allocation
- **Postcondition:** "The proposed ID is always strictly greater than any existing ADR-NNN found in either source."

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| proptest | Bash property-test loop over randomly generated fixture states | yes (bounded fixture sizes 0–50 ADRs) | auto-allocation path across all fixture sizes |

## Proof Harness Skeleton

```bash
# Property test: plugins/vsdd-factory/skills/create-adr/tests/test_id_monotonicity.sh
#
# Strategy: generate N random fixture states (0..50 ADRs, consistent filesystem+index),
# invoke create-adr --dry-run (or capture proposed ID without writing),
# assert proposed ID > max fixture ID.

for trial in $(seq 1 200); do
  n_adrs=$((RANDOM % 51))          # 0..50 existing ADRs
  max_id=$(generate_fixture $n_adrs)   # writes ADR-001..ADR-N to fixture dir + ARCH-INDEX

  proposed=$(invoke create-adr --title "Test $trial" --dry-run 2>&1 | grep "Proposed ID" | awk '{print $3}')
  proposed_n=$(echo $proposed | sed 's/ADR-//' | sed 's/^0*//')

  assert [ "$proposed_n" -gt "$max_id" ] \
    "Trial $trial: proposed ADR-$(printf '%03d' $proposed_n) not > ADR-$(printf '%03d' $max_id)"

  teardown_fixture
done
echo "VP-059: 200 trials passed"
```

| Proof complexity | Low — arithmetic comparison | The property is a simple greater-than check |
| Tool support | High — Bash arithmetic, grep | No compiled tooling required; requires `--dry-run` flag in skill or equivalent probe |
| Estimated proof time | 1 hour for harness + 200 trials run in seconds | Fast property check |

## Feasibility Note

The `--dry-run` flag (or equivalent probe mechanism that outputs the proposed ID without writing) must be implemented in the skill for this proof harness to work without filesystem side-effects per trial. If `--dry-run` is not implemented, the harness must invoke normally and then delete the created file — acceptable but slower.
