# VP-040: SDK HookPayload Round-Trips via Serde and Carries plugin_config

## Property Statement

SDK-side HookPayload serializes and deserializes correctly: event_name and session_id are preserved through a serde round-trip. When the registry sets plugin_config, it arrives populated in the SDK payload. tool_response is present for PostToolUse and absent for PreToolUse.

## Source Contract

- **BC:** BC-2.04.004 — payload_round_trip_via_serde: serialize → deserialize preserves event_name and session_id
- **Postcondition/Invariant:** DI-016 — each entry sees only its own plugin_config

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in hook-sdk/payload.rs | yes — 4 payload variants | PreToolUse, PostToolUse, lifecycle, round-trip |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_040_payload_round_trip() {
    let payload = HookPayload {
        event_name: "PreToolUse".into(),
        session_id: "sess-1".into(),
        ..Default::default()
    };

    let json = serde_json::to_string(&payload).unwrap();
    let decoded: HookPayload = serde_json::from_str(&json).unwrap();

    assert_eq!(decoded.event_name, "PreToolUse");
    assert_eq!(decoded.session_id, "sess-1");
}
```
