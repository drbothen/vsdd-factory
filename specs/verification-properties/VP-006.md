# VP-006: Setuid/Setgid Binaries Refused Unconditionally

## Property Statement

On Unix systems, exec_subprocess refuses to execute any binary that has the setuid or setgid bits set, regardless of capability configuration, binary_allow list, or shell_bypass_acknowledged. This refusal is not overridable by any registry configuration.

## Source Contract

- **BC:** BC-1.05.004 — exec_subprocess refuses setuid/setgid binaries categorically (Unix)
- **Postcondition/Invariant:** DI-006 — Setuid/setgid binaries are refused unconditionally on Unix

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in exec_subprocess.rs | yes — Unix-specific path | refuse_setuid check before spawn |

## Proof Harness Skeleton

```rust
#[cfg(unix)]
#[test]
fn verify_vp_006_setuid_refused_unconditionally() {
    use std::os::unix::fs::PermissionsExt;

    // Create a binary with setuid bit set
    let tmp = tempfile::NamedTempFile::new().unwrap();
    let mut perms = tmp.as_file().metadata().unwrap().permissions();
    perms.set_mode(0o4755); // setuid bit
    tmp.as_file().set_permissions(perms).unwrap();

    // Even with a permissive capability, setuid must be denied
    let caps = fully_permissive_caps();
    let result = exec_subprocess_with_caps(&caps, tmp.path(), &[], 100, 1024);

    assert_eq!(result, Err(CAPABILITY_DENIED));
}
```
