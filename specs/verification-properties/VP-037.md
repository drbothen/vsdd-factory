# VP-037: OTLP Resource Attributes — Operator Overrides Win Over Auto-Detected Defaults

## Property Statement

The OTLP sink merges auto-detected resource attributes (service.name, service.version, host.name) with operator-configured overrides. Operator-configured values always win over auto-detected defaults. The merged attributes appear on all LogRecords in the resource.attributes field.

## Source Contract

- **BC:** BC-3.03.013 — resource_attributes_merge_defaults_with_config: operator overrides win over auto-detected
- **Postcondition/Invariant:** DI-012 — sink failure isolated; resource attributes correct regardless of merge order

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_037_operator_overrides_win() {
    let config = OtelGrpcConfig {
        resource_attributes: BTreeMap::from([
            ("service.name".into(), "my-custom-service".into()),
        ]),
        ..Default::default()
    };

    let merged = build_resource_attributes(&config);

    // Operator value wins
    assert_eq!(merged.get("service.name"), Some(&"my-custom-service".into()));
```
