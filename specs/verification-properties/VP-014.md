# VP-014: Schema Version Mismatch Is a Hard Load Error

## Property Statement

REGISTRY_SCHEMA_VERSION=1 and schema_version=1 in observability config must match. Any mismatch causes a hard error (load returns Err, internal.dispatcher_error is emitted) and the dispatcher exits cleanly. No silent fallback processing with a mismatched schema occurs.

## Source Contract

- **BC:** BC-1.01.001 — Registry rejects unknown schema version
- **Postcondition/Invariant:** DI-014 — Schema version mismatch is a hard load error

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in registry.rs and sinks/mod.rs | yes — version values 0, 2, 999 | schema version validation path |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_014_schema_version_mismatch_is_hard_error() {
    for bad_version in &[0u32, 2, 999] {
        let toml = format!("schema_version = {}\n[[hooks]]\n...", bad_version);
        let result = Registry::parse_str(&toml);
        assert!(result.is_err(), "version {} should fail", bad_version);
    }

    // Correct version succeeds
    let good_toml = "schema_version = 1\n[[hooks]]\n...";
    assert!(Registry::parse_str(good_toml).is_ok());
}
```
