# VP-014: Schema Version Mismatch Is a Hard Load Error

## Property Statement

REGISTRY_SCHEMA_VERSION=2 (post-ADR-019) and schema_version=2 in observability config must match their respective expected versions. Any mismatch causes a hard error (load returns Err, internal.dispatcher_error is emitted) and the dispatcher exits cleanly. No silent fallback processing with a mismatched schema occurs.

## Source Contract

- **BC:** BC-1.01.001 — Registry rejects unknown schema version
- **Postcondition/Invariant:** DI-014 — Schema version mismatch is a hard load error

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in registry.rs and sinks/mod.rs | yes — version values 0, 1, 999 | schema version validation path |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_014_schema_version_mismatch_is_hard_error() {
    for bad_version in &[0u32, 1, 999] {
        let toml = format!("schema_version = {}\n[[hooks]]\n...", bad_version);
        let result = Registry::parse_str(&toml);
        assert!(result.is_err(), "version {} should fail", bad_version);
    }

    // Correct version succeeds (schema_version = 2 post-ADR-019)
    let good_toml = "schema_version = 2\n[[hooks]]\n...";
    assert!(Registry::parse_str(good_toml).is_ok());
}
```

## Notes

Both registry and observability-config schema versions are validated at load time. REGISTRY_SCHEMA_VERSION = 2 (post-ADR-019; was 1 pre-2026-05-07) and INTERNAL_EVENT_SCHEMA_VERSION = 1 are separate compile-time constants. A kani proof could exhaustively verify all u32 values != 2 (for registry) and != 1 (for INTERNAL_EVENT) produce Err.
