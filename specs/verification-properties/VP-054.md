# VP-054: Workflow Loop Blocks Are Bounded — max_iterations and exit_condition Required

## Property Statement

Every loop: block in a .lobster workflow declares both max_iterations (a positive integer) and exit_condition (a non-empty expression). Unbounded loops (missing either field) are a schema validation error. The orchestrator enforces max_iterations as a hard cap.

## Source Contract

- **BC:** BC-5.01.007 — loop blocks are bounded; require max_iterations and exit_condition
- **Postcondition/Invariant:** No workflow loop can run infinitely

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| manual | YAML schema validation of all .lobster files | no — all workflow files | loop block field presence |

## Proof Harness Skeleton

```python
# validate_loop_bounds.py
for workflow in lobster_files:
    for step in workflow['steps']:
        if step.get('type') == 'loop':
            assert 'max_iterations' in step['loop'], f"Missing max_iterations in {step['name']}"
            assert step['loop']['max_iterations'] > 0
            assert step['loop'].get('exit_condition'), f"Missing exit_condition in {step['name']}"
```
