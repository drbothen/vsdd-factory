# VP-027: HookPayload Parsing Is Robust for All Claude Code Envelope Types

## Property Statement

HookPayload::from_bytes successfully parses PreToolUse, PostToolUse, and lifecycle (SessionStart/End) envelopes. Both event_name and hook_event_name field names are accepted (serde alias). Non-empty event_name and session_id are required.

## Source Contract

- **BC:** BC-1.02.001 — HookPayload requires non-empty event_name and session_id
- **Postcondition/Invariant:** All Claude Code hook event shapes parse correctly

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in payload.rs | yes — 3 envelope types | PreToolUse, PostToolUse, SessionStart |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_027_all_payload_types_parse() {
    // PreToolUse
    let pre = HookPayload::from_bytes(pretooluse_json).unwrap();
    assert!(pre.tool_response.is_none());

    // PostToolUse
    let post = HookPayload::from_bytes(posttooluse_json).unwrap();
    assert!(post.tool_response.is_some());

    // SessionStart (no tool_name)
    let sess = HookPayload::from_bytes(sessionstart_json).unwrap();
    assert_eq!(sess.tool_name, "");

    // hook_event_name alias works
    let aliased = HookPayload::from_bytes(hook_event_name_json).unwrap();
    assert!(!aliased.event_name.is_empty());
}
```
