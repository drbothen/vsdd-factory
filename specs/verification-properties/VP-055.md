# VP-055: state-manager Runs Last in Every Burst

## Property Statement

In every orchestrator burst, state-manager is the final agent to execute. It commits all .factory/ artifact changes to git. No other agent writes .factory/ files after state-manager has committed in a burst. The orchestrator never delegates writes directly.

## Source Contract

- **BC:** BC-5.02.005 — orchestrator: state-manager runs LAST in every burst
- **Postcondition/Invariant:** Artifact write ordering is deterministic; no interleaved writes

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| manual | Orchestrator agent behavior review | no — behavioral contract | burst execution ordering |

## Proof Harness Skeleton

```
Manual verification checklist:
1. Review orchestrator agent instructions for burst ordering
2. Confirm state-manager is always last in burst dispatch list
3. Confirm no other agent calls state-manager or writes .factory/ directly
4. Confirm orchestrator never delegates writes via direct tool calls
```
