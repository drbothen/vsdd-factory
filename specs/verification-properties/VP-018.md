# VP-018: Registry Rejects Malformed Configurations at Load Time

## Property Statement

Registry::load rejects: (1) unknown schema version, (2) invalid tool regex, (3) unknown entry fields (typo guard), and (4) unknown on_error values. All rejections return Err before any plugin execution occurs. Malformed configs cannot reach the routing or execution layer.

## Source Contract

- **BC:** BC-1.01.001 — Registry rejects unknown schema version
- **Postcondition/Invariant:** DI-014 — Schema version mismatch is a hard load error

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in registry.rs | yes — 4 malformation categories | parse-time validation |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_018_registry_rejects_malformed() {
    // Unknown schema version
    assert!(Registry::parse_str("schema_version = 99\n...").is_err());

    // Unknown on_error value
    assert!(Registry::parse_str("...\non_error = \"shout\"\n...").is_err());

    // Invalid regex
    assert!(Registry::parse_str("...\ntool = \"[unclosed\"\n...").is_err());

    // Unknown entry fields
    assert!(Registry::parse_str("...\ntypo_field = true\n...").is_err());
}
```
