# VP-058: create-adr Atomicity — No Partial Repository State After Failure

## Property Statement

For any failed invocation of the `create-adr` skill: after the skill exits non-zero, the repository contains either (a) all side-effects fully applied (impossible for a failed invocation) or (b) zero side-effects — with the single documented exception that the new ADR file may remain on disk when the failure occurs at the validation gate (BC-6.20.011). In all cases, the ARCH-INDEX table and any supersession patch are always reverted on failure. There is no state where only some of {ADR file, ARCH-INDEX row, supersession patch} are applied.

## Source Contract

- **BC:** BC-6.20.012 — Atomicity / no partial state (primary)
- **BC:** BC-6.20.011 — Validation gate (partial-exception: file survives validation failure)
- **Postcondition/Invariant:** "No partial state is ever left in the repository after a failed invocation (with the single documented exception of the ADR file on validation failure)."

## Proof Harness Skeleton

```bash
# Integration test: plugins/vsdd-factory/skills/create-adr/tests/test_atomicity.sh

# Setup: fixture with ADR-001..ADR-013, valid ARCH-INDEX, valid old ADR for supersession

# Scenario 1: ARCH-INDEX insert fails after file write
inject_failure arch_index_insert
invoke create-adr --title "Test" --subsystems SS-06
assert_exit_nonzero
assert_file_absent decisions/ADR-014-test.md
assert_arch_index_row_absent ADR-014

# Scenario 2: supersession patch fails after file write
inject_failure supersession_patch
invoke create-adr --title "Test" --supersedes ADR-013 --subsystems SS-06
assert_exit_nonzero
assert_file_absent decisions/ADR-014-test.md
assert_arch_index_row_absent ADR-014
assert_old_adr_superseded_by_null ADR-013

# Scenario 3: validation failure — file survives, others reverted
mock_validate_script exit_code=1
invoke create-adr --title "Test" --supersedes ADR-013 --subsystems SS-06
assert_exit_nonzero
assert_file_present decisions/ADR-014-test.md   # exception: file survives
assert_arch_index_row_absent ADR-014            # reverted
assert_old_adr_superseded_by_null ADR-013       # reverted
```

## Notes

The "file survives validation failure" exception is intentional and documented in both BC-6.20.011 and BC-6.20.012. It exists to allow the user to inspect and fix the ADR before re-running validation. The test harness must explicitly assert this asymmetry (file present, index absent) to avoid a false-positive on the full-rollback path.
