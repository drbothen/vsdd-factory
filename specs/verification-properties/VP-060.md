# VP-060: Bidirectional Supersession — supersedes ↔ superseded_by is Symmetric After Skill Completion

## Property Statement

For any successful invocation of `create-adr --supersedes ADR-OLD`: after the skill exits 0, the following symmetry holds simultaneously: (1) the new ADR's frontmatter contains `supersedes: ADR-OLD`, and (2) ADR-OLD's frontmatter contains `superseded_by: ADR-NEW`. Neither direction of the relationship is present without the other. For any failed invocation, neither direction is present in the final repository state (both reverted per BC-6.20.012).

## Source Contract

- **BC:** BC-6.20.007 — Bidirectional supersession patch
- **Postcondition:** "The old ADR's frontmatter contains `superseded_by: ADR-<new>` [...] Atomicity: either both the new file and the patch are applied, or neither is applied."
- **Architecture compliance rule in S-6.01:** "Supersession is bidirectional: new ADR sets `supersedes`, old ADR must get `superseded_by`"

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| integration | Bash + YAML frontmatter parser (grep/awk) | yes (single supersession scenario) | success path + failure path + rollback path |

## Proof Harness Skeleton

```bash
# Integration test: plugins/vsdd-factory/skills/create-adr/tests/test_supersession_symmetry.sh

# --- Happy path ---
setup_fixture  # ADR-001..ADR-013 with superseded_by: null on all

invoke create-adr --title "New Decision" --supersedes ADR-013 --subsystems SS-06
assert_exit_zero

# Forward direction: new ADR points back
new_supersedes=$(parse_frontmatter decisions/ADR-014-new-decision.md supersedes)
assert_eq "$new_supersedes" "ADR-013"

# Reverse direction: old ADR points forward
old_superseded_by=$(parse_frontmatter decisions/ADR-013-*.md superseded_by)
assert_eq "$old_superseded_by" "ADR-014"

# --- Failure path: both reverted ---
inject_failure supersession_patch  # make patch step fail

invoke create-adr --title "Another Decision" --supersedes ADR-013 --subsystems SS-06
assert_exit_nonzero

# Neither direction present
assert_file_absent decisions/ADR-014-another-decision.md
old_superseded_by_after=$(parse_frontmatter decisions/ADR-013-*.md superseded_by)
assert_eq "$old_superseded_by_after" "null"  # restored to original

echo "VP-060: bidirectional symmetry verified"
```
