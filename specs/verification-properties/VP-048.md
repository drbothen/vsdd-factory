# VP-048: protect-secrets.sh Fails Closed When jq Is Missing

## Property Statement

If jq is unavailable, protect-secrets.sh exits 2 (block) rather than silently allowing the tool call. The script does not fall back to a permissive behavior when its dependency is missing. This fail-closed design prevents accidental secret exposure on misconfigured hosts.

## Source Contract

- **BC:** BC-7.03.053 — protect-secrets: identity & dual registry binding (exit codes 0/2; on_error=block)
- **Postcondition/Invariant:** BC-7.02.002 — exit code semantics: 0=pass/allow, 2=block (jq-missing must use exit 2)

```bash
# Test: run protect-secrets.sh with jq removed from PATH
PATH_WITHOUT_JQ=$(echo "$PATH" | tr ':' '\n' | grep -v jq | tr '\n' ':')
echo '{"event_name":"PreToolUse","tool_name":"Bash"}' | \
    PATH="$PATH_WITHOUT_JQ" bash plugins/vsdd-factory/hooks/protect-secrets.sh
```
