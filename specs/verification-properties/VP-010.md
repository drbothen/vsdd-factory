# VP-010: Plugin Stderr Capped at 4 KiB with Truncation Marker

## Property Statement

Plugin stderr is captured and truncated at STDERR_CAP_BYTES (4096 bytes). If a plugin writes more than 4096 bytes to stderr, the captured string ends with the literal marker "(stderr truncated)". Empty stderr is omitted from lifecycle events entirely.

## Source Contract

- **BC:** BC-1.03.005 — stderr captured per plugin and truncated at 4 KiB with marker
- **Postcondition/Invariant:** DI-010 — Plugin stderr is capped at 4 KiB with truncation marker

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in invoke.rs | yes — 4096-byte boundary | truncation marker + empty omission |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_010_stderr_capped_at_4kib() {
    let large_stderr = vec![b'x'; 8192];
    let result = truncate_stderr(&large_stderr, STDERR_CAP_BYTES);

    assert!(result.len() <= STDERR_CAP_BYTES + "(stderr truncated)".len());
    assert!(result.ends_with("(stderr truncated)"));
}

#[test]
fn verify_vp_010_empty_stderr_omitted() {
    let event = make_plugin_event(b"");
    assert!(event.stderr.is_none());
}
```

## Notes

STDERR_CAP_BYTES = 4096 is a hardcoded constant, not configurable per registry entry (DI-010). The truncation marker "(stderr truncated)" is a literal string appended after slicing. A kani proof could formally verify the output size bound: forall stderr_len >= 0, captured_len <= 4096 + marker_len.
