# VP-032: RoutingFilter Default Accepts All Events; Allow-List Is Whitelist; Deny Applied After Allow

## Property Statement

An empty RoutingFilter (no allow, no deny) passes every event. A non-empty allow list acts as a whitelist: only listed event types pass. Deny rules are applied after allow: an event that passes the allow check may still be excluded by a deny rule. Allow comparison is case-sensitive.

- **BC:** BC-3.06.001 — routing_filter_default_accepts_everything

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_032_routing_filter_semantics() {
    // Empty filter: all pass
    let f = RoutingFilter::default();
    assert!(f.accepts("any.event"));

    // Allow whitelist: only listed pass
    let f = RoutingFilter::with_allow(vec!["commit.made".into()]);
    assert!(f.accepts("commit.made"));
    assert!(!f.accepts("plugin.invoked"));

    // Case-sensitive: "Commit.Made" != "commit.made"
    assert!(!f.accepts("Commit.Made"));
}
```
