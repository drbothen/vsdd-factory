# VP-039: SDK Wire Format Encoding Is Symmetric with Dispatcher Decoding

## Property Statement

SDK-side encode_fields produces a length-prefix buffer that dispatcher-side decode_fields can decode without error. SDK-side encode_args produces a buffer that dispatcher-side decode_args decodes correctly. SubprocessResult envelope decoding is paranoid — rejects truncated input rather than panicking.

## Source Contract

- **BC:** BC-2.02.007 — encode_fields_uses_length_prefix: key_len|key|value_len|value LE-prefixed
- **Postcondition/Invariant:** Cross-crate wire format is symmetric

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in hook-sdk/host.rs | yes — single and multi-pair buffers | encode + cross-decode symmetry |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_039_encode_decode_symmetry() {
    let fields = vec![("key1", "value1"), ("key2", "value2")];

    // SDK encodes
    let buf = encode_fields(&fields);

    // Dispatcher decodes
    let decoded = decode_fields(&buf).unwrap();

    assert_eq!(decoded.len(), 2);
    assert_eq!(decoded[0], ("key1".into(), "value1".into()));
    assert_eq!(decoded[1], ("key2".into(), "value2".into()));
}
```
