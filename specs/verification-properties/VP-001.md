# VP-001: Tier Execution Is Sequential; Intra-Tier Is Parallel

## Property Statement

For any set of plugins grouped into tiers by priority: (1) all plugins in tier N complete (with any result) before any plugin in tier N+1 begins execution, and (2) plugins within the same tier execute concurrently as observed by elapsed wall-clock time less than the sum of individual plugin durations.

## Source Contract

- **BC:** BC-1.03.007 — Tier execution preserves between-tier order
- **Postcondition/Invariant:** DI-001 — Tiers execute sequentially; plugins within a tier execute in parallel

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| integration | wasmtime executor + tokio timing assertions | no | real multi-tier + concurrent execution paths |

## Proof Harness Skeleton

```rust
// Integration test: crates/factory-dispatcher/tests/executor_integration.rs
#[tokio::test(flavor = "current_thread")]
async fn verify_vp_001_tier_sequential_intra_parallel() {
    // Three plugins: early (priority=10), late-a (priority=100), late-b (priority=100)
    let summary = execute_tiers(inputs, tiers).await.unwrap();

    // Sequential invariant: early result is first
    assert_eq!(summary.per_plugin_results[0].plugin_name, "early");

    // Parallel invariant: wall time < sum of individual durations
    assert!(wall_ms < individual_sum_ms);
}
```
