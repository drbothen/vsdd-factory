# VP-044: Legacy Bash Adapter Exit Code Mapping Is Correct

## Property Statement

The legacy-bash-adapter maps bash script exit codes to HookResult: exit 0 → Continue, exit 2 → Block (reason = first stderr line or synthetic), any other exit code → Error (message includes script path + code + stderr). The plugin_config is stripped to Null before piping to bash.

## Source Contract

- **BC:** BC-4.01.003 — legacy-bash-adapter maps bash exit codes to HookResult
- **Postcondition/Invariant:** DI-003 — block_intent is aggregate; requires correct exit code mapping

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in legacy-bash-adapter/lib.rs | yes — exit codes 0, 2, 1, 99 | all exit code branches |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_044_exit_code_mapping() {
    assert_eq!(map_exit_code(0, ""), HookResult::Continue);
    assert_eq!(map_exit_code(2, "block reason"), HookResult::Block { reason: "block reason".into() });
    assert!(matches!(map_exit_code(1, "error msg"), HookResult::Error { .. }));
    assert!(matches!(map_exit_code(99, ""), HookResult::Error { .. }));
}
```
