# VP-029: File Sink Path Template Substitutes {date}, {name}, {project} Correctly

## Property Statement

The file sink path template substitutes: {date} → YYYY-MM-DD derived from event timestamp, {name} → sink's operator-assigned name, {project} → basename of project_dir. Unknown placeholders produce a hard error at config load. Unbalanced braces are treated as literals.

## Source Contract

- **BC:** BC-3.02.001 — template_date_only: {date} substitutes to YYYY-MM-DD
- **Postcondition/Invariant:** Path templates are deterministic and validated at config load

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in sink-file/lib.rs | yes — 6 template variants | all 3 placeholders + edge cases |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_029_template_substitution() {
    assert_eq!(resolve("{date}", ts("2026-04-25"), "audit", "/proj"), "2026-04-25");
    assert_eq!(resolve("{name}", ts_any, "audit", "/proj"), "audit");
    assert_eq!(resolve("{project}", ts_any, "audit", "/my/proj"), "proj");
    assert_eq!(resolve("{project}/{name}/{date}", ts("2026-04-25"), "audit", "/proj"),
               "proj/audit/2026-04-25");

    // None project → empty basename
    assert_eq!(resolve("{project}", ts_any, "audit", None), "");

    // Unbalanced brace is literal
    assert_eq!(resolve("foo{bar", ts_any, "audit", "/p"), "foo{bar");
}
```
