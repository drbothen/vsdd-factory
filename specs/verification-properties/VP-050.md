# VP-050: exec_subprocess Timeout Is Enforced — Hung Commands Are Killed

## Property Statement

A subprocess executing for longer than timeout_ms is killed (via child.kill()) and exec_subprocess returns Err(TIMEOUT). The polling loop respects the deadline; no hung command can stall the dispatcher indefinitely.

## Source Contract

- **BC:** BC-1.05.032 — timeout_enforced: command exceeding timeout_ms is killed and returns TIMEOUT
- **Postcondition/Invariant:** DI-002 — plugin timeout does not block siblings

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit test with sleep command | yes — sleep 5 with timeout_ms=200 | kill + TIMEOUT return |

## Proof Harness Skeleton

#[test]
fn verify_vp_050_subprocess_timeout_enforced() {
    let caps = exec_caps_with_sleep_allowed(200);

## Notes

The exec_subprocess timeout (timeout_ms per invocation) is distinct from the plugin epoch timeout (DI-001). A plugin can invoke exec_subprocess with its own timeout budget within its overall epoch deadline. Both timeouts must be enforced independently to satisfy DI-002.
