# VP-071: `validate-per-story-adversary-convergence` Block Invariant

## Property Statement

The `validate-per-story-adversary-convergence` WASM hook MUST return
`HookResult::block_with_fix(hook, reason, recommendation, code)` (canonical
Why/Fix/Code form per HOST_ABI.md §WASM hooks, lines 421–460) whenever ANY of
the following conditions holds for any story in the wave under review:

1. **Missing state file:** The convergence state file at
   `.factory/cycles/<cycle-id>/<story-id>/adversary-convergence-state.json`
   does not exist (read returns `None`).
2. **Insufficient passes:** The state file exists and parses correctly, but
   `passes_clean < 3`.
3. **Non-nitpick classification:** The state file exists and parses correctly,
   but `last_classification != "NITPICK_ONLY"` (including `null` as a
   non-nitpick value).

When blocking, the hook MUST supply:
- `hook` = `"validate-per-story-adversary-convergence"`
- `reason` = a deterministic string identifying which story and which condition
  triggered the block (e.g., `"story S-12.02 has passes_clean=1 < 3"`)
- `recommendation` = an actionable string (e.g., `"Run per-story adversary loop
  to convergence (3 NITPICK_ONLY passes) before dispatching wave-gate"`)
- `code` = `per_story_adversary_unconverged` (stable snake_case telemetry code)

Conversely (the happy-path invariant), the hook MUST return `HookResult::Continue`
(not block) when ALL stories satisfy BOTH of:
- `passes_clean >= 3`
- `last_classification == "NITPICK_ONLY"`

**Anti-pattern (explicitly forbidden):** Do NOT use `HookResult::Continue` +
stdout JSON outcome line. That is the deprecated advisory-block-mode pattern
(HOST_ABI.md §Advisory block-mode pattern, lines 146–189), used only by the
three named v1.0 plugins (`handoff-validator`, `pr-manager-completion-guard`,
`validate-pr-review-posted`). New hooks adopt the canonical `block_with_fix`
form per the slices 1+2 + Tier A migrations (commits c94f8bf, 4cf59bc).

## Formal Property Statement (kani harness)

```rust
#[cfg(kani)]
mod kani_proofs {
    use super::*;

    /// Block invariant — Part A: missing state file always returns Block.
    /// Regardless of how many stories are in the wave, a missing file
    /// for ANY story causes a canonical block (not advisory-mode Continue).
    #[kani::proof]
    fn proof_missing_state_file_always_blocks() {
        // Single-story case: state file absent
        let state: Option<ConvergenceState> = None;
        let result = hook_result_for(state);
        kani::assert(
            matches!(result, HookResult::Block { .. }),
            "missing state file must return HookResult::Block (not Continue)"
        );
    }

    /// Block invariant — Part B: passes_clean < 3 always returns Block.
    #[kani::proof]
    #[kani::unwind(4)]
    fn proof_insufficient_passes_always_blocks() {
        let passes: u32 = kani::any();
        kani::assume(passes < 3);

        let state = Some(ConvergenceState {
            passes_clean: passes,
            last_classification: Some("NITPICK_ONLY".to_string()),
            last_finding_count: Some(0),
            last_timestamp: None,
            deferred_findings: vec![],
        });
        let result = hook_result_for(state);
        kani::assert(
            matches!(result, HookResult::Block { .. }),
            "passes_clean < 3 must return HookResult::Block"
        );
    }

    /// Block invariant — Part C: non-NITPICK_ONLY classification always returns
    /// Block even when passes_clean >= 3.
    #[kani::proof]
    fn proof_non_nitpick_classification_always_blocks() {
        let state = Some(ConvergenceState {
            passes_clean: 3,
            last_classification: Some("HIGH".to_string()),
            last_finding_count: Some(2),
            last_timestamp: None,
            deferred_findings: vec![],
        });
        let result = hook_result_for(state);
        kani::assert(
            matches!(result, HookResult::Block { .. }),
            "non-NITPICK_ONLY classification must return HookResult::Block"
        );
    }

    /// Block invariant — Part D: null/None classification always returns Block.
    #[kani::proof]
    fn proof_null_classification_blocks() {
        let state = Some(ConvergenceState {
            passes_clean: 5,
            last_classification: None,  // null in JSON
            last_finding_count: None,
            last_timestamp: None,
            deferred_findings: vec![],
        });
        let result = hook_result_for(state);
        kani::assert(
            matches!(result, HookResult::Block { .. }),
            "null/None last_classification must return HookResult::Block"
        );
    }

    /// Block invariant — Part E: Block reason carries the required canonical segments.
    /// The `block_with_fix` constructor formats hook, reason, recommendation, and code
    /// into a single `reason` string (BLOCKED by <hook>: <reason>. Fix: <fix>. Code: <code>.).
    #[kani::proof]
    fn proof_block_reason_contains_canonical_segments() {
        let state: Option<ConvergenceState> = None;  // guaranteed block path
        let result = hook_result_for(state);
        if let HookResult::Block { reason } = result {
            kani::assert(!reason.is_empty(), "reason field must not be empty");
            kani::assert(
                reason.contains(HOOK_NAME),
                "reason must reference the hook name (HOOK_NAME)"
            );
            kani::assert(
                reason.contains("Fix:"),
                "reason must contain 'Fix:' segment (canonical block_with_fix format)"
            );
            kani::assert(
                reason.contains("Code:"),
                "reason must contain 'Code:' segment (canonical block_with_fix format)"
            );
            kani::assert(
                reason.contains(HOOK_CODE_BASE),
                "reason must contain the HOOK_CODE_BASE prefix in the Code segment"
            );
        }
    }

    /// Happy path: passes_clean >= 3 AND last_classification == NITPICK_ONLY
    /// must return HookResult::Continue (not block).
    #[kani::proof]
    #[kani::unwind(4)]
    fn proof_converged_story_produces_continue() {
        let passes: u32 = kani::any();
        kani::assume(passes >= 3);

        let state = Some(ConvergenceState {
            passes_clean: passes,
            last_classification: Some("NITPICK_ONLY".to_string()),
            last_finding_count: Some(0),
            last_timestamp: None,
            deferred_findings: vec![],
        });
        let result = hook_result_for(state);
        kani::assert(
            matches!(result, HookResult::Continue),
            "converged story (passes >= 3, NITPICK_ONLY) must return HookResult::Continue"
        );
    }
}
```

path. Malformed state JSON is handled by the injectable-callback layer (not in
the pure evaluation function). The function is a pure projection from
`Option<ConvergenceState>` to `HookResult` — no I/O, no global state.

## Amendment Note (v1.0 → v1.1, OQ-9, 2026-05-06)

**Contradiction resolved:** VP-071 v1.0 specified the advisory-block-mode pattern
(`HookResult::Continue` + stdout JSON outcome line), which directly contradicted
BC-4.10.001's requirement that the hook return `HookResult::block_with_fix(...)`.

**Resolution:** BC-4.10.001 is authoritative. For a new hook, the canonical
`block_with_fix` form is correct per HOST_ABI.md §WASM hooks (lines 421–460)
and the active slices 1+2 + Tier A canonical-format migration (commits c94f8bf,
4cf59bc). The advisory-block-mode pattern (HOST_ABI.md §Advisory block-mode
pattern, lines 146–189) is reserved for the three named v1.0 plugins; it is
explicitly deprecated for new hooks.

**Harness changes:** The `proof_hook_result_is_always_continue` harness (which
asserted `HookResult::Continue` on the block path) is replaced by:
- `proof_missing_state_file_always_blocks` — now asserts `HookResult::BlockWithFix`
- `proof_insufficient_passes_always_blocks` — now asserts `HookResult::BlockWithFix`
- `proof_non_nitpick_classification_always_blocks` — now asserts `HookResult::BlockWithFix`
- `proof_null_classification_blocks` — now asserts `HookResult::BlockWithFix`
- `proof_block_with_fix_fields_populated` (new) — verifies all four fields
  (hook, reason, recommendation, code) are non-empty on a block path
- `proof_converged_story_produces_continue` — unchanged; still asserts `HookResult::Continue`

## Amendment Note (v1.1 → v1.2, F5 pass-1 fix burst, 2026-05-07)

**Contradiction resolved:** VP-071 v1.1 incorrectly declared `HookResult::BlockWithFix { hook, reason, recommendation, code }` as an enum variant used in the kani proof harnesses. The actual SDK (`crates/hook-sdk/src/result.rs`) defines only three variants: `Continue`, `Block { reason: String }`, and `Error { message: String }`. The `block_with_fix(hook, reason, recommendation, code)` function is a *constructor* that produces `Block { reason: "BLOCKED by <hook>: <reason>. Fix: <recommendation>. Code: <code>." }`. There is no `BlockWithFix` variant in the SDK.

The implementation's kani harnesses (lib.rs) had already silently corrected this to `HookResult::Block { .. }` — the code was right, the spec was wrong.

**Resolution:** The spec is corrected to match the SDK. All six kani proof assertions now use `matches!(result, HookResult::Block { .. })`. Part E is renamed `proof_block_reason_contains_canonical_segments` and checks that the `reason` string contains `HOOK_NAME`, `"Fix:"`, `"Code:"`, and `HOOK_CODE_BASE` — verifying the canonical block_with_fix format is preserved in the reason string.

**Frontmatter corrected:** `bcs:` field updated from placeholder `BC-5.NN.002` to actual IDs `BC-4.10.001` and `BC-5.39.001`.

**Harness changes (v1.1 → v1.2):**
- All four `proof_*_always_blocks` assertions changed from `HookResult::BlockWithFix { .. }` to `HookResult::Block { .. }`
- `proof_block_with_fix_fields_populated` renamed to `proof_block_reason_contains_canonical_segments` and updated to destructure `HookResult::Block { reason }` and check canonical string segments instead of four separate fields
