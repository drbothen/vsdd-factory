# VP-017: dispatcher_trace_id Present on Every Emitted Event

## Property Statement

Every InternalEvent carries the UUID v4 dispatcher_trace_id generated from the stdin envelope at dispatch start. No event path emits an event without a dispatcher_trace_id. The reserved_fields list protects dispatcher_trace_id from plugin override via emit_event.

## Source Contract

- **BC:** BC-1.05.018 — reserved_fields_rejected: every name in RESERVED_FIELDS is recognized by is_reserved_field
- **Postcondition/Invariant:** DI-017 — dispatcher_trace_id is present on every emitted event

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in emit_event.rs and internal_log integration | yes — all 8 reserved field names | reserved-field protection + trace_id propagation |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_017_trace_id_on_every_event() {
    // The 4-event integration test verifies trace_id propagation
    let events = run_startup_flow_with_trace_id("my-trace-uuid");

    for event in &events {
        assert!(event.dispatcher_trace_id.is_some(), "event missing trace_id");
        assert_eq!(event.dispatcher_trace_id.as_deref(), Some("my-trace-uuid"));
    }
}

#[test]
fn verify_vp_017_reserved_fields_cannot_be_overridden() {
    // Plugin cannot inject its own dispatcher_trace_id
    let fields = vec![("dispatcher_trace_id", "plugin-injected")];
    let result = filter_reserved_fields(fields);
    assert!(result.is_empty());
}
```
