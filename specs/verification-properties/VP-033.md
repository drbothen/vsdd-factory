# VP-033: OTLP LogRecord Mapping Is Correct — type to body, ts_epoch to time_unix_nano

## Property Statement

SinkEvent → OTLP LogRecord mapping: the type field is lifted to LogRecord.body, ts_epoch (ms) is converted to time_unix_nano (ns = ts_epoch * 1_000_000), reserved fields are lifted to body/time, and non-reserved fields become attributes. type and ts_epoch do NOT appear as duplicate attributes.

## Source Contract

- **BC:** BC-3.03.008 — event_to_log_record_maps_reserved_fields
- **Postcondition/Invariant:** DI-017 — trace correlation fields arrive at the OTLP endpoint

## Proof Harness Skeleton

```rust
// Integration test: crates/factory-dispatcher/tests/sinks_otel_grpc.rs
#[tokio::test]
async fn verify_vp_033_otlp_mapping() {
    let (server, port) = mock_logs_service();
    let sink = OtelGrpcSink::new(format!("http://127.0.0.1:{}", port));

    sink.submit(make_event_with_ts_epoch(1_777_003_425_000));
    sink.flush();

    let record = server.last_record();
    assert_eq!(record.body, "plugin.invoked");
    assert_eq!(record.time_unix_nano, 1_777_003_425_000 * 1_000_000);
    assert!(record.attributes.contains_key("dispatcher_trace_id"));
    assert!(!record.attributes.contains_key("type"));
    assert!(!record.attributes.contains_key("ts_epoch"));
}
```

## Notes

The ms → ns conversion (ts_epoch * 1_000_000) must not overflow. For ts_epoch values up to u64::MAX / 1_000_000, the multiplication is safe. A kani proof could verify overflow absence for realistic timestamp ranges (year 2000–2200).
