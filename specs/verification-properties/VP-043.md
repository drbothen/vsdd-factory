# VP-043: Every hooks-registry.toml Entry Routes Through legacy-bash-adapter.wasm

## Property Statement

Every entry in the production hooks-registry.toml has its plugin field resolving to legacy-bash-adapter.wasm. No native WASM plugin has been added to the registry without proper story-tracked engineering work. Every entry also carries a non-empty plugin_config.script_path pointing to a hooks/*.sh file.

## Source Contract

- **BC:** BC-1.07.005 — every_entry_routes_through_legacy_bash_adapter
- **Postcondition/Invariant:** DI-016 — each entry sees only its own plugin_config (script_path)

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| integration | Rust integration test loading production registry | yes — all 45 entries | plugin file + script_path per entry |

## Proof Harness Skeleton

```rust
// Integration test: crates/factory-dispatcher/tests/loads_legacy_registry.rs
#[test]
fn verify_vp_043_all_entries_use_adapter() {
    let registry = Registry::load("plugins/vsdd-factory/hooks-registry.toml").unwrap();

    for entry in &registry.hooks {
        let plugin_basename = Path::new(&entry.plugin).file_name().unwrap();
        assert_eq!(plugin_basename, "legacy-bash-adapter.wasm");

        let script_path = entry.config_as_json()
            .get("script_path")
            .and_then(|v| v.as_str())
            .unwrap_or("");
        assert!(!script_path.is_empty());
    }
}
```
