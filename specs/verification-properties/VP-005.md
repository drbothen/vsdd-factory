# VP-005: Shell Interpreters Require Explicit shell_bypass_acknowledged

## Property Statement

The set {bash, sh, zsh, pwsh, fish, csh, tcsh, ksh} and their path variants (e.g., /bin/bash) is refused by exec_subprocess unless the capability entry provides a non-empty shell_bypass_acknowledged string. No default capability configuration bypasses this check.

## Source Contract

- **BC:** BC-1.05.003 — exec_subprocess denies shell interpreters without shell_bypass_acknowledged
- **Postcondition/Invariant:** DI-005 — Shell interpreters require explicit shell_bypass_acknowledged

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| unit-test | Rust unit tests in exec_subprocess.rs | yes — enumerated shell names | is_shell detection + denial gate |

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_005_shell_requires_acknowledgment() {
    let shell_names = ["bash", "/bin/bash", "sh", "zsh", "pwsh", "fish"];

    for shell in &shell_names {
        // Without acknowledgment: must deny
        let caps = exec_caps_without_shell_bypass(shell);
        let result = check_shell_gate(shell, &caps);
        assert_eq!(result, Err(CAPABILITY_DENIED), "shell {} not denied", shell);

        // With acknowledgment: gate passes
        let caps = exec_caps_with_shell_bypass(shell, "acknowledged");
        let result = check_shell_gate(shell, &caps);
        assert!(result.is_ok(), "shell {} denied with acknowledgment", shell);
    }
}
```
