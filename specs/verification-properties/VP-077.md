# VP-077: Dispatcher Partition Correctness

## Property Statement

For any set of matched plugins M and the partition function
`P(M, registry) → (sync_group, async_group)`:

1. **Totality:** every plugin in M lands in exactly one group — no plugin is
   dropped or duplicated.
2. **Async-field respect (post-parse scope):** applied to `RegistryEntry` struct
   values as they exist after serde deserialization — `plugin.async_flag == true`
   implies `plugin ∈ async_group`; `plugin.async_flag == false` implies
   `plugin ∈ sync_group`. Serde-default semantics (absent `async` field → `false`)
   are **out of scope for this VP** and verified separately by VP-078 Harness 4
   (Rust unit-test block covering the three cases: explicit `true`, explicit `false`,
   and field absent). This VP operates on typed `bool` values only; Kani's `bool`
   is binary and cannot represent field absence.
3. **Disjointness:** `sync_group ∩ async_group = ∅`.
4. **Union completeness:** `sync_group ∪ async_group = M`.
5. **Exit-code independence:** `dispatcher_exit` is a function only of
   `sync_group` results. Formally: for any two executions that differ only in
   the async_group's exit codes, `dispatcher_exit` is identical.
6. **Aggregation correctness:** `dispatcher_exit = 2` if and only if at least
   one plugin in `sync_group` returned `block_intent = true`; otherwise
   `dispatcher_exit = 0`.

These six properties hold for all possible input sets M (including empty M,
all-sync M, all-async M, and mixed M) and for all possible plugin results
including crash, timeout, and block. **Precondition:** (`name`, `event`, `tool`) tuples in M are
unique (enforced at registry-load time by BC-7.06.001 v1.5 Invariant 7 via
`registry.rs::validate()`; asserted in Harness 1 via `kani::assume`). Two
`protect-secrets` entries (PreToolUse + Bash; PreToolUse + Read) are valid because
they differ in `tool`; the uniqueness key is the full triple, not the pair.

### Why Kani

The partition function `P` will be a pure, deterministic function operating on
typed structs — no I/O, no database, no network, no global state. This is exactly
the category Kani verifies by exhaustive state-space exploration within the
bounded input domain. Properties 1–4 (set-theoretic) and 6 (aggregation
arithmetic) are straightforward to express as `kani::assert!` postconditions.
Property 5 (exit-code independence) is expressed as a universal quantification
over async_group result variants.

## Proof Harness Skeleton

```rust
// crates/factory-dispatcher/src/partition.rs (or a sibling proof module)
// #[cfg(kani)]
mod kani_proofs {
    use super::*;
    use kani;

    /// H1: proof_vp077_totality
    /// Properties: 1 (Totality) — cardinality invariant only.
    /// sync_group.len() + async_group.len() == input set size.
    #[kani::proof]
    fn proof_vp077_totality() {
        let n: usize = kani::any();
        kani::assume(n <= 4);

        let plugins: Vec<RegistryEntry> = (0..n)
            .map(|_| RegistryEntry { async_flag: kani::any(), on_error: kani::any(), .. })
            .collect();

        // Precondition: (name, event, tool) tuple uniqueness — per BC-7.06.001 v1.5 Invariant 7.
        // Real registry enforces this at load time via registry.rs::validate(). Assert it here
        // so Kani bounds its search to unique-(name,event,tool) worlds and the contains()-based
        // disjointness check in H2 is well-defined.
        // Two entries with the same (name, event) but different `tool` values are VALID
        // (e.g., protect-secrets: PreToolUse+Bash and PreToolUse+Read); only the full triple
        // must be unique. tool is Option<String>: None vs Some("Bash") are distinct values.
        for i in 0..plugins.len() {
            for j in (i + 1)..plugins.len() {
                kani::assume(
                    plugins[i].name != plugins[j].name
                        || plugins[i].event != plugins[j].event
                        || plugins[i].tool != plugins[j].tool
                );
            }
        }

        let part = partition_plugins(&plugins);
        let sync_g = &part.sync_group;
        let async_g = &part.async_group;

        let total = sync_g.len() + async_g.len();
        kani::assert!(total == plugins.len(), "partition must be total");
    }

    /// H2: proof_vp077_disjointness
    /// Properties: 3 (Disjointness) — no plugin appears in both groups.
    /// Uses contains() check (NOT just totality) to verify mutual exclusion.
    /// Separated from H1 so the disjointness invariant cannot be vacuously
    /// satisfied by a partition that duplicates every plugin into both groups.
    #[kani::proof]
    fn proof_vp077_disjointness() {
        let n: usize = kani::any();
        kani::assume(n <= 4);

        let plugins: Vec<RegistryEntry> = (0..n)
            .map(|_| RegistryEntry { async_flag: kani::any(), on_error: kani::any(), .. })
            .collect();

        // Precondition mirrors H1: (name, event, tool) uniqueness per BC-7.06.001 v1.5 Invariant 7.
        for i in 0..plugins.len() {
            for j in (i + 1)..plugins.len() {
                kani::assume(
                    plugins[i].name != plugins[j].name
                        || plugins[i].event != plugins[j].event
                        || plugins[i].tool != plugins[j].tool
                );
            }
        }

        let part = partition_plugins(&plugins);
        let sync_g = &part.sync_group;
        let async_g = &part.async_group;

        for p in sync_g {
            kani::assert!(!async_g.contains(p), "sync and async groups must be disjoint");
        }
        for p in async_g {
            kani::assert!(!sync_g.contains(p), "async and sync groups must be disjoint");
        }
    }

    /// H3: proof_vp077_async_field_respected
    /// Properties: 2 (Async-field respect) — async_flag == true ⇒ in async_group;
    /// async_flag == false ⇒ in sync_group.
    /// Operates on post-parse typed bool values only; serde-default (absent field → false)
    /// is out of scope and covered by VP-078 Harness 4.
    #[kani::proof]
    fn proof_vp077_async_field_respected() {
        let p: RegistryEntry = kani::any();

        let part = partition_plugins(&[p.clone()]);
        let sync_g = &part.sync_group;
        let async_g = &part.async_group;

        if p.async_flag {
            kani::assert!(async_g.contains(&p), "async=true must land in async_group");
            kani::assert!(!sync_g.contains(&p), "async=true must not be in sync_group");
        } else {
            kani::assert!(sync_g.contains(&p), "async=false must land in sync_group");
            kani::assert!(!async_g.contains(&p), "async=false must not be in async_group");
        }
    }

    /// H4: proof_vp077_union_completeness
    /// Properties: 4 (Union completeness) — every input plugin appears in exactly
    /// one group. This is distinct from H1 (cardinality) and H2 (mutual exclusion):
    /// H4 asserts the positive coverage invariant that no plugin is silently dropped.
    #[kani::proof]
    fn proof_vp077_union_completeness() {
        let n: usize = kani::any();
        kani::assume(n <= 4);

        let plugins: Vec<RegistryEntry> = (0..n)
            .map(|_| RegistryEntry { async_flag: kani::any(), on_error: kani::any(), .. })
            .collect();

        // Precondition mirrors H1: (name, event, tool) uniqueness per BC-7.06.001 v1.5 Invariant 7.
        for i in 0..plugins.len() {
            for j in (i + 1)..plugins.len() {
                kani::assume(
                    plugins[i].name != plugins[j].name
                        || plugins[i].event != plugins[j].event
                        || plugins[i].tool != plugins[j].tool
                );
            }
        }

        let part = partition_plugins(&plugins);
        let sync_g = &part.sync_group;
        let async_g = &part.async_group;

        for p in &plugins {
            kani::assert!(
                sync_g.contains(p) || async_g.contains(p),
                "every matched plugin must land in exactly one group"
            );
        }
    }

    /// H5: proof_vp077_exit_code_independent_of_async_group
    /// Properties: 5 (Exit-code independence) — dispatcher exit code is computed
    /// from sync_group results only. aggregate_exit_code takes &[PluginResult] and
    /// returns u8; it cannot receive async group results — they are excluded by the
    /// function signature.
    ///
    /// Note on enforcement: independence from async inputs is structural, not behavioural.
    /// The function signature `aggregate_exit_code(&[PluginResult])` accepts only sync
    /// results; async results cannot be passed in. Kani proves determinism on sync inputs
    /// (same inputs → same output); independence from async inputs is enforced at the
    /// type level by the signature, not by this proof. The proof is named
    /// `_of_async_group` to match the actual implementation.
    #[kani::proof]
    fn proof_vp077_exit_code_independent_of_async_group() {
        let sync_results: Vec<PluginResult> = vec![kani::any()];
        // Two different async result variants — the exit code must be the same
        // regardless of which is "in flight"
        let _async_result_a: PluginResult = kani::any();
        let _async_result_b: PluginResult = kani::any();

        // aggregate_exit_code takes only sync_results — async results are not
        // even accepted as an argument. Independence is structural.
        let exit_a = aggregate_exit_code(&sync_results);
        let exit_b = aggregate_exit_code(&sync_results);

        kani::assert!(exit_a == exit_b,
            "dispatcher_exit must not depend on async group results");
    }

    /// H6: proof_vp077_aggregation_correctness
    /// Properties: 6 (Aggregation correctness) — aggregate_exit_code returns 2
    /// if any sync result has exit_code=2 with on_error=Block; else 0.
    /// No async results are considered (BC-1.14.001 Invariant 3).
    ///
    /// Anchors: aggregate_exit_code in crates/factory-dispatcher/src/aggregator.rs
    /// (NEW module — to be created in implementer phase; see §Required Implementation
    /// Artifacts and §Appendix A below). ADR-019 + DI-019 cited.
    #[kani::proof]
    fn proof_vp077_aggregation_correctness() {
        let n: usize = kani::any();
        kani::assume(n <= 4);

        let sync_results: Vec<PluginResult> = (0..n).map(|_| kani::any()).collect();

        // Mirroring the spec: block condition is exit_code=2 AND on_error=Block
        let any_blocking = sync_results.iter().any(|r| {
            r.exit_code == 2 && r.on_error == OnError::Block
        });

        let exit_code = aggregate_exit_code(&sync_results);

        if any_blocking {
            kani::assert!(exit_code == 2, "any sync block (exit=2, on_error=Block) => dispatcher exit 2");
        } else {
            kani::assert!(exit_code == 0, "no sync block => dispatcher exit 0");
        }
    }
}
```

## Counter-Proof Scenarios (Fault Injection)

The harnesses above cover the following fault scenarios via `kani::any()`:

| Scenario | Expected invariant |
|----------|-------------------|
| One sync plugin crashes (PluginResult::Crashed) | Sibling plugins continue (VP-002 scope); exit code = 0 unless another sync plugin blocks |
| Two sync plugins return conflicting block_intent | Both block intents are honoured; aggregate exit = 2 (VP-003 scope) |
| All sync plugins timeout | exit = 0 (timeout is not block); async group unaffected |
| Async group plugin returns exit 2 | dispatcher_exit unchanged — async group is fire-and-forget |
| Empty matched set M | sync_g = [], async_g = []; dispatcher_exit = 0 |
| All plugins async | sync_g = []; dispatcher_exit = 0 regardless of async_group results |
| All plugins sync | async_g = []; standard tier execution applies |

## Appendix A — aggregate_exit_code Design Contract

This function must be authored in `crates/factory-dispatcher/src/aggregator.rs` (new file)
during the implementer phase. It is the implementation target for Harnesses H5 and H6.

```rust
use crate::registry::OnError;

/// A single plugin result as produced by the sync dispatch loop.
/// (Mirrors the existing PluginResult shape; exact field names subject to
/// implementer review against the live codebase.)
pub struct PluginResult {
    pub exit_code: u8,
    pub on_error: OnError,
}

/// Aggregate sync_group results into the dispatcher exit code
/// per BC-1.14.001 PC5 and ADR-019 async-semantics invariant.
///
/// Returns 2 if any sync_group plugin returned (exit_code == 2, on_error == Block).
/// Returns 0 otherwise.
///
/// async_group results are NOT accepted as a parameter and are NOT considered.
/// This structural exclusion is the mechanistic proof of BC-1.14.001 Invariant 3
/// ("Async group plugins are excluded from the exit-code aggregation").
///
/// PURITY CONTRACT: no I/O, no globals, no async, no logging.
/// This function must remain pure to be Kani-verifiable by H5 and H6.
///
/// Anchors: ADR-019 (async semantics at registry layer), DI-019 (drain window
/// constant — cited as invariant source for async-group exclusion semantics).
pub fn aggregate_exit_code(sync_results: &[PluginResult]) -> u8 {
    sync_results
        .iter()
        .any(|r| r.exit_code == 2 && r.on_error == OnError::Block)
        .then_some(2)
        .unwrap_or(0)
}
```

**Placement:** `crates/factory-dispatcher/src/aggregator.rs` (new file). Must be
`pub use`-exported from `crates/factory-dispatcher/src/lib.rs`.

**Purity invariant:** No I/O, no globals, no async executor calls, no logging side
effects. The function takes a slice and returns a `u8`. Any future modification that
introduces side effects must remove the Kani proof harnesses H5/H6 or restructure to
maintain a pure inner function.
