# VP-069: `validate-artifact-path` Registry-Load Purity — Parsing Never Panics on Arbitrary Input

## Property Statement

For all possible byte sequences supplied as the content of
`plugins/vsdd-factory/config/artifact-path-registry.yaml`, the registry-parsing
function in `validate-artifact-path/src/lib.rs` MUST:

1. Return a well-typed result (either a parsed registry or an explicit parse
   error) without panicking.
2. On malformed YAML input, return a graceful error value that causes the hook
   to emit a log-level error and return `HookResult::Continue` — never
   `HookResult::Block`.
3. On absent registry file (read returns `None`), return a graceful absent
   sentinel that causes the hook to emit a log-level warning and return
   `HookResult::Continue`.
4. Never propagate an `unwrap()` panic or an unhandled `expect()` on any
   registry parse path.

The registry-load function is pure in the sense that it takes a byte slice (or
`Option<String>`) as input and returns a typed result with no side effects. All
side effects (logging, event emission) are injected via the callback pattern
established in `regression-gate/src/lib.rs` and `handoff-validator/src/lib.rs`.

## Formal Property Statement (proptest)

```rust
use proptest::prelude::*;

proptest! {
    /// For any arbitrary byte sequence, parse_registry never panics.
    /// The function must return Ok(registry) or Err(parse_error) — never panic.
    #[test]
    fn prop_registry_parse_never_panics(input in any::<Vec<u8>>()) {
        // Convert arbitrary bytes to string (lossy — mimics real-world malformed YAML)
        let s = String::from_utf8_lossy(&input).into_owned();
        // parse_registry must not panic on any input; result may be Ok or Err
        let _ = parse_registry(Some(&s));
    }

    /// For any arbitrary string, parse_registry returns Continue (not Block)
    /// when input is malformed.
    #[test]
    fn prop_malformed_registry_produces_continue(input in ".*") {
        // Strings that are not valid YAML must not produce a block outcome
        // (non-parseable registry should be treated as non-fatal per graceful-degradation rule)
        let result = registry_load_outcome(Some(&input));
        assert_ne!(result, LoadOutcome::Block,
            "malformed registry MUST NOT produce a block outcome");
    }

    /// Empty registry (zero artifact entries) produces Continue for any path.
    #[test]
    fn prop_empty_registry_continues_for_all_paths(path in ".*") {
        let result = validate_path_against_registry(&path, &RegistryFixture::empty());
        assert_eq!(result, PathOutcome::Continue,
            "empty registry must never block any path");
    }
}
```

## Expected Outcome

All proptest trials (minimum 200 trials per property) pass without a panic.
The `parse_registry` function returns `Err(ParseError)` on invalid YAML; the
caller handles this by logging and returning `HookResult::Continue`. No
unchecked `unwrap()` or `expect()` calls appear on the parse path (enforced by
a `clippy::unwrap_in_result` lint or equivalent code review criterion).
