# VP-038: SDK HookResult Exit Codes Are Stable — Continue=0, Error=1, Block=2

## Property Statement

HookResult serialization uses a tagged outcome field. The exit codes are stable: Continue → exit code 0, Error → exit code 1, Block → exit code 2. HOST_ABI_VERSION is 1 in both the dispatcher and SDK crates. These values are not configurable.

## Source Contract

- **BC:** BC-2.01.002 — HookResult exit codes Continue=0 / Block=2 / Error=1
- **Postcondition/Invariant:** DI-004 — ABI stability enables capability enforcement

## Proof Harness Skeleton

```rust
#[test]
fn verify_vp_038_exit_codes_stable() {
    assert_eq!(HookResult::Continue.exit_code(), 0);
    assert_eq!(HookResult::Error { message: "".into() }.exit_code(), 1);
    assert_eq!(HookResult::Block { reason: "".into() }.exit_code(), 2);

    // ABI version matches dispatcher
    assert_eq!(HOST_ABI_VERSION, 1);
}
```

## Notes

Exit codes 0/1/2 are the fundamental contract between the dispatcher (as WASM host) and the SDK (as WASM guest). Any change requires bumping HOST_ABI_VERSION. The dispatcher's exit code 2 to Claude Code (bc_intent) maps to the SDK's Block(2), creating an end-to-end blocking chain.
