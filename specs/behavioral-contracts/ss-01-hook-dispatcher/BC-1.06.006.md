# Behavioral Contract BC-1.06.006: InternalEvent fields flatten to top-level JSON (no nested `fields`)

## Description

When an `InternalEvent` has extra fields, the serialized JSON has no `"fields"` wrapper key — extras live at top level. This makes downstream filtering with jq / OTel attribute promotion straightforward.

## Preconditions

1. Event has extra fields beyond the canonical envelope.

## Postconditions

1. Output JSON has no `"fields"` key.
2. Extras appear at top level.

## Invariants

1. Internal event JSON is flat.
