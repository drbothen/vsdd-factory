# Behavioral Contract BC-1.01.013: factory-dispatcher::registry::overrides_priority_per_entry — per-entry priority field overrides registry default

## Description

When two hooks declare `priority = 10` and `priority = 900` respectively, each entry's `priority(&defaults)` returns the entry-declared value, not the registry default of 500. Operators can fine-tune per-entry priority without changing global defaults.

## Preconditions

1. Two hooks declared with `priority = 10` and `priority = 900`.

## Postconditions

1. First entry's `priority(&defaults) == 10`.
2. Second entry's `priority(&defaults) == 900`.

## Invariants

1. Per-entry override always wins over registry default.
