# Behavioral Contract BC-1.03.001: Plugin in infinite loop times out via epoch interruption

## Description

A WASM plugin whose body is an infinite `(loop (br 0))` is terminated by epoch interruption when invoked with `timeout_ms = 50` (and unbounded fuel). `invoke_plugin` returns `PluginResult::Timeout { cause: TimeoutCause::Epoch }`.

## Preconditions

1. Plugin body is an infinite WASM loop.
2. `timeout_ms = 50`.
3. `fuel_cap` is unbounded.

## Postconditions

1. `invoke_plugin` returns `PluginResult::Timeout { cause: TimeoutCause::Epoch }`.

## Invariants

1. The dispatcher will always terminate an infinite loop within the configured `timeout_ms` via epoch.
