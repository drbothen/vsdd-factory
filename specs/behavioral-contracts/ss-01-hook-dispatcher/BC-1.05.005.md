# Behavioral Contract BC-1.05.005: exec_subprocess returns OUTPUT_TOO_LARGE when result exceeds buffer

## Description

When the combined exit_code + stdout + stderr envelope exceeds `result_buf_cap`, `exec_subprocess` returns `codes::OUTPUT_TOO_LARGE = -3`. Encodes the "no unbounded host fns" rule.

## Preconditions

1. Combined envelope exceeds `result_buf_cap`.

## Postconditions

1. Returns `codes::OUTPUT_TOO_LARGE = -3`.

## Invariants

1. Host fns never write past the caller-supplied buffer cap.
