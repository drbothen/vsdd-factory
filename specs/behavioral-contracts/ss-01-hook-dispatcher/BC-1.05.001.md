# Behavioral Contract BC-1.05.001: exec_subprocess denies when no exec_subprocess capability

## Description

A plugin calling `vsdd::exec_subprocess` without a corresponding `Capabilities.exec_subprocess` block in the registry entry receives `codes::CAPABILITY_DENIED = -1`. The dispatcher emits an `internal.capability_denied` event with `function = "exec_subprocess"` and `reason = "no_exec_subprocess_capability"`.

## Preconditions

1. Plugin calls `vsdd::exec_subprocess`.
2. Registry entry has no `[hooks.capabilities.exec_subprocess]` block.

## Postconditions

1. Returns `codes::CAPABILITY_DENIED = -1`.
2. Emits `internal.capability_denied` event with `function = "exec_subprocess"`, `reason = "no_exec_subprocess_capability"`.

## Invariants

1. Deny-by-default: missing capability declaration always denies.
