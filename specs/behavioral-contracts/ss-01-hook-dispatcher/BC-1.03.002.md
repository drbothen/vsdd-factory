# Behavioral Contract BC-1.03.002: Plugin in tight arithmetic loop runs out of fuel

## Description

A plugin running a tight i32-add loop with `fuel_cap = 10_000` and `timeout_ms = 60_000` exhausts fuel before timing out. `invoke_plugin` returns `PluginResult::Timeout { cause: TimeoutCause::Fuel }`.

## Preconditions

1. Plugin body is a tight i32 arithmetic loop.
2. `fuel_cap = 10_000`.
3. `timeout_ms = 60_000`.

## Postconditions

1. `invoke_plugin` returns `PluginResult::Timeout { cause: TimeoutCause::Fuel }`.

## Invariants

1. Fuel exhaustion always produces `TimeoutCause::Fuel`, never `Epoch`, for tight CPU-bound plugins.
