# Behavioral Contract BC-10.02.002: emit-event: I/O — argv key=value pairs become JSON top-level fields; stdin ignored; no stdout/s

> Section: emit-event CLI
> Source BC (audit ID): BC-AUDIT-2106

## Description

Args are `key=value` pairs split on the FIRST `=`; values may themselves contain `=`. Keys are sanitized to `[A-Za-z0-9_.]`. Synthetic jq variable names (v0, v1, ...) prevent caller-key/jq-syntax collision. Filter assembled with `["key"]=$varN` form so dotted keys stay flat. No stdout/stderr written on success; bad args silently skipped.
