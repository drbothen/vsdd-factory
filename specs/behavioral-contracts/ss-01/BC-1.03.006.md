# Behavioral Contract BC-1.03.006: Empty stderr is omitted from lifecycle events (noise reduction)

## Description

When a well-behaved plugin emits no stderr, the lifecycle event JSON omits the `stderr` field entirely (rather than serializing an empty string). `executor.rs::emit_lifecycle` applies a retain-non-empty filter for noise reduction.

## Preconditions

1. Plugin produces empty stderr.

## Postconditions

1. The lifecycle event JSON has no `stderr` key (key omitted, not empty string).

## Invariants

1. Lifecycle events do not carry empty optional fields.
