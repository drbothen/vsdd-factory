# Behavioral Contract BC-1.01.015: factory-dispatcher::routing::group_returns_empty_for_no_matches — no-match payload yields empty tier list

## Description

When no registry entry matches the payload's event (e.g., `event_name = "SessionStart"` for which no hooks are registered), `match_plugins` then `group_by_priority` produces an empty `Vec<Vec<...>>`. Caller can safely treat the empty case as no-op.

## Preconditions

1. The registry has no hooks for the given event.
2. A payload requests that event.

## Postconditions

1. `tiers.is_empty() == true`.
2. No error is produced.

## Invariants

1. Lifecycle events with no registered hooks produce empty tier output, not an error.
