# Behavioral Contract BC-1.03.003: Plugin trap (e.g., `unreachable`) reports as Crashed

## Description

A WASM module whose `_start` traps via `unreachable` is reported by the dispatcher as `PluginResult::Crashed { trap_string, ... }`. The crashing plugin does not bubble up to terminate the dispatcher.

## Preconditions

1. WASM module's `_start` instruction reaches `unreachable` (or other trap).

## Postconditions

1. `invoke_plugin` returns `PluginResult::Crashed { trap_string, ... }`.

## Invariants

1. Plugin traps never crash the dispatcher process.
