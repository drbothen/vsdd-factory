# Behavioral Contract BC-2.01.002: HookResult exit codes Continue=0 / Block=2 / Error=1

## Description

`HookResult::exit_code()` returns canonical Claude Code blocking-contract codes: Continue → 0, Block → 2, Error → 1. The dispatcher uses these to set its own process exit code at the end of tier execution.

## Preconditions

1. A `HookResult` variant is in scope.

## Postconditions

1. `Continue.exit_code() == 0`.
2. `Block.exit_code() == 2`.
3. `Error.exit_code() == 1`.

## Invariants

1. Exit-code mapping is total and stable across releases.
