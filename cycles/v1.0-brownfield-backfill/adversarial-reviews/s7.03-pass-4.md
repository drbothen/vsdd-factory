### F-101 (MEDIUM) — BC-INDEX F-002 Option B fix breaks GFM table rendering

**File / Line:** .factory/specs/behavioral-contracts/BC-INDEX.md lines 886-893

**Evidence:** Line 244 SS-05 table header (5-col); lines 245-886 SS-05 rows; line 887 blockquote `> **Note:** ...`; line 888 blank; lines 890-893 four pipe-prefixed rows (the reanchored BCs).

In GFM/CommonMark with table extension, a blockquote terminates a table block. The blockquote on line 887 terminates the SS-05 table at line 886. The four rows on 890-893 form a "headerless table fragment" — they have no `|---|---|` separator immediately above within the same block. They will render as broken markdown (visible pipe characters or degenerate one-row tables).

The pass-3 fix Option B was intended to be lower-blast-radius than promoting to a 6-column table; instead it is structurally worse — the rows are removed from the rendered SS-05 listing entirely.

Cross-evidence: SS-08 footer pattern (BC-INDEX line 1908) uses HTML comment `<!-- ... -->` AFTER the SS-08 table's last row. HTML comments do NOT terminate tables in GFM and don't break rendering. The SS-05 fix should have followed that pattern.

**Recommended fix (Option C/D, minimum-diff):**
- Option C: Move the blockquote on line 887 to AFTER line 893 — between current 893 and 894. Rows continue the table uninterrupted; blockquote becomes a footer comment.
- Option D: Replace blockquote with HTML comment `<!-- ... -->` placed after line 893, matching SS-08 line 1908 pattern verbatim.

Option C preferred (markdown-native, visible to readers).
