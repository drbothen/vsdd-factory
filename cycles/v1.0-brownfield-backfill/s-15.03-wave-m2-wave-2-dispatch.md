## Lessons from S-15.07 Cascade (Apply Preemptively in S-15.11)

These lessons materialized in S-15.07's 6-pass + 4-fix-burst cascade. Apply them from day 1.

1. **Paper-fix detection (TD-VSDD-059):** When an adversary surfaces a defect, the fix must
   STRUCTURALLY plumb the data. S-15.07 had a 3-pass paper-fix chain (F-001 → F-P2-001 → F-P3-001)
   that converged only when the `cited_raw` field was actually plumbed. Doc-comments and renames
   without structural data plumbing are false fixes.

2. **Sibling-sweep canonical-form lock (TD-VSDD-060):** When architect codifies a canonical
   identifier form (e.g., `Edit|Write`), the sweep must include ALL references: production registry +
   test inline registries + spec body + AC verification predicates + doc-comment narrative.
   S-15.07 took 4 fix-bursts to converge this sweep across all 5 reference classes.

3. **Test-assertion regression-codification smell:** When an impl change is driven by an adversary
   finding, do NOT update test assertions to match the new (defective) impl — that codifies the
   regression. S-15.07's implementer regression-codified `"1.5"` after `{:02}` was removed; the
   correct fix restored `"1.05"` body-literal preservation.

4. **Trajectory shape:** S-15.07 cascade: HIGH(6)→NITPICK→HIGH(1)→NITPICK→LOW→CLEAN (6 passes + 4
   fix-bursts). Each MEDIUM/HIGH pass introduced ONE new structural finding while closing prior.
   Expect similar shape for S-15.11; budget 6-8 passes + 3-5 fix-bursts.

5. **Recovery from API 500:** Implementer agent dispatches may hit transient 500s. Orchestrator
   recovery prompt must explicitly enumerate on-disk state inspection + verification gates before
   resuming. S-15.07 recovery caught 2 real bugs (UTF-8 panic on em-dash + envelope quoting).

6. **Inline registry in bats (architect Q6, factory-artifacts e91b5965):** Bats test inline
   registries are IN-SCOPE of the Q5 `Edit|Write` canonical lock. Every `_write_registry()` heredoc
   in bats files must use `tool = "Edit|Write"`, not `Write|Edit`.
