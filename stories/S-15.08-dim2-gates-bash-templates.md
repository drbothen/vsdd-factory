# S-15.08: Dim-2 Gate Template Library — 11 Canonical Bash Scripts for Fix-Burst Attestation

**Epic:** E-12 — Context Resolvers + Platform Stories (brownfield-backfill wave, S-15.03 PRIORITY-A M1)
**Delivery order:** M1 Story 2 of 3 (after S-15.06 SHIPPED, before S-15.16 Part A SHIPPED)
**Depends on:** none (M1 has no internal dependencies — S-15.06 and S-15.16 Part A are parallel)
**Blocks:** M2 WASM hook stories that enforce dim2-gate registration (S-15.07/09/11/14) need the
scripts to exist as registered artifacts before those hooks can reference them.
**Priority:** P1 (closes D-450 through D-454 sub-clauses; unblocks M2)

## Narrative

- **As a** state-manager or agent executing a Dim-2 fix-burst attestation block
- **I want to** invoke canonical, versioned bash scripts that implement each codified mechanical
  gate (trajectory-tail grep, banner wc-l, SHA currency, monotonic-row, propagation-completeness,
  layer-ordinal dual-sweep, META-LEVEL ack, freshness literal-stdout, block-label form, Dim-7
  sibling-sweep, Dim-1 cardinality arithmetic)
- **So that** every Dim-2 attestation block in the burst-log is produced by running a registered,
  reviewable template rather than an ad-hoc hand-rolled grep — preventing the scope-narrowing
  regression pattern codified as D-453(e) and eliminating the META-LEVEL-29 failure class

## Background

The F5 engine-discipline cycle (passes 70–74) identified a persistent pattern: mechanical gates
prescribed in decision-log codifications were hand-rolled as one-off grep commands at attestation
time, and each hand-roll silently narrowed the scope relative to the codified rule. D-453(e) and
D-454(c) mandate that every codified Dim-2 gate have a canonical bash template stored at
`plugins/vsdd-factory/hooks/dim2-gates/<gate-name>.sh`.

PR #137 (commit 21d444d8) scaffolded this directory with a README documenting 5 PLANNED templates.
S-15.08 instantiates all 11 templates (5 README-planned + 6 additional from D-450/451/452
codifications), moves all entries from PLANNED to ACTIVE, and delivers bats test coverage for
each script.

The README's "5 planned" entries correspond to the original D-453/454 scope. The additional 6
scripts implement the literal-shell mechanical gates from D-450(b)/(c)/(d)/(e), D-451(a), and
D-452(a)/(b). Note: the README's fifth entry was `propagation-count-gate.sh` (the D-452(a) pattern
described broadly), which maps to `propagation-completeness.sh`. The D-450(d) gate covers two
independent concerns (banner wc-l + SHA currency) that the dispatch package splits into two
separate scripts for clarity and reuse.

**Author-time tools:** These scripts are invoked by humans and agents during Dim-2 attestation,
NOT by the dispatcher hook chain. They are not registered in hooks-registry.toml. They ship
inside the plugin source tree so they are version-controlled and released alongside the plugin —
deployment target per D-453(e) is `.factory/hooks/dim2-gates/` in each factory project worktree.

## Behavioral Contracts Referenced

None. These are author-time operational tools without runtime dispatcher-enforced behavioral
contracts. See `behavioral_contracts_rationale` in frontmatter for the full justification.

The scripts implement the MECHANICAL SURFACE of the following decision-log codifications:

| Script | Closes | Decision Sub-Clause Summary |
|--------|--------|-----------------------------|
| `trajectory-tail-cell-grep.sh` | D-454(a) | Per-cell line-anchor grep for trajectory_tail at each canonical prescribed site (not file-level count) |
| `freshness-literal-stdout.sh` | D-454(b) | Freshness re-execution with literal stdout capture — command + exit code + raw output, no forward-narrative |
| `block-label-canonical-form.sh` | D-454(d) | Tri-way canonical form alignment: codification text = regex = document headers (D-444(c) canonical block types) |
| `banner-wc-l.sh` | D-450(d) | STATE.md banner "actual N lines" vs `wc -l` arithmetic gate with dual-margin verification |
| `propagation-completeness.sh` | D-452(a) | Post-derivation propagation-completeness check — derived value appears at ALL prescribed sites |
| `dim7-dispatched-count-sweep.sh` | D-450(b) | Dim-7 sibling-sweep across ALL prior burst-log entries for anachronism pattern |
| `dim1-file-count-arithmetic.sh` | D-450(c) | Dim-1 headline cardinality vs enumerated list count arithmetic gate |
| `active-branches-sha-currency.sh` | D-450(d) | `git rev-parse origin/<branch>` currency check for all Active Branches table rows |
| `decision-log-monotonic-rows.sh` | D-450(e) | Decision-log D-NNN ascending-order enforcement (both STATE.md + decision-log.md SoT) |
| `layer-ordinal-dual-direction.sh` | D-452(b) | Layer-N dual-direction sweep: positive Nth-layer confirmation + negative (N±1)th-layer drift detection |

## Acceptance Criteria

### AC-001 — trajectory-tail-cell-grep.sh delivered and passes bats PASS fixture (traces to D-454(a))
Script `plugins/vsdd-factory/hooks/dim2-gates/trajectory-tail-cell-grep.sh` exists, is executable,
accepts positional args `<factory-root> <tail-value> <site-list-file>`, exits 0 on PASS with
human-readable summary showing each site line-anchored grep result, exits 1 on FAIL citing the
specific failing site and expected vs found value. Bats PASS case green against fixture.

### AC-002 — freshness-literal-stdout.sh delivered and passes bats PASS fixture (traces to D-454(b))
Script `plugins/vsdd-factory/hooks/dim2-gates/freshness-literal-stdout.sh` exists, is executable,
accepts positional args `<command-to-rerun>` (quoted), captures and prints the command, its exit
code, and raw stdout verbatim. Exits 0 if command exits 0; exits 1 if command exits non-zero,
printing the failure stdout. Bats PASS case green against fixture.

### AC-003 — block-label-canonical-form.sh delivered and passes bats PASS fixture (traces to D-454(d))
Script `plugins/vsdd-factory/hooks/dim2-gates/block-label-canonical-form.sh` exists, is executable,
accepts positional args `<burst-log-path>`. Verifies all 9 canonical block labels from D-444(c)
("Parent-commit", "Adversary verdict", "Files touched", "Codifications", "Dim-2", "Dim-5", "Dim-6",
"Dim-7", "Closes") appear at least once in the file using the canonical document forms. Exits 0 on
PASS; exits 1 with list of missing labels on FAIL. Bats PASS case green; bats FAIL case detects a
missing label and exits 1.

### AC-004 — banner-wc-l.sh delivered and passes bats PASS fixture (traces to D-450(d) banner sub-clause)
Script `plugins/vsdd-factory/hooks/dim2-gates/banner-wc-l.sh` exists, is executable, accepts
positional arg `<state-md-path>`. Extracts the "actual N lines" figure from the STATE.md banner,
runs `wc -l` on the file, and asserts equality. Also verifies dual-margin arithmetic
(`500 - <wc-l> = <margin>`) matches STATE.md banner margin figure. Exits 0 on PASS with captured
wc-l output; exits 1 on FAIL citing expected vs actual counts and margin arithmetic. Bats PASS
case green; bats FAIL case with injected wrong count exits 1.

### AC-005 — propagation-completeness.sh delivered and passes bats PASS fixture (traces to D-452(a))
Script `plugins/vsdd-factory/hooks/dim2-gates/propagation-completeness.sh` exists, is executable,
accepts positional args `<derived-value> <prescribed-sites-file>` where prescribed-sites-file lists
`<file-path>:<grep-pattern>` pairs (one per line). For each line, runs the grep against the file
and reports PASS/FAIL per site. Exits 0 only if ALL sites pass. Exits 1 with a per-site report
showing which sites have the derived value and which are missing. Bats PASS case green; bats FAIL
case with one site missing exits 1.

### AC-006 — dim7-dispatched-count-sweep.sh delivered and passes bats PASS fixture (traces to D-450(b))
Script `plugins/vsdd-factory/hooks/dim2-gates/dim7-dispatched-count-sweep.sh` exists, is executable,
accepts positional arg `<burst-log-path>`. Sweeps all `## Burst: F5 pass-N` Dim-7 cells in the
file for the anachronism pattern where the dispatched count references a count inconsistent with
the burst number. Reports all matching lines with file location. Exits 0 if no anachronism lines
found; exits 1 with a list of offending lines on FAIL. Bats PASS case (clean fixture) exits 0;
bats FAIL case (injected anachronism) exits 1.

### AC-007 — dim1-file-count-arithmetic.sh delivered and passes bats PASS fixture (traces to D-450(c))
Script `plugins/vsdd-factory/hooks/dim2-gates/dim1-file-count-arithmetic.sh` exists, is executable,
accepts positional arg `<burst-log-path>`. For each Dim-1 headline matching
`^\*\*Files touched \(Dim-1\): [0-9]+ unique files\*\*`, counts the comma-delimited filenames in
the following list line and asserts equality with the headline integer. Exits 0 if all headlines
match; exits 1 with per-headline expected vs actual on FAIL. Bats PASS case green; bats FAIL case
with mismatched count exits 1.

### AC-008 — active-branches-sha-currency.sh delivered and passes bats PASS fixture (traces to D-450(d) SHA sub-clause)
Script `plugins/vsdd-factory/hooks/dim2-gates/active-branches-sha-currency.sh` exists, is executable,
accepts positional args `<factory-root> <state-md-path>`. For each branch row in the Active Branches
table (main, develop, factory-artifacts), runs `git rev-parse origin/<branch>` from `<factory-root>`
and compares to the SHA cell in STATE.md. Exits 0 if all branches match; exits 1 with per-branch
expected vs actual on FAIL. Output includes the literal `git rev-parse` stdout per D-449(a). Bats
PASS case green against fixture with matching SHAs; bats FAIL case with stale SHA exits 1.

### AC-009 — decision-log-monotonic-rows.sh delivered and passes bats PASS fixture (traces to D-450(e))
Script `plugins/vsdd-factory/hooks/dim2-gates/decision-log-monotonic-rows.sh` exists, is executable,
accepts positional arg `<decision-log-path>`. Extracts all `| D-NNN` rows from the file using
regex `^\| D-[0-9]+[( ]`, checks that extracted D-NNN integers are strictly ascending, and reports
any inversion. Exits 0 if strictly ascending; exits 1 with the first inversion pair (D-N before
D-M where N > M) on FAIL. Bats PASS case green; bats FAIL case with inverted rows exits 1.

### AC-010 — layer-ordinal-dual-direction.sh delivered and passes bats PASS fixture (traces to D-452(b))
Script `plugins/vsdd-factory/hooks/dim2-gates/layer-ordinal-dual-direction.sh` exists, is executable,
accepts positional args `<layer-n> <file1> [file2 ...]`. For the given N:
(a) Scans all provided files for positive form `<N>th-layer` (confirmation sweep);
(b) Scans all provided files for negative drift classes `<N-1>th-layer` and `<N+1>th-layer`
    (drift detection sweep).
Reports all occurrences per form, per file. Exits 0 if no drift-class occurrences found (positive
occurrences are informational only); exits 1 with the offending lines if any drift-class occurrence
is found. Bats PASS case (no drift) exits 0; bats FAIL case (injected N+1 drift) exits 1.

### AC-011 — meta-level-ack-grep.sh delivered and passes bats PASS fixture (traces to D-451(a))
Script `plugins/vsdd-factory/hooks/dim2-gates/meta-level-ack-grep.sh` exists, is executable,
accepts positional args `<meta-level-n> <burst-log> <lessons-md> <decision-log> <state-md>`.
Runs `grep -c "META-LEVEL-<N> CANDIDATE CONFIRMED"` against each of the 4 files and prints the
literal counts per file (stdout captured, D-449(a) compliant). Exits 0 if total count across all
4 files is >= 1; exits 1 with per-file counts and total if the acknowledgment is absent from all
4 files. Bats PASS case (acknowledgment present in at least 1 file) exits 0; bats FAIL case
(no acknowledgment in any file) exits 1.
