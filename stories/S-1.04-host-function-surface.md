# S-1.04: Host function surface implementation

**Epic:** E-1 — Dispatcher Foundation
**Phase:** Phase 1 — Dispatcher foundation
**Milestone:** `1.0.0-beta.1`
**Tier:** B.x
**Blocks:** S-1.05, S-2.01, S-3.01, S-3.02, S-3.03
**Blocked by:** S-1.01, S-1.02 (routing types), S-1.03 (ABI contract)
**Estimated effort:** L

## Narrative
- **As a** WASM plugin executing inside the dispatcher
- **I want to** call host functions like `emit_event`, `log`, `read_file`, and `exec_subprocess`
- **So that** plugins can observe and interact with the system through a capability-gated interface

## Goal

Implement the dispatcher-side of every host function the SDK declares:
`emit_event`, `log`, `read_file`, `exec_subprocess`, `session_id`,
`dispatcher_trace_id`, `plugin_root`, `plugin_version`, `cwd`, `env`.
Enforces capability policy and emits `internal.capability_denied` events.

## Contracts

**Host function surface (consumed by S-1.05):**

```
emit_event:        (type_ptr, type_len, fields_ptr, fields_len) -> ()
log:               (level: u32, msg_ptr, msg_len) -> ()
read_file:         (path_ptr, path_len, max_bytes, timeout_ms, out_ptr_out, out_len_out) -> i32
exec_subprocess:   (cmd_ptr, cmd_len, args_ptr, args_len, timeout_ms, max_output_bytes, result_ptr_out) -> i32
session_id:        (out_ptr, out_cap) -> u32
dispatcher_trace_id: (out_ptr, out_cap) -> u32
plugin_root:       (out_ptr, out_cap) -> u32
plugin_version:    (out_ptr, out_cap) -> u32
cwd:               (out_ptr, out_cap) -> u32
env:               (name_ptr, name_len, out_ptr, out_cap) -> i32
```

## Capability Anchor Justification

Anchoring to **CAP-002: "Hook Claude Code tool calls with sandboxed WASM plugins"** per capabilities.md §CAP-002.

> "Every Claude Code tool invocation (Bash, Edit, Write, etc.) triggers the dispatcher, which routes to matching WASM plugins by event type and tool-regex. Plugins run in wasmtime with bounded fuel + epoch timeout."

The host function surface is the capability-gated API through which WASM plugins interact with the host system. Without `exec_subprocess`, `read_file`, `env`, `log`, `emit_event`, and the context getter functions (`session_id`, `dispatcher_trace_id`, `plugin_root`, `plugin_version`, `cwd`) registered with the wasmtime `Linker`, sandboxed plugins cannot perform any meaningful work. This story implements the entire bidirectional interface between the WASM sandbox and the host — the enforcement layer of "sandboxed" in CAP-002.
