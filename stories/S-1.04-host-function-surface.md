# S-1.04: Host function surface implementation

**Epic:** E-1 — Dispatcher Foundation
**Phase:** Phase 1 — Dispatcher foundation
**Milestone:** `1.0.0-beta.1`
**Tier:** B.x
**Blocks:** S-1.05, S-2.01, S-3.01, S-3.02, S-3.03
**Blocked by:** S-1.01, S-1.02 (routing types), S-1.03 (ABI contract)
**Estimated effort:** L

## Narrative
- **As a** WASM plugin executing inside the dispatcher
- **I want to** call host functions like `emit_event`, `log`, `read_file`, and `exec_subprocess`
- **So that** plugins can observe and interact with the system through a capability-gated interface

## Goal

Implement the dispatcher-side of every host function the SDK declares:
`emit_event`, `log`, `read_file`, `exec_subprocess`, `session_id`,
`dispatcher_trace_id`, `plugin_root`, `plugin_version`, `cwd`, `env`.
Enforces capability policy and emits `internal.capability_denied` events.

## Acceptance criteria

- [ ] Each of the 10 host functions implemented and registered with wasmtime `Linker`
- [ ] String arguments read from guest memory via `ptr + len` per the ABI contract in S-1.03
- [ ] `emit_event` enqueues events for sink routing (stubbed for S-1.08/S-1.09)
- [ ] `read_file` rejects paths outside preopened roots with error code; emits `internal.capability_denied`
- [ ] `exec_subprocess` enforces binary allow-list, no shell interpretation, env/cwd allow-lists, timeout, max_output_bytes
- [ ] `env` respects per-plugin allow-list; unknown vars return empty
- [ ] All functions handle OOB memory access gracefully
- [ ] Panic in a host function converts to `internal.host_function_panic` event
- [ ] Each host function covered by unit + integration tests

## Tasks

1. Create `crates/factory-dispatcher/src/host/` module with one file per host function.
2. Common helpers: `read_wasm_string()` and `write_wasm_string()` with bounds-checking.
3. Implement each of the 10 host functions with capability enforcement.
4. Wire all host functions into a shared `Linker` builder.
5. Write unit tests for each function's success and error paths.

## Contracts

**Host function surface (consumed by S-1.05):**

```
emit_event:        (type_ptr, type_len, fields_ptr, fields_len) -> ()
log:               (level: u32, msg_ptr, msg_len) -> ()
read_file:         (path_ptr, path_len, max_bytes, timeout_ms, out_ptr_out, out_len_out) -> i32
exec_subprocess:   (cmd_ptr, cmd_len, args_ptr, args_len, timeout_ms, max_output_bytes, result_ptr_out) -> i32
session_id:        (out_ptr, out_cap) -> u32
dispatcher_trace_id: (out_ptr, out_cap) -> u32
plugin_root:       (out_ptr, out_cap) -> u32
plugin_version:    (out_ptr, out_cap) -> u32
cwd:               (out_ptr, out_cap) -> u32
env:               (name_ptr, name_len, out_ptr, out_cap) -> i32
```

## Architecture Mapping

| Component | Module | Pure/Effectful |
|-----------|--------|---------------|
| host/emit_event.rs | crates/factory-dispatcher/src/host/ | effectful-shell (enqueues events) |
| host/exec_subprocess.rs | crates/factory-dispatcher/src/host/ | effectful-shell (spawns processes) |
| host/read_file.rs | crates/factory-dispatcher/src/host/ | effectful-shell (reads filesystem) |
| host/helpers.rs | crates/factory-dispatcher/src/host/ | pure-core (memory read/write helpers) |

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | `read_file` path outside preopen | Reject with error code; emit `internal.capability_denied` |
| EC-002 | `exec_subprocess` binary not in allow-list | Reject; emit `internal.capability_denied` |
| EC-003 | OOB guest memory access | Return error without crashing dispatcher |
| EC-004 | Host function panics | Convert to `internal.host_function_panic` event |
