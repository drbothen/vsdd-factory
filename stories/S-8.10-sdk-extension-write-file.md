# S-8.10: SDK extension: host::write_file (D-6 Option A unblocker)

**Epic:** E-8 — Native WASM Migration Completion
**Phase:** Phase 1 — SDK Extension (W-15, pre-Tier-2)
**Milestone:** `v1.1`
**Tier:** SDK Infrastructure (D-6 Option A)
**Blocks:** S-8.04 (update-wave-state-on-merge), S-8.09 (regression-gate + adapter retirement)
**Blocked by:** S-8.00
**Estimated effort:** S (1 day, 3 pts)

## Narrative

- **As a** vsdd-factory hook plugin author
- **I want to** call `host::write_file(path, contents, timeout_ms)` from a native WASM plugin
- **So that** state-mutating hooks (update-wave-state-on-merge, regression-gate) can be ported to native WASM without depending on a missing SDK function

## Capability Anchor Justification

**Primary anchor:** CAP-022 ("Port hook plugins from bash to native WASM") per
`domain-spec/capabilities.md` §CAP-022. S-8.10 is a direct enabling dependency for
CAP-022: porting `update-wave-state-on-merge.sh` (S-8.04) and `regression-gate.sh`
(S-8.09) to native WASM requires `host::write_file` in the SDK, which is currently
absent (confirmed via `crates/hook-sdk/src/host.rs` — `write_file` does not appear
in the file; `read_file` exists at line 187).

**Subsystem anchor:** SS-02 (Hook SDK and Plugin ABI). The new host function
is added to `crates/hook-sdk/` and `crates/factory-dispatcher/src/host/`, which are
the canonical SS-02 modules per ARCH-INDEX line 75. SS-02 owns `crates/hook-sdk/`
and `crates/hook-sdk-macros/`. SS-04 (Plugin Ecosystem, line 77) owns
`crates/hook-plugins/` — NOT the SDK surface. The anchor is SS-02 only.

**ABI Policy anchor:** D-6 Option A (additive extension) — adding a new host export
does not break existing callers because wasmtime resolves exports by name and existing
plugins that do not import `write_file` are unaffected. HOST_ABI_VERSION stays at 1
per D-6 (Option B — version bump — is explicitly disallowed for v1.x; Option A
additive extension is the approved path). This is confirmed by BC-2.01.003 (line 150
in BC-INDEX): "HOST_ABI_VERSION is 1 in both crates" — our story must NOT bump this.

## Open Questions

**OQ-1 — BC anchor (BLOCKING for status=ready):**
No existing BC in BC-INDEX covers `host::write_file`. The closest family is BC-2.01.x
(SS-02 ABI invariants; BC-2.01.003 covers HOST_ABI_VERSION = 1) and BC-7.04.x
(host facade behaviors for Tier 2 validator hooks). The recommended new BC slot is
**BC-2.01.005** (or the next available slot after BC-2.01.004 per BC-INDEX line 151):
"host::write_file exists in vsdd-hook-sdk and round-trips a write call to dispatcher."
PO must author and anchor BC-2.01.005 (or equivalent) before this story transitions
to `ready`. This OQ is the reason `behavioral_contracts: []` above — per Spec-First
Gate S-7.01.

**OQ-2 — HOST_ABI_VERSION negotiation (resolved as STAY=1):**
D-6 Option A "additive extension" means plugins compiled against SDK 0.1.x that do
not import `write_file` continue to work against a dispatcher that exports `write_file`.
Wasmtime silently ignores additional host exports that a plugin does not import.
Decision: HOST_ABI_VERSION stays at 1. No missing-import detection or backward-compat
negotiation is needed. This is consistent with E-8 Goal #5 ("HOST_ABI_VERSION = 1
unchanged throughout E-8") and D-6 Option B ("never in v1.x"). Document this as
`AS-DEC` (architectural decision) in the PR description at delivery time.

## Acceptance Criteria

### AC-1: SDK write_file wrapper
`host::write_file(path: &str, contents: &[u8], timeout_ms: u32) -> Result<(), HostError>`
exists in `crates/hook-sdk/src/host.rs` and round-trips a write call through the
dispatcher's `vsdd::write_file` host import. The function signature is symmetric to
`host::read_file` (same error codes via `HostError::from_code`).
(traces to OQ-1: BC-2.01.005 postcondition 1 — pending BC authorship)

### AC-2: Dispatcher write_file binding
A new file `crates/factory-dispatcher/src/host/write_file.rs` implements the
host-side wasmtime binding for `vsdd::write_file` with:
- Path validation: same `path_allowed` policy as `read_file.rs` (deny-by-default;
  paths must be in `capabilities.write_file.path_allow`).
- Timeout enforcement: `timeout_ms` parameter accepted for ABI stability (identical
  to `read_file.rs` line 36: "accepted for ABI stability; enforced in S-1.5 via
  epoch interruption").
- Content received as `(contents_ptr: u32, contents_len: u32)` from guest memory via
  `memory::read_wasm_bytes` (symmetric to `read_file`'s `read_wasm_string`).
- Registered in `mod.rs::setup_linker` as `write_file::register(&mut linker)?;`.
(traces to OQ-1: BC-2.01.005 postcondition 2 — pending BC authorship)

### AC-3: HOST_ABI_VERSION stays at 1
`grep HOST_ABI_VERSION crates/hook-sdk/src/lib.rs` returns `pub const HOST_ABI_VERSION: u32 = 1;`.
`grep HOST_ABI_VERSION crates/factory-dispatcher/src/lib.rs` returns `pub const HOST_ABI_VERSION: u32 = 1;`.
No version bump occurs. PR description records AS-DEC: "additive write_file export
is ABI-stable; D-6 Option A applies; HOST_ABI_VERSION = 1 unchanged."
(traces to BC-2.01.003 invariant 1 — HOST_ABI_VERSION = 1 in both crates)

### AC-4: SDK unit tests
`crates/hook-sdk/src/host.rs` unit tests (non-wasm target) cover:
- (a) `write_file` stub returns a non-panic `Err(HostError::CapabilityDenied)` (matching stub behavior of `read_file` stub which returns -1).
- (b) Timeout parameter is accepted without panic (stub behavior identical to `read_file`).
- (c) `HostError::from_code` mapping is unchanged (no new error codes added).
(traces to OQ-1: BC-2.01.005 postcondition 3 — pending BC authorship)

### AC-5: Dispatcher integration tests
`crates/factory-dispatcher/src/host/write_file.rs` has unit tests (symmetric to
`crates/factory-dispatcher/src/host/read_file.rs` test block, lines 151-205) covering:
- (a) Denies when no `capabilities.write_file` block is present.
- (b) Writes allowed file to a tmp dir when path is within `path_allow`.
- (c) Rejects path outside `path_allow` list.
- (d) Rejects content exceeding a `max_bytes` cap (if a cap parameter is introduced;
  if omitted for MVP, document the omission as a known gap in the PR).
The integration test may use `tempfile::tempdir()` (already in dev-dependencies via
`read_file.rs` tests).
(traces to OQ-1: BC-2.01.005 postcondition 4 — pending BC authorship)

### AC-6: Legacy plugin regression preserved
Existing plugins that use only `host::read_file` and `host::emit_event` compile
without change and pass `cargo test`. No existing test in the dispatcher test suite
regresses. The `crates/factory-dispatcher/tests/host_functions.rs` test file must
pass after this story's changes.
(traces to BC-2.01.003 invariant 2 — ABI backward compatibility)

### AC-7: vsdd-hook-sdk minor version bump
`crates/hook-sdk/Cargo.toml` version bumped from `0.1.0` to `0.2.0`. A CHANGELOG
entry is added in the SDK crate (or the workspace CHANGELOG) under "## Added" with:
"host::write_file — write a file at the given path through the dispatcher's bounded
host function."
(traces to OQ-1: BC-2.01.005 postcondition 5 — pending BC authorship)

### AC-8: ABI catalog documentation
`docs/ABI.md` (or `crates/hook-sdk/HOST_ABI.md` if that is the canonical path)
lists `write_file` in the host export catalog with:
- Signature: `vsdd::write_file(path_ptr, path_len, contents_ptr, contents_len, timeout_ms) -> i32`.
- Timeout semantics: same epoch-interruption policy as `read_file`.
- Safety policy: path must be within plugin's declared `write_file.path_allow` list;
  traversal attempts return `CAPABILITY_DENIED (-1)`.
(traces to OQ-1: BC-2.01.005 postcondition 6 — pending BC authorship)

## Architecture Compliance Rules

1. **SS-02 boundary:** All changes are within `crates/hook-sdk/` and
   `crates/factory-dispatcher/src/host/`. No files outside these two modules are
   modified (except CHANGELOG and docs/ABI.md for AC-7/AC-8).
2. **Deny-by-default capability model:** `write_file` follows the same
   deny-by-default pattern as `read_file`: no capability block = `CAPABILITY_DENIED`.
   See `read_file.rs` lines 73-77 for the reference implementation pattern.
3. **Memory safety:** Guest memory access uses `memory::read_wasm_bytes` (or the
   equivalent read helper in `memory.rs`). No raw pointer arithmetic.
4. **Error code stability:** No new negative error codes introduced. The existing set
   {-1 CapabilityDenied, -2 Timeout, -3 OutputTooLarge, -4 InvalidArgument, -99 InternalError}
   is sufficient. `write_file` returns 0 on success, -1 on capability denial, -4 on
   path traversal, -99 on io error.
5. **HOST_ABI_VERSION invariant:** Both `crates/hook-sdk/src/lib.rs:58` and
   `crates/factory-dispatcher/src/lib.rs:43` must still read `= 1` after this story.
   CI must verify this (or the story-writer's AC-3 covers it via grep assertion).
6. **Workspace Cargo.toml:** No new crate is added. This story modifies two existing
   crates; `Cargo.toml` workspace `members` array is unchanged.
