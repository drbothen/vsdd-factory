# S-4.05: Dead letter queue implementation

**Epic:** E-4 — Observability Sinks and RC Release
**Phase:** Phase 4 — Observability sinks
**Milestone:** `1.0.0-rc.1`
**Tier:** E
**Blocks:** S-4.07, S-4.08
**Blocked by:** S-4.04 (retries need to exhaust before DLQ)
**Estimated effort:** S

## Narrative
- **As a** factory operator
- **I want to** have events that exhaust their retries written to a dead letter file
- **So that** I can inspect and replay dropped events rather than losing them silently

## Goal

When events are dropped — because the sink's retry attempts exhausted,
or the outbound queue overflowed — write them to a per-sink dead letter
file. This prevents silent data loss.

## Acceptance criteria

- [ ] Per-sink dead letter file at `.factory/logs/dlq/<sink-name>.jsonl`
- [ ] Events written to DLQ when all retries exhausted
- [ ] Events written to DLQ when outbound queue overflows (queue full)
- [ ] DLQ file has daily rotation matching file sink rotation
- [ ] `internal.sink_dlq_write` event emitted for each DLQ write
- [ ] Integration test: sink with mock server always returning 5xx; verify events in DLQ
