# S-4.07: End-to-end observability integration tests

**Epic:** E-4 — Observability Sinks and RC Release
**Phase:** Phase 4 — Observability sinks
**Milestone:** `1.0.0-rc.1`
**Tier:** E
**Blocks:** S-4.08
**Blocked by:** S-3.01..S-3.04, S-4.01..S-4.06
**Estimated effort:** M

## Narrative
- **As a** release engineer preparing 1.0.0-rc.1
- **I want to** have end-to-end integration tests covering all sink configurations and event flows
- **So that** rc.1 ships with verified observability behavior under realistic configurations

## Goal

End-to-end validation that the observability system works under
realistic configurations. Exercises: zero-disk mode, hybrid mode,
multi-sink fan-out, routing filters, DLQ, circuit breaker.

## Acceptance criteria

- [ ] Test: zero-disk mode (otel-grpc only; no file sink) — events appear in mock OTLP receiver
- [ ] Test: hybrid mode (file + otel-grpc) — events in both destinations
- [ ] Test: multi-sink fan-out with routing filters — audit events in audit sink, metrics in metrics sink
- [ ] Test: sink failure → DLQ — mock 5xx; events appear in DLQ file
- [ ] Test: circuit breaker — repeated failures → circuit opens; `internal.sink_circuit_opened` emitted
- [ ] All tests green in CI
