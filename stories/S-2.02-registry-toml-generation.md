# S-2.02: hooks-registry.toml auto-generation from existing hooks.json

**Epic:** E-2 — Legacy Adapter and Beta Release
**Phase:** Phase 2 — Legacy bash adapter
**Milestone:** `1.0.0-beta.1`
**Tier:** C
**Blocks:** S-2.07, S-2.08
**Blocked by:** S-2.01
**Estimated effort:** S

## Narrative
- **As a** factory operator upgrading from v0.79.x
- **I want to** have `hooks-registry.toml` auto-generated from the existing `hooks.json`
- **So that** the v1.0 dispatcher starts with full parity to the existing hook set without manual entry

## Goal

Auto-generate `hooks-registry.toml` from the existing `hooks/hooks.json`
so the v1.0 dispatcher starts life with full parity to the v0.79.x hook
set. Every entry in the existing hooks.json becomes a `[[hooks]]` entry
in the TOML pointing to `legacy-bash-adapter.wasm` with the original
script path as a config parameter.

## Acceptance criteria

- [ ] `scripts/generate-registry-from-hooks-json.sh` reads `hooks/hooks.json` and produces `hooks-registry.toml`
- [ ] Every hook entry in JSON maps to a `[[hooks]]` block with `plugin = "legacy-bash-adapter.wasm"` and `config.script_path = "<original command>"`
- [ ] Generator is idempotent — re-running produces identical output
- [ ] 6 bats tests covering: normal generation, idempotency, missing input, schema mapping
- [ ] Generated TOML validates against `schema_version = 1`

## Capability Anchor Justification

Anchoring to CAP-007: "Deploy and activate the plugin on any supported platform" per capabilities.md §CAP-007.
Justification: S-2.02 generates hooks-registry.toml from hooks.json, bridging the v0.79.x hook set into the v1.0 dispatcher format. The generated registry is the input to VP-049 (Generated hooks-registry.toml Round-Trips Through Registry::load), which validates that the deployment artifact the dispatcher consumes is structurally valid across all supported platforms. BC-9.01.005 co-anchors this story because generate-registry-from-hooks-json.sh reads hooks.json (the gitignored runtime file that activate writes), confirming that the activation gate precondition is correctly set up before the generator runs.

Note: S-2.02 is also SS-07 scoped (legacy bash adapter ecosystem). The primary architectural driver for this story's SS-09 contribution is registry generation feeding into the deployment pipeline.
