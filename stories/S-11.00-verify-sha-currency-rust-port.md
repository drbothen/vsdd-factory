# Story S-11.00: Port verify-sha-currency.sh to Rust binary for cross-OS support

## Status

**STUB** — full story authoring (acceptance criteria, BCs, anchor justification, test plan) deferred to story-writer post-E-9 completion. This file exists to register the user-directed work item in the index.

## Description

Port `verify-sha-currency.sh` (currently a bash script in `plugins/vsdd-factory/templates/`) to a Rust binary for cross-OS support (Linux, macOS, Windows). The script is a state-manager burst hygiene gate that verifies cross-branch SHA agreement, multi-commit-chain detection, and tense-flip patterns in active-pass narrative.

This is a follow-up to E-9 (Tier 2 Native WASM Migration). E-9's `validate-wave-gate-prerequisite` (S-9.07) invokes `verify-sha-currency.sh` via `host::exec_subprocess`. Once E-9 lands, the WASM-ported hook will still be invoking a bash script — limiting cross-OS portability. Migrating the script itself to a Rust binary closes that gap.

## Motivation

User directive (2026-05-06): "we should migrate that script as well to rust just like we did the rest of our system, so it can work across OS's."

Cross-OS support enables:
- Windows-based downstream customers to run state-manager burst hygiene gates
- Removal of bash dependency in `verify-sha-currency.sh` invocation chain
- Consistency with the broader Rust/WASM migration pattern (E-3, E-8, E-9)

## Scope (Preliminary — to be refined)

- Implement Rust binary (likely under `crates/state-manager-tooling/` or similar)
- Behavioral parity with current bash script:
  - Cross-branch SHA agreement check (develop SHA cited in STATE.md/HANDOFF.md vs actual develop HEAD)
  - Cross-record SHA agreement (wave-state.yaml gate_pass_N remediation_sha vs STATE.md frontmatter)
  - Multi-commit-chain guard (chains > 2 commits on factory-artifacts)
  - Tense-flip detection in active-pass narrative (advisory)
- Cross-platform git operations (likely via gitoxide crate or cmd::Command-based git invocation)
- Update plugin distribution to include the new binary (alongside or replacing the bash script)
- Update state-manager checklist + agents/state-manager.md to reference new binary path
