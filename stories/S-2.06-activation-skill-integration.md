**Epic:** E-2 — Legacy Adapter and Beta Release

## Capability Anchor Justification

CAP-007 ("Deploy and activate the plugin on any supported platform") per capabilities.md §CAP-007. This story is the integration that makes CAP-007's full outcome real: it takes the platform-detection result from S-0.03 and the CI-generated hooks.json variants from S-0.04, copies the correct per-platform hooks.json, verifies the dispatcher binary, and writes the activation block to settings.local.json. This is the direct implementer of "/vsdd-factory:activate succeeds on all five platforms without manual path configuration."

## Narrative
- **As a** factory operator running `/vsdd-factory:activate`
- **I want to** have the skill detect my platform and copy the correct `hooks.json.<platform>` into place
- **So that** the v1.0 dispatcher is wired into Claude Code automatically

## Goal

Take the platform-detection logic from S-0.03 and the CI-generated
`hooks.json.<platform>` variants from S-0.04, and wire them into the
`/vsdd-factory:activate` skill so that activation copies the right variant.

## Acceptance criteria

- [ ] `/vsdd-factory:activate` detects platform and copies `hooks.json.<platform>` to `hooks.json`
- [ ] Activation verifies the target binary exists at the expected path before copying
- [ ] Post-copy validation: reads the copied `hooks.json`, confirms dispatcher path matches platform
- [ ] Activation records `activated_platform` in `.claude/settings.local.json`
- [ ] Activation success message includes platform string and binary path
- [ ] Full end-to-end test: activation on darwin-arm64 produces correct hooks.json
