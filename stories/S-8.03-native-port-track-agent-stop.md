**Complexity note (Low — 3 pts):** track-agent-stop is the simplest Tier 1 hook. It:

conditional blocking (on_error = "continue"). The BLOCKED regex is shared with
pr-manager-completion-guard for parity (same pattern).

## Narrative

- **As a** vsdd-factory operator monitoring subagent lifecycle telemetry
- **I want** track-agent-stop to execute as a native WASM plugin
- **So that** agent.stop telemetry events are emitted on all platforms (including
  Windows without git-bash), with lower latency, and without adapter indirection

## Goal

Port `plugins/vsdd-factory/hooks/track-agent-stop.sh` to a native Rust WASM crate at
`crates/hook-plugins/track-agent-stop/`. The hook fires for ALL SubagentStop events
(no agent scoping), reads `agent_type` and `last_assistant_message` from stdin JSON,
classifies the result into one of three exit classes:

- `empty`: trimmed non-whitespace length == 0
- `blocked`: result matches the BLOCKED regex (`^(Status:\|##?\s*)?\s*BLOCKED`)
- `ok`: all other cases

Then emits `host::emit_event` with `type=agent.stop hook=track-agent-stop
matcher=SubagentStop subagent=<agent_type> exit_class=<class> result_len=<N>`.
Always exits 0. The `on_error = "continue"` registry setting means dispatcher errors
do not block; this is preserved.

The BLOCKED regex reuses the same pattern as pr-manager-completion-guard (BC-7.03.047)
for consistency per the bash comment: "Matches the BLOCKED detection used by
pr-manager-completion-guard for consistency."

## Acceptance Criteria

### AC-001 (traces to BC-7.03.081 postcondition 1)
WASM crate `crates/hook-plugins/track-agent-stop/` exists with `Cargo.toml` targeting
`wasm32-wasi`, implements vsdd_hook_sdk hook interface, builds successfully. Registry
entry updated: `plugin = "hook-plugins/track-agent-stop.wasm"`, `script_path` and
`shell_bypass_acknowledged` removed. Binding preserved: `event = "SubagentStop"`,
`priority = 930`, `on_error = "continue"`, `timeout_ms = 5000`.

### AC-002 (traces to BC-7.03.081 invariant 1)
`hooks.json` command entry for `track-agent-stop` deleted from all platform-specific
files and template. `plugins/vsdd-factory/hooks/track-agent-stop.sh` deleted.

### AC-003 (traces to BC-7.03.082 postcondition 1)
WASM plugin:
- Computes `RESULT_LEN` = trimmed non-whitespace byte count of `last_assistant_message`
  (fallback: `result`, then empty string).
- Classifies `EXIT_CLASS`: `empty` if RESULT_LEN == 0; `blocked` if result matches
  BLOCKED pattern; `ok` otherwise.
- Calls `host::emit_event` with `type=agent.stop hook=track-agent-stop
  matcher=SubagentStop subagent=<agent_type> exit_class=<EXIT_CLASS>
  result_len=<RESULT_LEN>`.
- Always exits 0.

### AC-004 (traces to BC-7.03.082 postcondition 1)
Bats parity tests cover: (a) empty result → EXIT_CLASS=empty, result_len=0, exit 0;
(b) result with `Status: BLOCKED` → EXIT_CLASS=blocked, exit 0; (c) non-empty non-BLOCKED
result → EXIT_CLASS=ok, result_len=N, exit 0; (d) missing `last_assistant_message` field
→ treated as empty (RESULT_LEN=0), EXIT_CLASS=empty, exit 0. All tests pass.

### AC-005 (traces to BC-7.03.082 invariant 1)
`bin/emit-event` calls replaced with `host::emit_event`. No reference to `bin/emit-event`
remains in `crates/hook-plugins/track-agent-stop/`. `bin/emit-event` binary NOT removed
(E-8 D-10). WASM warm-invocation perf measured; within 20% of S-8.00 Tier 1 baseline.
