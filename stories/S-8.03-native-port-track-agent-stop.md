## Narrative

- **As a** vsdd-factory operator monitoring subagent lifecycle telemetry
- **I want** track-agent-stop to execute as a native WASM plugin
- **So that** agent.stop telemetry events are emitted on all platforms (including
  Windows without git-bash), with lower latency, and without adapter indirection

## Goal

Port `plugins/vsdd-factory/hooks/track-agent-stop.sh` to a native Rust WASM crate at
`crates/hook-plugins/track-agent-stop/`. The hook fires for ALL SubagentStop events
(no agent scoping), reads `agent_type` and `last_assistant_message` from stdin JSON
via the `HookPayload` typed-projection (BC-2.02.012), classifies the result into one
of three exit classes:

- `empty`: trimmed non-whitespace length == 0
- `blocked`: result matches the BLOCKED regex (`^(Status:\s*|##?\s*)?\s*BLOCKED`)
- `ok`: all other cases

Then emits `host::emit_event` with positional `event_type = "agent.stop"` and
KV fields `hook=track-agent-stop matcher=SubagentStop subagent=<agent_type>
exit_class=<class> result_len=<N>`. Always exits 0. The `on_error = "continue"`
registry setting means dispatcher errors do not block; this is preserved.

The BLOCKED regex reuses the same pattern as pr-manager-completion-guard (BC-7.03.047)
for consistency per the bash comment: "Matches the BLOCKED detection used by
pr-manager-completion-guard for consistency."

### AC-003 (traces to BC-7.03.082 postcondition 1; traces to BC-2.02.012 postcondition 5; traces to BC-2.02.012 postcondition 6)
WASM plugin:
- Resolves agent identity using the canonical typed-projection fallback chain per
  BC-2.02.012 Postcondition 5: `payload.agent_type.as_deref().or(payload.subagent_name.as_deref()).unwrap_or("unknown")`.
- Resolves assistant-message content using the canonical typed-projection fallback chain
  per BC-2.02.012 Postcondition 6: `payload.last_assistant_message.as_deref().or(payload.result.as_deref()).unwrap_or("")`.
- Computes `RESULT_LEN` = trimmed non-whitespace BYTE count of the resolved message
  (mirroring bash `tr -d '[:space:]' | wc -c` byte semantics). Canonical Rust form:
  `.bytes().filter(|b| !b.is_ascii_whitespace()).count()`.
  Note: `.len()` returns byte count for ASCII-only inputs but is NOT equivalent for
  mixed-content strings; use the filter form for correctness on all inputs.
- Classifies `EXIT_CLASS`: `empty` if RESULT_LEN == 0; `blocked` if result matches
  BLOCKED pattern (with `(?m)` multiline flag so `^` matches start-of-line, mirroring
  bash `grep` per-line semantics); `ok` otherwise.
- Calls `host::emit_event` with positional `event_type = "agent.stop"` and KV fields:
  `hook=track-agent-stop`, `matcher=SubagentStop`, `subagent=<agent_type>`,
  `exit_class=<EXIT_CLASS>`, `result_len=<RESULT_LEN>`.
- Always exits 0.

### AC-004 (traces to BC-7.03.082 postcondition 1; traces to BC-2.02.012 postcondition 5; traces to BC-2.02.012 postcondition 6)
Bats parity tests cover: (a) empty result → EXIT_CLASS=empty, result_len=0, exit 0;
(b) result with `Status: BLOCKED` → EXIT_CLASS=blocked, exit 0; (c) non-empty non-BLOCKED
result → EXIT_CLASS=ok, result_len=N, exit 0; (d) missing `last_assistant_message` field
but `result` field present → typed-projection fallback chain advances to `result` field
(BC-2.02.012 Postcondition 6), EXIT_CLASS derived accordingly, exit 0; (e) both
`last_assistant_message` and `result` absent/null → typed-projection resolves to `""`
(BC-2.02.012 Postcondition 6 final `unwrap_or("")`), RESULT_LEN=0, EXIT_CLASS=empty,
exit 0; (f) `agent_type` absent but `subagent_name` present → typed-projection fallback
chain resolves identity to `subagent_name` value (BC-2.02.012 Postcondition 5), correct
`subagent` field in emit_event. All tests pass.

### AC-007 (traces to BC-7.03.081 invariant 1)
Malformed or missing JSON stdin → graceful exit 0; no panic, no stderr backtrace.
Bats fixture: send invalid JSON (e.g., `echo "not-json"` piped to hook); assert exit 0,
assert stderr contains no Rust panic/backtrace text.
Note: input trimming uses `is_ascii_whitespace()` which excludes vertical tab (0x0B);
bash trim via `tr -d '[:space:]'` includes 0x0B. Behavioral divergence on 0x0B inputs
is accepted and not tested (0x0B is pathological in dispatcher JSON envelopes).
