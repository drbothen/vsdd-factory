**Complexity note (Medium — 5 pts):** This hook is more complex than handoff-validator
because it: (a) scopes to pr-manager agents only (pattern match on agent_type), (b)
counts STEP_COMPLETE emissions via regex, (c) detects BLOCKED status as a legitimate
early exit, (d) computes LAST_STEP/NEXT_STEP from grep output, (e) selects one of 9
step-specific hint strings, and (f) exits 2 (hard block) with a multi-line stderr
injection. The `gh` CLI in the registry capabilities (`binary_allow = ["bash", "gh", "jq"]`) is
present as a dormant allowlist entry; the bash version does not call `gh` — it reads
stdin JSON and writes to stderr only. The WASM port preserves this no-gh behavior;
`host::exec_subprocess` is NOT required.

## Narrative

- **As a** vsdd-factory operator running pr-manager subagents
- **I want** pr-manager-completion-guard to execute as a native WASM plugin
- **So that** the FM4 guard (detecting early stop before all 9 PR lifecycle steps complete) runs
  on all platforms without git-bash, with lower latency and deterministic step-hint injection

## Goal

Port `plugins/vsdd-factory/hooks/pr-manager-completion-guard.sh` to a native Rust WASM
crate at `crates/hook-plugins/pr-manager-completion-guard/`. The hook:

1. Reads SubagentStop JSON from stdin; extracts `agent_type` (with 3-arm fallback
   `agent_type ?? subagent_name ?? "unknown"`) and the result text via fallback chain
   `last_assistant_message ?? result ?? ""` (mirrors bash line 26:
   `jq -r '.last_assistant_message // .result // empty'`).
2. Scopes to pr-manager agents only (pattern: contains `pr-manager` or contains
   `pr_manager`); passes for all other agents.
3. Counts *lines containing* `STEP_COMPLETE:` in result (line-occurrence semantics,
   matching `grep -c`; one line with multiple tokens = 1); if count >=8, passes (exit 0).
4. If result matches `BLOCKED` pattern at line start, passes (exit 0) — legitimate early exit.
5. Otherwise: computes LAST_STEP (highest step= number in STEP_COMPLETE lines, or 0),
   NEXT_STEP = LAST_STEP + 1, selects one of 9 hint strings, emits hook.block event,
   writes multi-line stderr injection, exits 2.

`on_error = "block"` at the registry level means dispatcher errors block pr-manager stop.
This is preserved in the migrated registry entry.

### AC-004 (traces to BC-7.03.047 postcondition 1)
When `agent_type` contains pr-manager pattern AND result matches BLOCKED pattern at line
start (`^(Status:|##?\s*)?\s*BLOCKED`): WASM plugin exits 0 (legitimate early exit).
Note: Rust regex crate uses ERE-style alternation; do not escape `|` — `\|` is a
parse error or literal match, not alternation. Use unescaped `|` as in bash ERE.
Verify bats test: fixture with `Status: BLOCKED` result → exit 0, no block.

### AC-005 (traces to BC-7.03.048 postcondition 1)
When agent is pr-manager, STEP_COMPLETE count < 8, no BLOCKED: WASM plugin calls
`host::emit_event` with event_type `"hook.block"` and fields:
`hook=pr-manager-completion-guard`, `matcher=SubagentStop`,
`reason=pr_manager_incomplete_lifecycle`, `subagent=<agent>`,
`step_count=<N>`, `last_step=<L>`, `next_step=<L+1>`.

**Agent field rendering:** `<agent>` is resolved via the 3-arm fallback chain
`agent_type ?? subagent_name ?? "unknown"`. If the chain resolves to `"unknown"`
(all three arms absent or null), emit `("subagent", "unknown")` as a literal field
value — do NOT omit the field.

**Stderr injection — pinned line:** The following stderr line MUST appear verbatim:
```
CONTINUE TO STEP <N> NOW: <hint>
```
where `<N>` is the decimal value of NEXT_STEP and `<hint>` is selected by NEXT_STEP
from the 10-arm table below. The remaining stderr lines (preamble, separator, footer)
are implementation-defined; their exact wording is not contracted here (rationale:
the bash source produces output in a specific style but no downstream consumer
parses those lines programmatically — only the CONTINUE line is machine-readable).

**Hint table (verbatim from bash source):**

| NEXT_STEP | Hint string (verbatim) |
|-----------|----------------------|
| 1 | populate PR description from template |
| 2 | verify demo evidence (or emit status=na for chore PRs) |
| 3 | create PR via github-ops |
| 4 | spawn security-reviewer via Agent tool |
| 5 | spawn pr-reviewer/pr-review-triage via Agent tool; handle findings; converge |
| 6 | spawn github-ops: gh pr checks --watch |
| 7 | verify all dependency PRs merged |
| 8 | spawn github-ops: gh pr merge --squash --delete-branch (AUTHORIZE_MERGE=yes mode) |
| 9 | confirm branch deletion; write review-findings.md; emit final STEP_COMPLETE |
| * (wildcard, NEXT_STEP >= 10 or unknown) | continue the 9-step lifecycle |

Exits 2.
