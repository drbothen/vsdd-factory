## Capability Anchor Justification

**Primary anchor:** CAP-022 ("Port hook plugins from bash to native WASM") per
`domain-spec/capabilities.md` §CAP-022. S-8.02 ports `pr-manager-completion-guard.sh`
to native WASM, eliminating legacy-bash-adapter indirection for this FM4-prevention hook.

**Stretch-Anchor Disclosure:** S-8.02 anchors to CAP-022 but spans two subsystems:
- **SS-07 Hook Bash Layer** owns `pr-manager-completion-guard.sh` and its BC-7.03.045–048
  behavioral contracts. SS-07 is the source-of-truth for this hook's behavior.
- **SS-01 Hook Dispatcher Core** owns the hooks-registry.toml routing and the
  host fn surface used by the native crate. The `gh` CLI subprocess call in the bash
  version is replaced by `host::exec_subprocess` in the WASM version (or equivalent
  host fn if available in vsdd-hook-sdk).

Both subsystem anchors verified against ARCH-INDEX Subsystem Registry:
- SS-01 Hook Dispatcher Core: canonical name confirmed.
- SS-07 Hook Bash Layer: canonical name confirmed.
(Per POLICY 5 anchor justification requirement + POLICY 6 architecture-is-source-of-truth.)

**Complexity note (Medium — 5 pts):** This hook is more complex than handoff-validator
because it: (a) scopes to pr-manager agents only (pattern match on agent_type), (b)
counts STEP_COMPLETE emissions via regex, (c) detects BLOCKED status as a legitimate
early exit, (d) computes LAST_STEP/NEXT_STEP from grep output, (e) selects one of 9
step-specific hint strings, and (f) exits 2 (hard block) with a multi-line stderr
injection. The `gh` CLI in the registry capabilities (`binary_allow = ["bash", "gh",
"jq"]`) is present for potential future use but the current bash version does not
directly call `gh` — it parses result text only. The WASM port preserves this behavior.

## Narrative

- **As a** vsdd-factory operator running pr-manager subagents
- **I want** pr-manager-completion-guard to execute as a native WASM plugin
- **So that** the FM4 guard (detecting early stop before all 9 PR lifecycle steps complete) runs
  on all platforms without git-bash, with lower latency and deterministic step-hint injection

## Goal

Port `plugins/vsdd-factory/hooks/pr-manager-completion-guard.sh` to a native Rust WASM
crate at `crates/hook-plugins/pr-manager-completion-guard/`. The hook:

1. Reads SubagentStop JSON from stdin; extracts `agent_type` and `last_assistant_message`.
2. Scopes to pr-manager agents only (pattern: `*pr-manager*` or `*pr_manager*`); passes for all other agents.
3. Counts `STEP_COMPLETE:` occurrences in result; if >=8, passes (exit 0).
4. If result matches `BLOCKED` pattern at line start, passes (exit 0) — legitimate early exit.
5. Otherwise: computes LAST_STEP (highest step= number in STEP_COMPLETE lines, or 0),
   NEXT_STEP = LAST_STEP + 1, selects one of 9 hint strings, emits hook.block event,
   writes multi-line stderr injection, exits 2.

`on_error = "block"` at the registry level means dispatcher errors block pr-manager stop.
This is preserved in the migrated registry entry.

### AC-004 (traces to BC-7.03.047 postcondition 1)

### AC-005 (traces to BC-7.03.048 postcondition 1)
When agent is pr-manager, STEP_COMPLETE count < 8, no BLOCKED: WASM plugin calls
`host::emit_event` with `type=hook.block hook=pr-manager-completion-guard
matcher=SubagentStop reason=pr_manager_incomplete_lifecycle subagent=<agent>
step_count=<N> last_step=<L> next_step=<L+1>`. Writes multi-line stderr injection
including `"CONTINUE TO STEP <N> NOW: <hint>"` where hint is one of the 9 step-specific
strings (steps 1–9 per bash source). Exits 2.

### AC-006 (traces to BC-7.03.048 postcondition 1)
Bats parity tests cover all 9 step positions: each test fixture has `STEP_COMPLETE:
step=N` for N=0..8, verifies NEXT_STEP hint text matches the bash version verbatim.
Tests also cover: (a) non-pr-manager agent → exit 0, no block; (b) >=8 steps → exit 0;
(c) BLOCKED result → exit 0; (d) 0 steps → NEXT_STEP=1, hint="populate PR description".
All bats tests pass.

### AC-007 (traces to BC-7.03.045 postcondition 1)
`bin/emit-event` calls replaced with `host::emit_event`. No reference to `bin/emit-event`
remains in `crates/hook-plugins/pr-manager-completion-guard/`. `bin/emit-event` binary
