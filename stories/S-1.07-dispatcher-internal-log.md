# S-1.07: dispatcher-internal.jsonl writer

**Epic:** E-1 — Dispatcher Foundation
**Phase:** Phase 1 — Dispatcher foundation
**Milestone:** `1.0.0-beta.1`
**Tier:** B.x
**Blocks:** S-1.08, S-2.07, S-4.07
**Blocked by:** S-1.01, S-1.02
**Estimated effort:** S

## Narrative
- **As a** factory operator debugging dispatcher behavior
- **I want to** inspect `.factory/logs/dispatcher-internal.jsonl`
- **So that** every lifecycle event and internal telemetry is always persisted, regardless of sink configuration

## Goal

Implement the always-on internal self-telemetry log at
`.factory/logs/dispatcher-internal.jsonl`. Every lifecycle event and
`internal.*` event written by the dispatcher goes here in addition to
(or instead of) user-configured sinks. This log is the operator's
reliable fallback for debugging.

## Acceptance criteria

- [ ] `internal_log.rs` implements `InternalLogger` with JSONL append to `.factory/logs/dispatcher-internal.jsonl`
  _(traces to BC-1.06.001 postcondition: best-effort writes; never panics; BC-1.06.010 postcondition: log_dir pointing at existing file → silent return without panic)_
- [ ] Log file path derived from `CLAUDE_PLUGIN_ROOT` env var
  _(traces to BC-1.06.003 postcondition: parent directories auto-created on first write; BC-1.06.005 postcondition: prune_old is no-op when log dir missing)_
- [ ] All 17+ internal event constants defined (`dispatcher.started`, `dispatcher.completed`, `internal.dispatcher_error`, `plugin.invoked`, etc.)
  _(traces to BC-1.06.006 postcondition: InternalEvent fields flatten to top-level JSON; BC-1.06.008 postcondition: None optional fields skipped in serialization)_
- [ ] Each JSONL entry: `{"timestamp":"...", "type":"...", "trace_id":"...", "attrs":{...}}`
  _(traces to BC-1.06.007 postcondition: 10 events write 10 JSONL lines with expected shape; BC-1.06.009 postcondition: 4-event startup flow round-trips through JSONL with correct envelope)_
- [ ] `InternalLogger::emit(event)` is synchronous (no async buffering for reliability)
  _(traces to BC-1.06.001 invariant: synchronous write guarantees event is persisted before emit() returns or fails silently)_
- [ ] Log file created on first write if missing; parent dirs created
  _(traces to BC-1.06.003 postcondition: auto-creates missing parent directories)_
- [ ] Daily log rotation: new file per calendar day (UTC) e.g. `dispatcher-internal.2026-04-25.jsonl`
  _(traces to BC-1.06.002 postcondition: separate files per UTC date; invariant: filename derived from event timestamp not wall clock)_
- [ ] Integration test: invoke dispatcher, verify expected events in log
  _(traces to BC-1.06.004 postcondition: prune_old removes only `dispatcher-internal-*.jsonl` files older than threshold; BC-1.06.005 postcondition: prune_old no-op when log dir missing)_

## Tasks

## Capability Anchor Justification

Anchoring to **CAP-010: "Always-on dispatcher self-telemetry independent of sink config"** per capabilities.md §CAP-010.

> "`dispatcher-internal-YYYY-MM-DD.jsonl` is written for every invocation regardless of whether any sink is configured or healthy. 30-day rotation."

This story is the direct implementation of CAP-010. It creates `internal_log.rs` with `InternalLogger`, which appends JSONL events to daily-rotated files under `CLAUDE_PLUGIN_ROOT/logs/`. The "independent of sink config" guarantee is a behavioral contract (BC-1.06.001: best-effort, never panic) rather than a configuration option. The `prune_old` function provides the 30-day rotation. This story is the sole origin of this capability.
