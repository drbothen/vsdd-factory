# S-15.04: Tighten bats trace_id assertion to canonical wire format (TD #66 closure)

**Epic:** E-15 — Plugin Async Semantics — Registry-Layer Partition (single-shot delivery)
**Delivery order:** Fourth story in E-15; TD cleanup following S-15.01 merge.
**Depends on:** S-15.01 (serde rename `dispatcher_trace_id` → `trace_id` on internal-log wire field must be live)
**Blocks:** none
**Priority:** P2 (test hygiene; closes transitional workaround from PR #113)

## Narrative

- **As a** maintainer of the dispatcher's internal-log wire format
- **I want to** have the bats regression assertion require the canonical `trace_id` key (and forbid the legacy `dispatcher_trace_id` key)
- **So that** any future regression in serialization is caught at the integration test layer rather than masked by the transitional dual-accept regex inherited from PR #113

## Background

S-15.01 task T-3a renamed the internal-log wire field from `dispatcher_trace_id` to `trace_id`
via `#[serde(rename = "trace_id")]` on `InternalEvent.dispatcher_trace_id`
(at `crates/factory-dispatcher/src/internal_log.rs:116-117`). The Rust unit tests in
`internal_log_integration.rs:105-113` already verify zero `dispatcher_trace_id` in wire output.

The bats test at `regression-v1.0.bats:118` was widened to `'"(dispatcher_)?trace_id":"'` at
PR #113 as a transitional workaround to absorb the rename across the delivery window. Investigation
confirms the widening is purely defensive dead-code: the field is canonical `trace_id` and has
been since S-15.01 shipped. The optional-group regex is live test surface with no behavioral
justification.

**Critical distinction — two separate fields:**

The plugin stdin envelope's `dispatcher_trace_id` (documented in `crates/hook-sdk/HOST_ABI.md`)
is a SEMANTICALLY DISTINCT field — different surface (plugin input envelope, not internal log
output), different purpose (correlation ID passed INTO the plugin), intentionally named
`dispatcher_trace_id`. TD #66 does NOT touch HOST_ABI. The rename in S-15.01 applied exclusively
to the internal-log wire format; the SDK-facing envelope name remains `dispatcher_trace_id` and
is out of scope for this story.
