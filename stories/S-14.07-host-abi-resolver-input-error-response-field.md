# S-14.07: HOST_ABI Resolver-Input Schema: Add error-response Field

**Epic:** E-14 — Engine Discipline Pass-2 — Process-Gap Remediation
**Delivery order:** SEVENTH in E-14. Can be delivered independently.
**Depends on:** (none — HOST_ABI.md is a docs-only change)
**Blocks:** (none)
**Priority:** P1

## Narrative

- **As a** WASM resolver author reading HOST_ABI.md to understand the resolver ABI
- **I want** the ResolverInput schema documented in HOST_ABI.md to include the
  error-response field that BC-4.12.002 mandates
- **So that** resolver implementations correctly handle the error-response payload
  and don't silently drop it, and downstream adversarial passes don't re-raise
  the same gap

## Background

F-P2-010 (pass-2 adversarial finding) identified that the HOST_ABI.md resolver-input
schema section is missing the `error_response` field that BC-4.12.002 mandates as
part of the resolver ABI payload schema. A WASM resolver author reading HOST_ABI.md
as the canonical reference for the resolver ABI would not know this field exists.

This finding was triaged in D-378 as FOLLOW-UP STORY requiring:
1. HOST_ABI.md amendment — add `error_response` field to ResolverInput schema table
2. BC-4.12.002 update — add cross-reference to the HOST_ABI section
3. Unit test confirming the field is present in the serialized ResolverInput struct

## Acceptance Criteria

### AC-001: Read current HOST_ABI.md ResolverInput schema section
- Locate the ResolverInput schema table in HOST_ABI.md (Context Injection Contract section)
- Document all currently listed fields
- Confirm `error_response` is absent (or document if it was added since F-P2-010)

### AC-002: Add error_response field to HOST_ABI.md ResolverInput schema
- Add `error_response` field documentation matching BC-4.12.002 ABI payload spec
- Include: field name, type, description, when present vs absent (Option semantics)
- Maintain the existing table/section format

### AC-003: Update BC-4.12.002 cross-reference
- Add or update the HOST_ABI anchor in BC-4.12.002 to reference the updated section
- Bump BC-4.12.002 version by minor
- Add changelog entry citing F-P6-002/S-14.07

### AC-004: Add unit test for error_response field presence in ResolverInput
- In the appropriate crate (crates/factory-dispatcher or vsdd-context-resolvers),
  add a unit test asserting `ResolverInput` serialization includes `error_response`
- Test must be part of `cargo test --workspace --all-targets`
