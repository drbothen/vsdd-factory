# S-5.04: PostToolUseFailure hook wiring

**Epic:** E-5 — New Hook Events and 1.0.0 Release
**Phase:** Phase 5 — New Claude Code hook events

**Tier:** G

## Capability Anchor Justification

Primary: CAP-013 ("Capture post-execution activity (PostToolUse hooks)") per capabilities.md §CAP-013. PostToolUseFailure extends the PostToolUse capture capability to error events — capturing `tool.error` events that track tool call failures for observability dashboards.

FR anchor: FR-007 ("Legacy hook routing compatibility + dispatcher main") per PRD §2.1, extended. PostToolUseFailure is a new event type that extends the dispatcher's event routing, part of proposed FR-045. It also relates to CAP-013 outcome ("tool errors for audit and observability purposes").

Cross-subsystem note: The PostToolUseFailure payload includes `tool_name` alongside standard envelope fields, making BC-1.02.004 (PostToolUse envelope with tool_response) at SS-01 the closest existing dispatcher-side BC. The SS-04 plugin behavior has no existing BCs; all are v1.1 candidates.

## Narrative
- **As a** factory operator tracking tool reliability
- **I want to** receive tool.error events when a tool call fails
- **So that** I can track tool error rates in my observability dashboards

## Goal

Wire `PostToolUseFailure` — fires after a tool call fails. Emits
`tool.error` events that feed a Factory ROI error-rate dashboard panel.
Event stream is in v1.0; dashboard panel is post-1.0.
