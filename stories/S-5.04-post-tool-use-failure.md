Primary: CAP-002 ("Emit structured telemetry events for lifecycle and tool activity") per capabilities.md §CAP-002. CAP-002 covers the full lifecycle event emission pipeline including failure paths; PostToolUseFailure is one of the five FR-046 lifecycle events (S-5.01–S-5.04) all anchored to CAP-002. CAP-002 was widened during ADV-S5.04-P01 sibling sweep to explicitly include the failure path for audit and observability purposes, consistent with S-5.01/S-5.02/S-5.03 capability anchors. Subsystems: SS-01 (dispatcher routing), SS-04 (WASM plugin), SS-07 (bash layer hooks.json.template).

FR anchor: FR-046 ("New Claude Code lifecycle hook events: SessionStart/SessionEnd/WorktreeCreate/WorktreeRemove/PostToolUseFailure") per PRD §FR-046, extended. PostToolUseFailure is the fourth of four Tier F lifecycle events. BC pattern mirrors S-5.02/S-5.03 (zero-capability, emit-only plugin).

BC pattern: BC-4.08.001 (plugin emits tool.error), BC-4.08.002 (hooks.json.template Layer 1 registration), BC-4.08.003 (hooks-registry.toml Layer 2 routing). 3 BCs for a single-event zero-capability plugin — fewer than S-5.01 (5 BCs + subprocess) and S-5.02 (5 BCs) because PostToolUseFailure has no idempotency BC (no once:true at Layer 1; fires per-failure).

S-5.01/02/03 lessons-applied note: All critical corrections from S-5.01 (14-pass), S-5.02 (9-pass), S-5.03 (14-pass) applied up-front: 4+4 opaque RESERVED_FIELDS grouping; event_name (not event_type) in envelope dispatch; ZERO declared capabilities (Option A; mirrors S-5.02/S-5.03); once key ABSENT (fires per-failure; mirrors S-5.03 worktree pattern); session_id is RESERVED_FIELDS host-enriched (per BC-1.05.012); plugin MUST NOT set it; F-07 Option 1 semantic scope SS-04 only (narrowed from legacy SS-01+SS-04); platform variants must be regenerated after template edit (S-5.03 PR-cycle-1 lesson). S-5.03 BC-4.07.001 was bumped to v1.3 retroactively as part of ADV-S5.04-P01 HIGH-P01-002 sibling sweep (no S-5.03 story body change needed per merged-stories-untouched convention).

RESERVED_FIELDS treatment (4+4 opaque grouping per BC-4.08.001/S-5.03 pass-2 reversal): 8 RESERVED_FIELDS the plugin MUST NOT set, opaque from spec layer — (a) 4 host-enriched (`dispatcher_trace_id`, `session_id`, `plugin_name`, `plugin_version`) set by `emit_event` host fn from HostContext; (b) 4 construction-time (`ts`, `ts_epoch`, `schema_version`, `type`) set by dispatcher between plugin `emit_event` call and final wire format. Implementation provenance of group (b) is opaque from the spec layer.

## Narrative
- **As a** factory operator tracking tool reliability
- **I want to** receive tool.error events when a tool call fails
- **So that** I can track tool error rates in my observability dashboards

## Goal

Wire `PostToolUseFailure` — fires after a tool call fails. Emits `tool.error` events (with `tool_name` + `error_message` fields) that feed a Factory ROI error-rate dashboard panel. Event stream is in v1.0; dashboard panel is post-1.0.
