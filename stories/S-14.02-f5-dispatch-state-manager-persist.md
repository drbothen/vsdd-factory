# S-14.02: F5 Dispatch → State-Manager Persist Automation

**Epic:** E-14 — Engine Discipline Pass-2 — Process-Gap Remediation
**Delivery order:** SECOND in E-14 (after S-14.01). Forward-looking for pass-2 cycle.
**Depends on:** S-14.01 (bootstrap backfill must be settled before documenting the
  canonical F5 dispatch flow)
**Blocks:** (none)
**Priority:** P1

## Narrative

- **As a** vsdd-factory orchestrator dispatching F5 adversarial review
- **I want** the F5 phase skill (or lobster workflow step) to automatically dispatch
  state-manager after adversary returns chat output, so the adversary review is
  persisted to the canonical path without manual intervention
- **So that** no F5 dispatch fails to write its output because the adversary's
  read-only profile prevents direct writes, and the orchestrator does not need to
  remember to manually dispatch state-manager afterward

## Background

F5 pass-1 adversarial review (adv-cycle-pass-1.md) surfaced that the adversary
agent's dispatch was expected to write its own review document — but `agents/adversary.md`
explicitly specifies `tool_access: read-only`, meaning the adversary returns findings
as chat text only. The current F5 dispatch flow depends on a manual post-dispatch step
where the orchestrator captures the chat output and dispatches state-manager to persist
it. This works when the orchestrator remembers to do it, but fails silently when it
doesn't (as observed in the failed F5 dispatch that preceded the successful one).

The fix is to automate the capture-and-persist sequence as a documented step in the
F5 phase skill or lobster workflow: (1) dispatch adversary as subagent, (2) capture
chat text, (3) dispatch state-manager with the captured text as the content to write
to the canonical `adv-<slug>.md` path.

**Source:** F5 pass-1 OBS-2 / F-LOW-2, fix-plan Section 5 PG-1. The adversary
text: "The dispatch process has a mismatch with the agent's documented tool profile.
Future F5 dispatches will keep hitting the same friction."

## Behavioral Contracts

No existing BCs cover the F5 dispatch-to-persist automation. New BCs are required.
They are declared here for authoring at story implementation start:

- **BC-X.NN.001 (to be authored at story implementation start):** F5 phase skill
  MUST NOT dispatch the adversary with instructions to write directly to any `.factory/`
  path. Adversary is dispatched read-only; all writes are delegated to state-manager.
- **BC-X.NN.002 (to be authored at story implementation start):** F5 phase skill
  MUST dispatch state-manager after adversary subagent returns, passing the full
  adversary chat text as the content to persist to the canonical adversary-review path.
  The persist step MUST NOT be optional or conditional on orchestrator memory.

The BC subsystem assignment (SS-NN) and sequence numbers will be determined by the
architect when these BCs are authored. They belong in SS-05 (Pipeline Orchestration)
or SS-06 (Skill Catalog) depending on whether the implementation targets the lobster
workflow or the phase-5 skill itself.

## Acceptance Criteria

### AC-001 (traces to F5 fix-plan PG-1 — adversary dispatched read-only)

The F5 adversarial-refinement skill (`plugins/vsdd-factory/skills/phase-f5-scoped-adversarial/SKILL.md`
or equivalent — implementer must verify exact path) contains NO instruction that asks
the adversary to write, save, or output to a file path. All such instructions are
removed or replaced with instructions to return findings as chat text only.

**Falsifiable test:** `grep -in "write\|save to\|output to\|the path\|emit to file"
plugins/vsdd-factory/skills/phase-f5-scoped-adversarial/SKILL.md` returns zero hits
for any language directing the adversary to write to disk. (Note: the skill may still
reference the canonical path for labeling purposes — what's prohibited is instructions
to perform the write.)

### AC-002 (traces to F5 fix-plan PG-1 — state-manager persist step)

The F5 phase skill (or its lobster workflow equivalent) includes a documented post-adversary
step labeled "Persist adversary review" (or equivalent). This step: (a) captures the
adversary subagent's full chat output, (b) dispatches state-manager with the content,
(c) specifies the canonical write path as `.factory/cycles/<cycle-id>/adv-<slug>.md`.
The step is not wrapped in any optional or conditional block — it fires unconditionally
after adversary dispatch.

**Falsifiable test:** The phase skill or workflow file contains a step or section
containing the phrase "state-manager" and "persist" (or "write") in proximity.
`grep -n "state-manager" plugins/vsdd-factory/skills/phase-f5-scoped-adversarial/SKILL.md`
(or the lobster file) returns ≥1 match.

### AC-003 (traces to PG-1 intent — no manual orchestrator memory required)

The MANDATORY STEPS section of orchestrator AGENT.md (or orchestrator.md — implementer
must verify exact path) is updated to explicitly document the two-step F5 dispatch
sequence: (1) dispatch adversary subagent (read-only), (2) dispatch state-manager to
persist chat output. The mandatory step removes any language implying the orchestrator
must "remember" to persist the review after dispatch.

**Falsifiable test:** `grep -n "state-manager.*persist\|persist.*state-manager\|F5.*state-manager"
plugins/vsdd-factory/agents/orchestrator/orchestrator.md` (or equivalent path) returns
≥1 match showing the two-step sequence is documented.

### AC-004 (traces to PG-1 intent — adversary.md read-only profile clarification)

`plugins/vsdd-factory/agents/adversary.md` contains a clarifying note in the
`tool_access: read-only` section that explicitly states: "Findings are returned as
chat text only. The F5 dispatch flow is responsible for persisting via state-manager.
Never include instructions for the adversary to write to any path." The note cites
the F-LOW-2 process-gap finding as its origin.

**Falsifiable test:** `grep -n "chat text\|state-manager\|persist" plugins/vsdd-factory/agents/adversary.md`
returns ≥1 match confirming the note is present.

### AC-005 (traces to PG-1 intent — lobster workflow step if applicable)

If a lobster workflow file controls F5 dispatch (check for
`plugins/vsdd-factory/workflows/phases/phase-5*.lobster` or similar), it contains:
- A step `dispatch-adversary` with the adversary skill/agent as the target
- A step `persist-adversary-review` (or `backup-adversary-review`) immediately after,
  with `depends_on: [dispatch-adversary]`, targeting state-manager for write

If no lobster file governs F5, this AC is satisfied by documenting in the PR that F5
is driven by the skill directly (not a lobster workflow) and the skill content satisfies
AC-001 through AC-004.

**Falsifiable test:** If lobster file exists: `grep "persist-adversary-review\|backup-adversary"
<lobster-file>` returns ≥1 match. If no lobster file: PR description documents this
explicitly.
