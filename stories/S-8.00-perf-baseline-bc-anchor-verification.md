## Narrative

- **As a** story-writer and implementer preparing to port 9 Tier 1 bash hooks to native WASM
- **I want to** know (a) the actual bash hook warm-invocation latency on the CI runner

  regression baseline, and no latent test coverage gaps that would surface mid-port

## Goal

Establish the two enabling artifacts that all 9 Tier 1 port stories (S-8.01..S-8.09)
depend on:

1. **Responsibility A (Perf baseline):** Measure actual warm-invocation latency for
   one representative bash hook from each tier (Tier 1, Tier 2, Tier 3) under the
   project's CI runner profile. Commit results to `.factory/measurements/E-8-bash-baseline.json`.
   If the measured per-plugin latency differs materially from the assumed ~10ms/plugin,
   adjust E-8 epic AC-7b's 200ms p95 ceiling and Goal #6 accordingly via fix-burst
   on E-8 v1.7 → v1.8.

2. **Responsibility B (BC-anchor verification):** For all 9 Tier 1 hooks, identify the

## Tasks (MANDATORY)

### Responsibility A — Perf Benchmark Baseline

- [ ] A.0: Verify `hyperfine` >=1.18 is available on the CI runner (ubuntu-latest).
       Run `hyperfine --version` on the runner; if absent, install via
       `sudo apt-get install hyperfine` (preferred on ubuntu-latest CI runner, ~2s binary
       install) or `cargo install hyperfine` (fallback for non-Debian runners; compiles
       from source) and confirm the CI devtools matrix includes it before proceeding.
       hyperfine is REQUIRED for reproducible median measurement (per F-P2-004: `time`
       builtin is NOT acceptable).
- [ ] A.1: Create `tests/perf/` directory (confirmed absent; not the same as `tests/`
       integration tests which also don't exist — create both if needed as scaffold)
- [ ] A.2: Write `tests/perf/E-8-bash-baseline.bats` bats test file:
       - Select 3 representative hooks for timing: `handoff-validator.sh` (Tier 1,
         SubagentStop, Medium complexity — block-mode, JSON parse + threshold logic),
         `validate-bc-title.sh` (Tier 2, Edit|Write, Low complexity — grep+awk),
         `protect-bc.sh` (Tier 3, PreToolUse, Medium complexity — block-mode,
         permissionDecision envelope)
       - Each test invokes the hook via its full path with a representative fixture
         input (stdout/stderr captured; exit code checked)
       - Each test runs 10 warm invocations and records median wall-clock time using
         `hyperfine --warmup 3 --runs 10`. The bats harness wraps the hyperfine
         invocation per hook. (Rationale: reproducibility of median measurement is
         load-bearing for E-8 epic AC-7b ceiling adjudication; `hyperfine` provides
         statistical warmup + runs count in one command, eliminating shell-builtin
         variance. The `time` builtin branch is NOT used.)
       - Justification for bats + shell timing vs Criterion Rust benchmarks: the
         bash hooks run as shell processes — Criterion measures Rust fn internals,
