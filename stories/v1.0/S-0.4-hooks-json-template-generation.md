# S-0.4: hooks.json.template + CI generation

**Epic:** [v1.0 Factory Plugin Kit](EPIC.md)
**Phase:** Phase 0 — Infrastructure prep
**Milestone:** `1.0.0-beta.1`
**Tier:** A (contract-frozen; binaries reference comes from S-1.x)
**Blocks:** S-2.3, S-2.6, S-2.8
**Blocked by:** none
**Estimated effort:** M

## Goal

Establish the source-of-truth `hooks.json.template` format and a CI step
that renders it into 5 platform-specific `hooks.json.<platform>` variants.
The template uses placeholders that the generator substitutes with the
correct platform-specific dispatcher-binary path for each variant. CI
commits the generated variants as part of the release chore commit and
verifies there's no drift between template and variants on every push.

## Acceptance criteria

- [ ] `plugins/vsdd-factory/hooks/hooks.json.template` exists, committed,
  documented
- [ ] Placeholder syntax is explicit and minimal (proposed: `{{PLATFORM}}`
  and `{{EXE_SUFFIX}}`)
- [ ] Generator script (`scripts/generate-hooks-json.sh` or similar) reads
  the template and produces 5 variants at
  `plugins/vsdd-factory/hooks/hooks.json.<platform>`
- [ ] Each generated variant has an absolute-path-style command field:
  `${CLAUDE_PLUGIN_ROOT}/hooks/dispatcher/bin/<platform>/factory-dispatcher[.exe]`
- [ ] `hooks.json` (the canonical one) is in `.gitignore`
- [ ] CI job "generate-hooks-json-variants" runs on every push to main (and
  on tags); diffs output against committed variants; fails if drift
- [ ] Release workflow regenerates variants as part of the release chore
  commit so the committed versions always match the template at tag-time
- [ ] Generator is covered by bats tests (input template + expected outputs)

## Tasks

1. Design the template file. Single template structure; only the dispatcher
   binary path varies per platform.
2. Define the placeholder set. Start with `{{PLATFORM}}` and `{{EXE_SUFFIX}}`
   (the latter is `.exe` on Windows, empty elsewhere).
3. Write `scripts/generate-hooks-json.sh` — POSIX bash, iterates the 5
   platforms, sed-substitutes placeholders, writes output files.
4. Write a matching PowerShell variant `scripts/generate-hooks-json.ps1`?
   **No** — generation happens only in CI (Linux runner), not on Windows
   dev machines. Skip the PS1.
5. Add `.gitignore` entry for `plugins/vsdd-factory/hooks/hooks.json`.
6. Commit initial versions of `hooks.json.template` and the 5
   `hooks.json.<platform>` files. These won't reference real binaries yet
   (S-1.x produces those) — they're structural placeholders.
7. Add CI job that runs the generator and diffs against committed output.
   Job fails if drift detected.
8. Add bats test: temp-dir-based run of the generator with a fixture
   template, assertions on each generated file.

## Contracts

**Template placeholder contract:**

```
{{PLATFORM}}      → "darwin-arm64" | "darwin-x64" | "linux-x64" |
                    "linux-arm64" | "windows-x64"
{{EXE_SUFFIX}}    → "" for Unix, ".exe" for Windows
```

**Generated variant shape** (example, `hooks.json.darwin-arm64`):

```json
{
  "hooks": {
    "PreToolUse":        [ { "hooks": [ { "type": "command", "command": "${CLAUDE_PLUGIN_ROOT}/hooks/dispatcher/bin/darwin-arm64/factory-dispatcher", "timeout": 10000 } ] } ],
    "PostToolUse":       [ { "hooks": [ { "type": "command", "command": "${CLAUDE_PLUGIN_ROOT}/hooks/dispatcher/bin/darwin-arm64/factory-dispatcher", "timeout": 10000, "async": true } ] } ],
    "PermissionRequest": [ { "hooks": [ { "type": "command", "command": "${CLAUDE_PLUGIN_ROOT}/hooks/dispatcher/bin/darwin-arm64/factory-dispatcher", "timeout": 10000 } ] } ],
    "Stop":              [ { "hooks": [ { "type": "command", "command": "${CLAUDE_PLUGIN_ROOT}/hooks/dispatcher/bin/darwin-arm64/factory-dispatcher", "timeout": 10000, "async": true } ] } ],
    "SubagentStop":      [ { "hooks": [ { "type": "command", "command": "${CLAUDE_PLUGIN_ROOT}/hooks/dispatcher/bin/darwin-arm64/factory-dispatcher", "timeout": 10000, "async": true } ] } ],
    "SessionStart":      [ { "hooks": [ { "type": "command", "command": "${CLAUDE_PLUGIN_ROOT}/hooks/dispatcher/bin/darwin-arm64/factory-dispatcher", "timeout": 10000, "async": true, "once": true } ] } ],
    "SessionEnd":        [ { "hooks": [ { "type": "command", "command": "${CLAUDE_PLUGIN_ROOT}/hooks/dispatcher/bin/darwin-arm64/factory-dispatcher", "timeout": 10000, "async": true, "once": true } ] } ]
  }
}
```

Non-blocking events (PostToolUse, Stop, SubagentStop, SessionStart,
SessionEnd) get `"async": true` per design doc's Q3 resolution — ensures
Claude Code doesn't hang on a pathological dispatcher. Blocking events
(PreToolUse, PermissionRequest) stay sync so `exit 2` still blocks.

## Test plan

- **Generator tests**: fixture template + run generator + diff each
  expected variant. Covers placeholder substitution correctness.
- **CI drift check**: simulate a hand-edit of a variant, verify CI fails
  with a useful diff.
- **Template schema validity**: each generated variant must pass JSON
  parsing and, ideally, a hook-config schema validator once we have one.

## Notes

The 7 hook events listed in the template (PreToolUse, PostToolUse,
PermissionRequest, Stop, SubagentStop, SessionStart, SessionEnd) cover
every event the dispatcher plugs into for v1.0. Additional events from
Phase 5 (SessionStart/End are in this set; WorktreeCreate/Remove and
PostToolUseFailure are added in S-5.x when those stories land).

`"once": true` on SessionStart/End prevents re-entry if the same session
re-evaluates (harmless safety; CC docs recommend it for these events).

Template rendering is simple sed substitution; no Jinja/Mustache dep.
If templating complexity grows (multi-line conditionals, loops), revisit.
