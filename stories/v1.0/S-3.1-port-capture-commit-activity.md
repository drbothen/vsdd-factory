# S-3.1: Port capture-commit-activity to WASM

**Epic:** [v1.0 Factory Plugin Kit](EPIC.md)
**Phase:** Phase 3 — High-value port to native WASM
**Milestone:** `1.0.0-rc.1`
**Tier:** E
**Blocks:** S-4.7, S-4.8
**Blocked by:** S-2.8 (beta.1 shipped), S-3.4 (emit_event host fn)
**Estimated effort:** M

## Goal

Rewrite `capture-commit-activity` from bash to native WASM using the
`vsdd-hook-sdk`. First of four priority ports. Parses `git commit`
invocations from PostToolUse/Bash payloads, extracts sha/branch from
stdout, emits `commit.made` events. Port removes subprocess overhead,
adds Windows support, and validates the SDK against a real hook.

## Acceptance criteria

- [ ] `capture-commit-activity.wasm` produced by `cargo build --target
  wasm32-wasip1 --release`
- [ ] Matches the bash version's behavior exactly:
  - Only fires on PostToolUse Bash with `git commit` (argv-matched via
    the same regex as the bash version)
  - Parses `[<branch> <sha>]` preamble from stdout
  - Extracts sha (7+ hex chars), branch, message subject
  - Detects `--amend` and sets `amended: "true"`
  - Emits `commit.made` event with fields matching current schema
  - Silent on failures (always returns Continue)
- [ ] No subprocess calls; pure Rust
- [ ] Uses `vsdd-hook-sdk` primitives only (no FFI direct calls)
- [ ] Registered in `hooks-registry.toml` with priority same as current
  priority tier for commit telemetry
- [ ] Replaces the legacy-bash-adapter routing for this hook
- [ ] Event parity test: 20 real-world `git commit` stdout samples, run
  through both bash version + WASM version, verify identical events
- [ ] Latency < 5ms per invocation (vs ~50ms bash subprocess)
- [ ] Works on all 5 CI platforms (including Windows — first hook that
  does)

## Tasks

1. Create `crates/hook-plugins/capture-commit-activity/`.
2. Implement `on_hook` in Rust matching the bash logic:
   - Check tool_name == "Bash"
   - Parse command, match `git commit` via regex
   - Parse stdout first line for `[...]` preamble
   - Tokenize bracket contents; extract sha, branch
   - Validate sha format (7-40 hex)
   - Check for `--amend` flag
   - emit_event("commit.made", [("commit_sha", ...), ("branch", ...),
     ("message_subject", ...), ("amended", ...)])
3. Update `hooks-registry.toml` to point this hook at the new WASM file
   instead of the adapter. Same tool filter, same priority, same
   `on_error`.
4. Add unit tests in the plugin crate: 20 sample bash commit outputs,
   verify parsing extracts correct sha/branch.
5. Write event-parity integration test: real factory sandbox, run
   against both implementations, diff events.
6. Benchmark: 1000 invocations, verify < 5ms p99.

## Contracts

**Event emitted (matches current schema exactly):**

```json
{
  "type": "commit.made",
  "commit_sha": "abc1234",
  "branch": "main",
  "message_subject": "feat: something",
  "amended": "true"              // only if --amend
}
```

Plus automatic dispatcher enrichments (`dispatcher_trace_id`,
`session_id`, `plugin_name`, `plugin_version`, `ts`, `ts_epoch`,
`schema_version`).

**No behavior change** observed by Factory ROI dashboard or any downstream
consumer. This is a pure migration.

## Test plan

- Unit: 20+ fixture `(command, stdout, exit_code)` tuples; verify
  emitted event structure
- Event parity: same fixtures fed to both bash + WASM, `jq --sort`
  compare of events
- Latency benchmark p50/p95/p99 < 5ms
- Cross-platform: CI builds + runs the plugin tests on all 5 platforms

## Notes

First native port — this validates the whole SDK design in practice.
Expect some friction; feed discoveries back into S-1.3 if ergonomics
issues surface.

The bash version is at
`plugins/vsdd-factory/hooks/capture-commit-activity.sh`. Keep it on disk
(marked deprecated) until the adapter is retired post-1.0; useful
reference during the port.

Regex crate caveat: `regex` compiles to reasonable wasm32-wasip1 size
(~200KB added), but verify it fits the overall budget. If not,
hand-parse the `git commit` match with string operations.

Fuel cap: this plugin is trivially fast; default 10M fuel is plenty.
