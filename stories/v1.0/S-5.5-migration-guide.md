# S-5.5: Migration guide (0.79.x → 1.0)

**Epic:** [v1.0 Factory Plugin Kit](EPIC.md)
**Phase:** Phase 5 — New Claude Code hook events
**Milestone:** `1.0.0`
**Tier:** G
**Blocks:** S-5.7
**Blocked by:** S-4.8 (rc.1 shipped; all behavior stable)
**Estimated effort:** M

## Goal

Write the complete migration guide at `docs/guide/migrating-from-0.79.md`
(skeleton from S-0.5). Walks operators through the upgrade step-by-step:
what changed, what's required, what's optional, troubleshooting, rollback.
Gate 16 of the 1.0.0 success criteria requires this to be validated
against at least one real 0.79.x factory.

## Acceptance criteria

- [ ] `docs/guide/migrating-from-0.79.md` complete (no TODO markers)
- [ ] Covers: what changed, prerequisites, step-by-step upgrade,
  verification, rollback, troubleshooting, Windows-specific notes,
  observability migration, known regressions
- [ ] Validated by following the guide on at least one real 0.79.x
  factory that isn't vsdd-factory itself (dogfood covered by S-4.8)
- [ ] External validator's feedback incorporated
- [ ] Docs linked from CHANGELOG entry for 1.0.0 and from the GH Release
  notes

## Tasks

1. Draft content based on resolved design decisions.
2. Sections to cover:
   - **Why v1.0**: matcher bug, cross-platform, observability flexibility
   - **Prerequisites**: Claude Code version, git, jq, Python NOT required
   - **Upgrade procedure**:
     - `/plugin update vsdd-factory@vsdd-factory` to 1.0.0
     - `/vsdd-factory:activate` — new required step
     - Full Claude Code session restart
     - `/vsdd-factory:factory-health` — verify
   - **Verification checklist**:
     - Hooks fire (make a test commit, see `commit.made` event)
     - Existing dashboards still work (file sink preserved)
     - Dispatcher-internal log exists
   - **Observability migration**:
     - Default config is file-sink-only, matches 0.79.x behavior
     - To add Datadog/Honeycomb: write `observability-config.toml`
     - Zero-disk mode: requires disabling file sink
   - **Windows-specific notes**:
     - What works in 1.0: 4 native WASM hooks (capture-commit-activity,
       capture-pr-activity, block-ai-attribution, emit_event)
     - What doesn't yet: 26 hooks still on legacy-bash-adapter
       (requires git-bash on Windows)
     - Recommended: install git-bash for now; expect more native ports
       post-1.0
   - **Known regressions**: empty if S-2.7 found none
   - **Rollback**: how to pin back to 0.79.4 if v1.0 breaks
   - **Troubleshooting**: common symptoms and fixes
3. Walk an external user through it. Gather feedback. Revise.
4. Link from CHANGELOG + release notes.

## Contracts

**External validation:**

The guide must be followable end-to-end by a user who has never
touched the factory internals. Success = they complete the migration
without asking questions not answered by the doc.

## Test plan

- One external dogfood run
- Rollback test: install 1.0.0, find a simulated issue, follow the
  rollback section, verify 0.79.4 functional again

## Notes

This is the docs-heavy story in Phase 5. Allocate time for iteration —
first-draft migration guides are always incomplete until a real user
tries them.

Rollback is a real concern. Be specific: `git checkout 0.79.4 tag`
in the marketplace cache, `/vsdd-factory:deactivate && re-activate
from the old version's hooks.json`, restart session.

Windows expectations in the doc should be frank — most hooks remain
bash-adapter-routed and thus Linux/macOS only until post-1.0 ports
happen. Over-promising Windows support would disappoint operators.
