# S-1.7: dispatcher-internal.jsonl writer

**Epic:** [v1.0 Factory Plugin Kit](EPIC.md)
**Phase:** Phase 1 — Dispatcher foundation
**Milestone:** `1.0.0-beta.1`
**Tier:** B.x
**Blocks:** S-1.8, S-2.7, S-4.7
**Blocked by:** S-1.1, S-1.2
**Estimated effort:** S

## Goal

Implement the always-on internal self-telemetry log at
`.factory/logs/dispatcher-internal.jsonl`. Every lifecycle event and
`internal.*` event written by the dispatcher itself goes here in addition
to the configured sinks. Per Q6 Option B, this file exists independent
of `observability-config.toml` so the dispatcher stays debuggable even
when all sinks are misconfigured or down.

## Acceptance criteria

- [ ] Dispatcher writes `dispatcher.started` event on launch with
  `dispatcher_version`, `host_abi_version`, `platform`, `pid`,
  `dispatcher_trace_id`, `registry_path`, `loaded_plugin_count`
- [ ] `plugin.loaded` event written for each plugin in the registry after
  successful load
- [ ] `plugin.load_failed` written for any plugin that fails to load (with
  reason)
- [ ] `plugin.invoked` and `plugin.completed` events written around every
  plugin invocation
- [ ] `plugin.timeout`, `plugin.crashed` on abnormal exits
- [ ] `internal.capability_denied` on host function capability rejections
- [ ] `internal.host_function_panic` on host function panics
- [ ] `internal.sink_error`, `internal.sink_queue_full`,
  `internal.sink_circuit_opened`, `internal.sink_circuit_closed` — all
  the `internal.*` events also written here (in addition to sink routing)
- [ ] Append-only JSONL; one event per line
- [ ] Log rotation: file per day (`dispatcher-internal-YYYY-MM-DD.jsonl`)
- [ ] Retention: delete files older than 30 days on dispatcher startup
- [ ] Writes are atomic (PIPE_BUF semantics, like `emit-event` today)
- [ ] Works even if `.factory/logs/` doesn't exist yet (mkdir -p)
- [ ] Silent failure if disk full / read-only / permissions — dispatcher
  must never block on this log
- [ ] Unit + integration tests

## Tasks

1. Create `crates/factory-dispatcher/src/internal_log.rs`.
2. Locate the log dir via the same resolution logic as `emit-event`
   (v0.70+): main worktree's `.factory/logs` if inside a git repo, else
   cwd. For dispatcher purposes, prefer the registry-relative path.
3. Implement `InternalLog` struct with `write(event: &Event) -> ()` —
   always-infallible (errors are swallowed with a `log::warn!`).
4. Daily rotation: determine filename from current date at write time; no
   rollover logic needed beyond that.
5. Retention: on startup, `readdir` the log dir, delete files matching
   `dispatcher-internal-*.jsonl` older than 30 days.
6. Wire into: dispatcher main (for `dispatcher.started`), plugin_loader
   (for `plugin.loaded`/`load_failed`), executor (for `plugin.invoked`/
   `completed`/`timeout`/`crashed`), host functions (for
   `capability_denied`/`host_function_panic`), sink pipeline (once S-1.8
   exists, for `sink_*` events).
7. Tests: write 10K events, verify no data loss; simulate disk-full via
   mocked writer, verify dispatcher continues; verify rotation by mocking
   the clock.

## Contracts

**Log file path:** `.factory/logs/dispatcher-internal-YYYY-MM-DD.jsonl`

**Line format (one JSON object per line):**

```json
{
  "type": "plugin.invoked",
  "dispatcher_trace_id": "uuid",
  "session_id": "claude-code-session-id",
  "plugin_name": "capture-commit-activity",
  "plugin_version": "0.1.0",
  "ts": "2026-04-24T20:13:45-0500",
  "ts_epoch": 1777003425,
  "schema_version": 1,
  "...event-specific fields..."
}
```

**Event type catalog (v1.0):**

- `dispatcher.started`, `dispatcher.shutting_down`
- `plugin.loaded`, `plugin.load_failed`
- `plugin.invoked`, `plugin.completed`, `plugin.timeout`, `plugin.crashed`
- `internal.capability_denied`, `internal.host_function_panic`
- `internal.sink_error`, `internal.sink_queue_full`,
  `internal.sink_circuit_opened`, `internal.sink_circuit_closed`
- `internal.dispatcher_error` (any uncategorized dispatcher failure)

Documented in `docs/guide/observability-sinks.md` (from S-0.5).

**Always-on invariant:** the internal log writes happen unconditionally,
regardless of `observability-config.toml`. The only kill switch is:
disk failure / permission error, in which case errors are silently swallowed
and tracing::warn! is called.

## Test plan

- Unit: write events, verify file contents parse as JSONL.
- Unit: simulate disk-full by passing a non-writable path; verify no
  panic, no dispatcher exit.
- Unit: retention test — seed files with ages 1, 10, 30, 31, 60 days;
  verify only 31 and 60-day files are removed on startup.
- Integration: full dispatcher invocation with a happy-path plugin,
  verify `dispatcher.started` + `plugin.loaded` + `plugin.invoked` +
  `plugin.completed` all appear in the internal log.

## Notes

This log is for debugging the dispatcher, not for user-facing observability.
It should not contain sensitive payload data — specifically, `tool_input`
should NOT be recorded verbatim in this log (it could contain user
credentials, file paths revealing org structure, etc.). Truncate /
redact consistently.

`emit-event` (the bash bin from 0.79.x) is separate. This internal log
is a new path. They coexist during migration; after full migration the
old `emit-event` is retired.

Rotation is time-based rather than size-based for simplicity. Typical
factory runs produce < 1MB of internal telemetry per day.
