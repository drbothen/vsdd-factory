# S-5.2: SessionEnd hook wiring

**Epic:** [v1.0 Factory Plugin Kit](EPIC.md)
**Phase:** Phase 5 — New Claude Code hook events
**Milestone:** `1.0.0`
**Tier:** G
**Blocks:** S-5.7
**Blocked by:** S-4.8
**Estimated effort:** S

## Goal

Wire `SessionEnd` through the dispatcher. Emits `session.ended` with
duration, tool-call count, and cost. Pairs with `SessionStart` (S-5.1)
to give operators session-level telemetry.

## Acceptance criteria

- [ ] Dispatcher accepts `SessionEnd` events
- [ ] `session-end-telemetry` WASM plugin emits `session.ended` with:
  - `session_id`
  - `duration_seconds` (from session.started timestamp to now)
  - `tool_call_count` (read from file sink for this session)
  - `estimated_cost_usd` (if Claude's OTel metrics are being scraped
    and available via a local Prometheus query; else omitted)
  - `exit_reason` (from Claude Code's SessionEnd payload)
- [ ] Registered in `hooks-registry.toml` with `once = true`
- [ ] Works on all 5 platforms
- [ ] Integration test

## Tasks

1. Create `crates/hook-plugins/session-end-telemetry/`.
2. Implement:
   - Query file sink for session.started with matching session_id to
     compute duration
   - Count tool-call events (pre/post) in the file sink for this session
   - Optional cost lookup (skip if Prometheus not reachable)
3. Update registry + hooks.json.template.
4. Tests.

## Artifacts

- `crates/hook-plugins/session-end-telemetry/`
- `hooks-registry.toml`

## Contracts

**`session.ended` event:**

```json
{
  "type": "session.ended",
  "session_id": "...",
  "duration_seconds": 1842,
  "tool_call_count": 127,
  "estimated_cost_usd": 0.47,
  "exit_reason": "user_quit"
}
```

## Test plan

- Integration: synthesize session.started + tool events + SessionEnd;
  verify emitted session.ended has correct fields

## Notes

Cost lookup is best-effort. If Prometheus isn't running or doesn't have
current-session data, emit without the cost field. Factory ROI dashboard
already gets cost from OTel metrics — this event is duplicative for
durable storage.

Duration + tool_call_count require reading the file sink, which needs
`read_file` capability scoped to `.factory/logs/`.
