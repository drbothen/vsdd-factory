# S-1.8: sink-file driver

**Epic:** [v1.0 Factory Plugin Kit](EPIC.md)
**Phase:** Phase 1 — Dispatcher foundation (feeds into Phase 4 sink system)
**Milestone:** `1.0.0-beta.1`
**Tier:** B.x
**Blocks:** S-2.7, S-4.7
**Blocked by:** S-1.1, S-1.7
**Estimated effort:** M

## Goal

Implement the `file` sink type — JSONL append to a configurable path.
This is the minimum-viable sink required for beta.1 and preserves the
current `.factory/logs/events-*.jsonl` behavior that every existing
consumer (OTel collector, Factory ROI dashboard, factory-query) depends on.
Also defines the sink-driver trait that S-4.x drivers will implement.

## Acceptance criteria

- [ ] `Sink` trait defined — the interface every sink driver implements
- [ ] `FileSink` implements `Sink`: takes a path template, writes JSONL
  appends
- [ ] Path templating: `{date}` → `YYYY-MM-DD`, `{name}` → sink name,
  `{project}` → `CLAUDE_PROJECT_DIR` basename
- [ ] Events flow through the dispatcher's mpsc channel (S-1.4) → sink
  router → enabled FileSinks
- [ ] Writes are atomic (PIPE_BUF atomic-append)
- [ ] Parent directory auto-created
- [ ] Per-sink routing filter applied (if configured)
- [ ] Per-sink tag enrichment applied (if configured)
- [ ] Retry / circuit-breaker / queue config fields accepted but not
  enforced (S-4.4 adds the resilience runtime; v1.0-beta.1 ships with
  basic behavior)
- [ ] Dead-letter behavior not implemented yet (S-4.5)
- [ ] Emits `internal.sink_error` on write failure (disk full, permission
  denied, etc.)
- [ ] Respects `enabled = false` to skip the sink entirely
- [ ] Integration test: multiple events through a FileSink, verify file
  contents match expected JSONL

## Tasks

1. Create `crates/sink-file/` crate (add to workspace).
2. Define `Sink` trait in a new `crates/sink-core/` crate (shared types):
   ```rust
   pub trait Sink: Send + Sync {
       fn name(&self) -> &str;
       fn config(&self) -> &SinkConfig;
       fn accepts(&self, event: &Event) -> bool;  // routing filter
       fn submit(&self, event: Event);             // non-blocking enqueue
       fn flush(&self) -> Result<()>;              // called at tier boundaries
       fn shutdown(&self);                         // graceful
   }
   ```
3. Implement `FileSink`:
   - Internal async writer loop (tokio task) reading from mpsc
   - File opened once per (sink, date); rotate on date change
   - Atomic JSONL append (newline-terminated)
4. Create `crates/factory-dispatcher/src/sinks/mod.rs` — loads sinks from
   `observability-config.toml` via the `SinkRegistry`.
5. Wire into dispatcher main: construct the SinkRegistry at startup;
   connect the `HostContext` event channel to the SinkRegistry's router.
6. Apply routing + tags per Q6 contract.
7. Integration tests.

## Contracts

**`Sink` trait** (consumed by S-1.9, S-4.1–S-4.6):

Listed above. Implementations must be `Send + Sync` (tokio tasks may run
on different threads). `submit()` must be non-blocking.

**Path templating:**

```
{date}     → YYYY-MM-DD (date of event emission)
{name}     → sink's configured name
{project}  → basename of CLAUDE_PROJECT_DIR
```

**Event serialization contract:**

```json
{
  "type": "commit.made",
  "dispatcher_trace_id": "uuid",
  "session_id": "sid",
  "plugin_name": "capture-commit-activity",
  "plugin_version": "0.1.0",
  "ts": "2026-04-24T20:13:45-0500",
  "ts_epoch": 1777003425,
  "schema_version": 1,
  "...event-specific fields..."
}
```

Matches the current `.factory/logs/events-*.jsonl` format so existing
consumers (OTel filelog, factory-query, factory-report) require no
changes.

**Backpressure (v1.0-beta.1 behavior; full implementation in S-4.4):**

Bounded mpsc channel with default depth 1000. On full: drop oldest,
emit `internal.sink_queue_full` (once per second throttled to avoid
emit storm).

## Notes

File sinks are fast; the bounded queue is unlikely to fill under normal
load. But for crash-protection, size the queue generously (1000 events
≈ ~1MB memory at 1KB/event).

Date-based file rotation happens lazily — check event's timestamp,
compare to currently-open file, reopen if needed. Rotation at midnight
local time is acceptable.

The existing bash `emit-event` will continue to work alongside this
during migration; both write to `.factory/logs/events-*.jsonl`. When the
last bash hook is ported (post-1.0 backlog), `emit-event` is retired.
