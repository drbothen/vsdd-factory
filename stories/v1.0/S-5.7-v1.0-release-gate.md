## Goal

Gate story for the final 1.0.0 tag. Semver stability engages here —
everything committed as stable in S-5.6 becomes binding. First
production-ready release of the Factory Plugin Kit.

## Acceptance criteria

  SessionEnd, WorktreeCreate, WorktreeRemove, PostToolUseFailure)

  that isn't vsdd-factory itself

  frozen

## Contracts

**Semver commitment engages:** as of tagging `1.0.0`, all schemas and
APIs listed in S-5.6 are stable. Breaking them requires 2.0.

**Marketplace positioning:**

- Stable label moves from 0.79.4 → 1.0.0
- `/plugin install vsdd-factory@vsdd-factory` now installs 1.0.0
- 0.79.4 remains available as a specific-version pin for users not ready

**Support commitment:**

- 1.0.x maintained with bug fixes until 1.1 ships (or indefinitely if
  no 1.1 is scheduled)
- Breaking behaviors discovered post-1.0 handled per S-5.6's deprecation
  process

## Test plan

- Walk each gate with evidence
- "No rebuild" live demo
- Marketplace install of 1.0.0 on a fresh environment, verify
  end-to-end
- Post-release smoke: make a commit, verify `commit.made` event, check
  Factory ROI dashboard shows it

## Notes

Release notes should narrate what this is and why it matters, not just
list changes. This is the first marketing moment for the v1.0 work;
write it to be read.

Retrospective is valuable — capture what was hard, what the team learned,
what we'd do differently. Feeds into 2.0 planning (WASI preview 2,
more native ports, etc.).

After 1.0 ships, the epic closes. Post-1.0 work (more bash-to-WASM
ports, new sink types, additional CC hook event coverage) is tracked
in a new epic.
