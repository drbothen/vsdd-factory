# S-12.02: validate-per-story-adversary-convergence WASM Hook

**Epic:** E-12 â€” Engine Governance â€” Per-Story Adversarial Convergence Discipline
**Delivery order:** THIRD in cycle (after S-13.01 and S-12.01). Implements mechanical
enforcement of the workflow gate defined in S-12.01.
**Depends on:** S-12.01 (needs per-story state file schema as input contract), S-13.01
(path discipline must be live)
**Blocks:** (none â€” final story in cycle delivery order)
**Priority:** P1

## Narrative

- **As a** vsdd-factory wave-gate dispatcher
- **I want** a WASM hook that blocks wave-gate dispatch when any story in the wave
  lacks a cleared adversary convergence state file (`passes_clean >= 3 AND
  last_classification == "NITPICK_ONLY"`)
- **So that** no wave can close and no PR merge proceeds when stories have skipped
  the per-story adversary gate, making enforcement mechanical rather than behavioral

## Acceptance Criteria

### AC-001 (traces to BC-4.10.001 postcondition 1 â€” state file reading)

Hook reads `.factory/cycles/<cycle-id>/<story-id>/adversary-convergence-state.json`
for each story in the current wave. The JSON schema is exactly as specified in
BC-5.39.001 postcondition 2 (fields: `passes_clean`, `last_finding_count`,
`last_classification`, `last_timestamp`, `deferred_findings`). The hook uses the
injectable-callback pattern: `read_file` is a closure injected into `fn hook_logic(...)`.

**Falsifiable test:** Rust unit test: inject a read_file callback returning a valid JSON
fixture with `passes_clean: 3`, `last_classification: "NITPICK_ONLY"`. Assert
`HookResult::Continue` is returned.

### AC-002 (traces to BC-4.10.001 postcondition 2/3/4/8 + VP-071 kani â€” block invariant)

Hook returns `HookResult::block_with_fix(hook, reason, recommendation, code)` (canonical
block form per BC-4.10.001 PC8 and OQ-9/D-349) when ANY story in the wave fails any of
the three conditions. The block is immediate (exit code 2); the hook does NOT return
`HookResult::Continue` on block. Block codes:

- Missing state file â†’ `code: "CONVERGENCE_STATE_MISSING"`
- `passes_clean < 3` â†’ `code: "CONVERGENCE_PASSES_INSUFFICIENT"`
- `last_classification != "NITPICK_ONLY"` â†’ `code: "CONVERGENCE_CLASSIFICATION_INSUFFICIENT"`

VP-071 kani harness (all 6 proof functions) passes without counterexamples:
`proof_missing_state_file_always_blocks`, `proof_insufficient_passes_always_blocks`,
`proof_non_nitpick_classification_always_blocks`, `proof_null_classification_blocks`,
`proof_converged_story_produces_continue`, `proof_hook_result_is_block_on_failure`.

**Falsifiable test:** Three Rust unit tests (one per block code), each asserting the
correct `code` value and `HookResult::Block { reason }` return. Plus: `cargo kani -p
validate-per-story-adversary-convergence` completes without counterexamples.

### AC-003 (traces to BC-4.10.002 postcondition 1â€“5 â€” graceful degrade outside wave-gate)

When the hook cannot confirm wave-gate context (cycle-id not determinable, payload
missing `subagent_name`/`agent_type`, or agent type is not wave-gate dispatch), the hook
returns `HookResult::Continue` immediately. A single advisory message is logged via
`host::log_info` (HOST_ABI v1 has no `log_debug`; per BC-4.10.002 PC3 v1.1). No block
signal emitted. No `hook.warn` or `hook.block` events.

**Falsifiable test:** Rust unit test: inject payload without wave-gate indicator (e.g.,
agent_type missing or wrong). Assert `HookResult::Continue` returned. Assert read_file
callback NOT invoked (graceful degrade exits before file I/O per BC-4.10.002 invariant 2).

### AC-004 (traces to BC-4.10.002 invariant 3 â€” absent cycle directory graceful degrade)

When `.factory/cycles/` directory does not exist (read_file returns `Err` for the cycle
path), hook returns `HookResult::Continue`. This follows the same pattern as
`validate-wave-gate-prerequisite.sh` lines 64â€“70 and `regression-gate` BC-7.03.074.

**Falsifiable test:** Rust unit test: read_file callback returns `Err` for the
`.factory/cycles/<cycle-id>/` path. Assert `HookResult::Continue` returned and no
block event emitted. Assert execution time in this path < 50ms (BC-4.10.002 postcondition 6).

### AC-005 (traces to BC-4.10.001 postcondition 6 â€” deferred_findings aggregation)

When all stories pass convergence (`passes_clean >= 3 AND last_classification ==
"NITPICK_ONLY"`), the hook aggregates the `deferred_findings` arrays from all stories
and emits them to the wave-gate context as a structured log entry (not a block).
The hook returns `HookResult::Continue`.

**Falsifiable test:** Rust unit test: wave with 2 stories, both converged, one with
non-empty `deferred_findings`. Assert `HookResult::Continue`. Assert log entry emitted
containing deferred findings content (via injected log callback).

### AC-006 (traces to BC-4.10.001 postcondition 5 â€” first-failure block)

When multiple stories fail convergence, hook blocks on the FIRST non-cleared story
found. Emits a single block message identifying the first failure. Does not enumerate
all failures in one block message.

**Falsifiable test:** Rust unit test: wave with 2 stories, both uncleaned. Assert only
one block message emitted (for the first story processed).

### AC-007 (traces to BC-4.10.001 edge case EC-002 â€” malformed JSON state file)

Malformed JSON in the state file is treated as non-cleared. Hook emits block with
`code: "CONVERGENCE_STATE_MALFORMED"`. Does not panic or return an unhandled error.

**Falsifiable test:** Rust unit test: read_file callback returns malformed JSON string
(not valid JSON). Assert `HookResult::Block { reason }` returned with reason containing
`code: "CONVERGENCE_STATE_MALFORMED"`.

### AC-008 (traces to BC-4.10.001 edge case EC-003 â€” missing schema field)

State file exists and is valid JSON, but `last_classification` field is absent (null or
missing). Treated as non-cleared. Hook emits block with `code:

**Falsifiable test:** Rust unit test: read_file returns JSON with `passes_clean: 3`

## Architecture Compliance Rules

Extracted from BC-4.10.001, BC-4.10.002, and ADR-017:

1. **No bare `HookResult::block()`:** All block-signaling uses the canonical
   `block_with_fix` form per BC-4.10.001 PC8 (OQ-9/D-349): call
   `HookResult::block_with_fix(hook, reason, recommendation, code)`. This produces
   `HookResult::Block { reason }` (exit code 2, immediate block). The deprecated
   advisory-block-mode pattern (returning `HookResult::Continue` with stdout JSON)
   is NOT used for this hook â€” it is reserved only for the three legacy v1.0 plugins
   named in HOST_ABI.md. Study `BC-4.10.001 PC8` and `VP-071 v1.2` for the correct form.
2. **Graceful degrade BEFORE file reads:** The graceful degrade check (BC-4.10.002
   invariant 2) must occur at the start of `hook_logic`, before any `read_file` calls.
   If context cannot be confirmed â†’ return `HookResult::Continue` immediately.
3. **`HOST_ABI_VERSION = 1`:** No new host functions. Strictly `host::read_file`,
   `host::log_*`, `host::emit_event`.
4. **Read-only:** The hook MUST NOT call `host::write_file` or any file-mutation host
   function. It is a read-only observer of state files.
5. **Event scope in hooks-registry.toml:** The `SubagentStop` event filter MUST be
   scoped to wave-gate dispatch only. Per BC-4.10.001 invariant 1, this hook MUST NOT
   fire on per-story `SubagentStop` events (those trigger BC-5.39.001's convergence loop,
   a different mechanism).
6. **Forbidden dependencies â€” `validate-per-story-adversary-convergence` crate:** This
   crate MUST NOT depend on `validate-artifact-path`, `factory-dispatcher`, or any crate
   beyond `vsdd-hook-sdk` and `serde_json`. If this crate gains a dependency on
   `factory-dispatcher`, the build MUST fail.

## Dev Notes

OQ-E12-01 (state file format decision may affect scope filter options).

**State file schema compatibility:** If S-12.01 implementation reveals the schema
differs from BC-5.39.001 postcondition 2 (e.g., field name change), DO NOT implement
against the wrong schema. Raise a BC amendment and wait for resolution before
