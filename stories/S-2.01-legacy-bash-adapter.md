# S-2.01: legacy-bash-adapter WASM plugin

**Epic:** E-2 — Legacy Adapter and Beta Release
**Phase:** Phase 2 — Legacy bash adapter
**Milestone:** `1.0.0-beta.1`
**Tier:** C
**Blocks:** S-2.02, S-2.07, S-2.08
**Blocked by:** S-1.03, S-1.04, S-1.05, S-1.06
**Estimated effort:** M

## Narrative
- **As a** factory operator with existing bash hooks
- **I want to** have all 30+ legacy bash scripts run unchanged through the new WASM dispatcher
- **So that** I can adopt v1.0 without rewriting any existing hooks

## Goal

Implement the `legacy-bash-adapter` WASM plugin. It takes a bash script
path via a per-hook configuration parameter, invokes it via
`exec_subprocess()` with the existing stdin payload, propagates stdout/
stderr + exit code back to the dispatcher, preserving full backward compat.

## Acceptance criteria

- [ ] `legacy-bash-adapter.wasm` compiles from `crates/hook-plugins/legacy-bash-adapter/`
- [ ] Plugin reads `script_path` from per-hook config (passed via plugin config in registry entry)
- [ ] Invokes the bash script via `exec_subprocess()` with stdin from original payload
- [ ] Passes stdout to `emit_event` for `hook.bash_output`
- [ ] Propagates exit code: exit 2 from bash → `HookResult::Block`, exit 0 → `HookResult::Continue`
- [ ] stderr from bash script emitted as `hook.bash_stderr` events
- [ ] 14 unit tests covering success, block, error, timeout paths
- [ ] All 45 existing `hooks-registry.toml` entries routed through this adapter

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | `script_path` not in config | Return `HookResult::Error("script_path missing in config")` |
| EC-002 | Script exits with code 1 | Treated as Continue (only exit 2 is block) |
| EC-003 | Script times out | `exec_subprocess` timeout → `HookResult::Error` |
| EC-004 | Script produces large stderr | Truncated per `max_output_bytes` |
