## Capability Anchor Justification

Primary: CAP-008 ("Gate tool calls with pre-execution behavioral checks (PreToolUse hooks)") per capabilities.md §CAP-008. The legacy-bash-adapter is the universal bridge that routes all 30+ existing bash hooks (many of which are PreToolUse gates) through the WASM dispatcher, directly enabling the behavioral gating capability.

FR anchor: FR-013 ("Legacy bash adapter plugin") per PRD §2.4. FR-013 owns all 12 SS-04 BCs for the adapter.

## Narrative
- **As a** factory operator with existing bash hooks
- **I want to** have all 30+ legacy bash scripts run unchanged through the new WASM dispatcher
- **So that** I can adopt v1.0 without rewriting any existing hooks

## Goal

Implement the `legacy-bash-adapter` WASM plugin. It takes a bash script
path via a per-hook configuration parameter, invokes it via
`exec_subprocess()` with the existing stdin payload, propagates stdout/
stderr + exit code back to the dispatcher, preserving full backward compat.
