## Capability Anchor Justification

**Primary anchor:** CAP-022 ("Port hook plugins from bash to native WASM") per
`domain-spec/capabilities.md` §CAP-022. S-8.04 ports `update-wave-state-on-merge.sh`
to native WASM, eliminating legacy-bash-adapter indirection and the python3 runtime
dependency for this state-mutation hook.

**Stretch-Anchor Disclosure:** S-8.04 anchors to CAP-022 but spans two subsystems:
- **SS-07 Hook Bash Layer** owns `update-wave-state-on-merge.sh` and its BC-7.03.083–086
  behavioral contracts. SS-07 is the source-of-truth for this hook's behavior.
- **SS-01 Hook Dispatcher Core** owns the hooks-registry.toml routing and the
  host fn surface. Notably, the WASM port replaces both the `bash` AND `python3`
  subprocess dependencies (the bash script delegates YAML mutation to an inline python3
  snippet). The native Rust crate handles YAML parsing directly.

Both subsystem anchors verified against ARCH-INDEX Subsystem Registry:
- SS-01 Hook Dispatcher Core: canonical name confirmed.
- SS-07 Hook Bash Layer: canonical name confirmed.
(Per POLICY 5 anchor justification requirement + POLICY 6 architecture-is-source-of-truth.)

**Complexity note (Medium — 4 pts):** This hook is more complex than handoff-validator
or track-agent-stop because it: (a) scopes to pr-manager agents only, (b) detects merge
success signals via regex, (c) extracts a story ID from the result text, (d) reads and
writes `.factory/wave-state.yaml` via YAML parse/serialize, (e) appends the story ID
to `stories_merged`, (f) conditionally flips `gate_status` to `pending` and sets
`next_gate_required`. The python3 YAML logic is non-trivial to translate — care is
required to preserve the key-ordering and flow-style semantics (`sort_keys=False`,
`default_flow_style=False`, `allow_unicode=True`).

## Narrative

- **As a** vsdd-factory operator managing wave-based story delivery
- **I want** update-wave-state-on-merge to execute as a native WASM plugin
- **So that** wave-state.yaml is automatically updated when pr-manager completes a
  merge, on all platforms, without a python3 runtime dependency

## Goal

Port `plugins/vsdd-factory/hooks/update-wave-state-on-merge.sh` to a native Rust WASM
crate at `crates/hook-plugins/update-wave-state-on-merge/`. The hook fires on
SubagentStop for pr-manager agents only, and only when a merge completion signal is
detected in the result text. When triggered, it:

1. Extracts the story ID from the result (pattern `S-[0-9]+\.[0-9]+` or `STORY-[0-9]+`).
2. Reads `.factory/wave-state.yaml` (if present).
3. Finds the wave containing the story ID in its `stories` list.
4. Appends the story ID to `stories_merged` (if not already present).
5. Writes the updated YAML back to disk.
6. If `stories == stories_merged` (all stories merged), sets `gate_status = "pending"`
   and `next_gate_required = wave_name`; writes a stderr reminder.
7. Emits `host::emit_event` with `type=hook.action hook=update-wave-state-on-merge
   matcher=SubagentStop reason=wave_merge_recorded` plus story_id, wave, total, merged,
   gate_transitioned fields.
8. Always exits 0.

The `on_error = "continue"` registry setting means dispatcher errors do not block;
preserved in the migrated entry. The `timeout_ms = 10000` (longer than other SubagentStop
hooks) reflects the YAML read/write time; preserved.

### AC-004 (traces to BC-7.03.085 postcondition 1)
When agent is pr-manager AND merge signal detected: WASM plugin extracts story_id

with equivalent semantics to python3 `yaml.dump(..., default_flow_style=False,
sort_keys=False, allow_unicode=True)`. Emits `host::emit_event` with all required
fields (type=hook.action, reason=wave_merge_recorded, story_id, wave, total, merged,
gate_transitioned). Exits 0.

### AC-005 (traces to BC-7.03.086 postcondition 1)

WASM plugin sets `wave_data.gate_status = "pending"` and
`state.next_gate_required = wave_name`. Writes stderr message:
`"update-wave-state-on-merge: all stories in <wave> merged. gate_status → pending.\n  Run the wave integration gate before starting the next wave.\n"`.
Emits event with `gate_transitioned=true`. Exits 0.

### AC-006 (traces to BC-7.03.084 postcondition 1 + BC-7.03.085 postcondition 1)

(d) wave-state.yaml absent → graceful exit 0, no YAML write; (e) story_id not found
in any wave → exit 0, no mutation. All bats tests pass.

### AC-007 (traces to BC-7.03.083 postcondition 1)
`bin/emit-event` calls replaced with `host::emit_event`. No reference to `bin/emit-event`
in `crates/hook-plugins/update-wave-state-on-merge/`. `bin/emit-event` binary NOT
