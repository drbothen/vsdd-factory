**Epic:** E-3 — WASM Port: High-Value Hooks
**Phase:** Phase 3 — High-value port to native WASM
**Milestone:** `1.0.0-rc.1`
**Tier:** E

## Capability Anchor Justification

Primary: CAP-008 ("Gate tool calls with pre-execution behavioral checks (PreToolUse hooks)") per capabilities.md §CAP-008. The block-ai-attribution plugin is a PreToolUse gate — it blocks `git commit` commands containing AI attribution patterns before execution, directly implementing the behavioral gating capability.

FR anchor: FR-013 ("Legacy bash adapter plugin") per PRD §2.4. FR-013 covers the bash-to-WASM plugin bridging that this story extends. The blocking behavior is also covered by FR-032 ("PreToolUse gate hooks") at the SS-07 bash layer; however, since this story produces a native WASM plugin in SS-04, FR-013 is the closer anchor. Related contracts from the SS-01 executor (BC-1.03.NNN plugin execution lifecycle) are informational.

Note: architecture compliance rules reference BC-1.03 (plugin execution lifecycle) and BC-1.01 (registry schema) — these are cross-subsystem informational references, not SS-04 anchors.

## Narrative
- **As a** repository maintainer
- **I want to** have `block-ai-attribution` running as native WASM in the PreToolUse/Bash hook
- **So that** commits containing AI attribution strings are blocked before they reach git

## Goal

Rewrite `block-ai-attribution` from bash to native WASM. This is a
PreToolUse/Bash gate — it must return `HookResult::Block` to prevent
the Bash tool from running if the commit message contains AI attribution patterns.

## Acceptance criteria

- [ ] `block-ai-attribution.wasm` compiled and registered as PreToolUse hook
  - _trace: BC-4.02.003 precondition 1 (registry entry routes through plugin)_ [process-gap: no SS-04 BC contracts block-ai-attribution as a distinct plugin — v1.1 BC candidate]

  - _trace: CAP-008 ("blocks dangerous commands before execution: … AI attribution injection")_ [process-gap: no SS-04 BC contracts the attribution pattern detection logic — v1.1 BC candidate]

  - _trace: BC-4.02.002 postcondition 2 (exit 2 → Block with synthetic reason from stderr) — the WASM plugin returns exit 2 which the adapter maps to Block_

  - _trace: BC-4.02.002 (exit code mapping contract unchanged; bats tests verify exit codes)_
