# S-3.03: Port block-ai-attribution to WASM

**Epic:** E-3 — WASM Port: High-Value Hooks
**Phase:** Phase 3 — High-value port to native WASM
**Milestone:** `1.0.0-rc.1`
**Tier:** E
**Blocks:** S-4.07, S-4.08
**Blocked by:** S-2.08, S-3.04
**Estimated effort:** S

## Narrative
- **As a** repository maintainer
- **I want to** have `block-ai-attribution` running as native WASM in the PreToolUse/Bash hook
- **So that** commits containing AI attribution strings are blocked before they reach git

## Goal

Rewrite `block-ai-attribution` from bash to native WASM. This is a
PreToolUse/Bash gate — it must return `HookResult::Block` to prevent
the Bash tool from running if the commit message contains AI attribution patterns.

## Acceptance criteria

- [ ] `block-ai-attribution.wasm` compiled and registered as PreToolUse hook
- [ ] Scans `tool_input.command` for `git commit` with AI attribution patterns (Co-Authored-By, Generated with Claude, etc.)
- [ ] Returns `HookResult::Block(reason)` if patterns found
- [ ] Returns `HookResult::Continue` for clean commits or non-git commands
- [ ] Block message is human-readable and includes the matched pattern
- [ ] `hooks-registry.toml` updated; `on_error = "block"` set for this entry
- [ ] Existing bats tests for block-ai-attribution still pass

| Component | Module | Pure/Effectful |

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | Non-git command | Return Continue |
| EC-002 | `git commit --allow-empty` with attribution | Block |
| EC-003 | Attribution in commit body (not subject) | Block |
