## Narrative

- **As a** vsdd-factory operator running test suites during implementation sessions
- **I want** regression-gate to execute as a native WASM plugin instead of through the legacy-bash-adapter
- **So that** test pass/fail state tracking and regression warnings run on all platforms
  (including Windows without jq), with lower latency and no adapter indirection;
  and so that all 9 Tier 1 hooks are confirmed native, closing the W-15 wave

## Goal

**Deliverable 1 — WASM port:** Port `plugins/vsdd-factory/hooks/regression-gate.sh` to
a native Rust WASM crate at `crates/hook-plugins/regression-gate/`. The hook fires on
PostToolUse (no tool filter in registry; body filters to Bash commands). It reads stdin
JSON (PostToolUse envelope), checks if the Bash command matches one of 9 test-runner
patterns, derives pass/fail status from exit_code/interrupted fields, reads prior status
from `.factory/regression-state.json`, writes new status, and emits a warning on
pass-to-fail transition.

**Deliverable 2 — OQ-6 audit:** Security-reviewer audits regression-gate.sh to produce
a confirmed capability profile (`binary_allow` and file access declarations) for the
WASM crate. WASM implementation of regression-gate MUST NOT begin until audit is complete
(AC-010 gate). The audit result is recorded in
`.factory/cycles/v1.0-brownfield-backfill/E-8-oq6-capability-profile.md`.

**Deliverable 3 — Tier 1 pre-retirement audit:** Scan `hooks-registry.toml` and confirm
all 9 Tier 1 hook entries now reference native `.wasm` plugins (not
`legacy-bash-adapter.wasm`). Record result in
`.factory/cycles/v1.0-brownfield-backfill/E-8-tier1-native-audit.md`.
Adapter crate and `bin/emit-event` are NOT deleted in this story (deferred to S-8.28).

## OQ-6 Capability Anchor Justification

**OQ-6 (from E-8 epic):** "regression-gate.sh appears to invoke external test runners
(bats, cargo test). The WASM sandbox restricts subprocess calls to `binary_allow` list.
If the native port needs to exec arbitrary test runners, this may require a more
permissive capability declaration than current hooks use."

**Resolution (from bash source analysis):** regression-gate does NOT invoke test runners.
It is a PostToolUse OBSERVER hook — it receives the output of already-completed Bash
tool calls via the stdin JSON envelope. The hook's subprocess usage is limited to:
- `jq` for stdin JSON parsing (same as other Tier 1 hooks)
- `jq -n ... > file` for writing the state JSON file
- `date -u` for UTC timestamp generation
- `bin/emit-event` for telemetry (replaced by host::emit_event in WASM port)

Preliminary capability profile: `binary_allow = ["jq"]` (or empty if the WASM port
uses serde_json + chrono instead, which is preferred). Security-reviewer must confirm.

**OQ-6 validation method (per `assumption_validations: [OQ-6]`):** AC-010 requires
a security-reviewer to produce a signed-off capability profile document
before S-8.09 implementation begins. The bats test AC-007(e) then validates the
capability profile is correctly declared in the final registry entry.

### AC-002 (traces to BC-7.03.071 invariant 1)
`hooks.json` command entry for `regression-gate` is deleted from all platform-specific
`hooks.json.*` files and the template.
`plugins/vsdd-factory/hooks/regression-gate.sh` is deleted from the repository.

### AC-003 (traces to BC-7.03.072 postcondition 1)
WASM plugin checks `tool_name` field from stdin; if not `"Bash"`, exits 0 immediately.
For Bash tool calls, extracts the `tool_input.command` string and matches against the
9 test-runner patterns:
`cargo test`, `cargo nextest`, `pytest`, `pnpm test`, `npm test`, `go test`,
`just test`, `just ci`, `yarn test`.
If no pattern matches, exits 0 immediately (no state file write, no event).
Pattern matching uses substring contains (same as bash `case` glob `*"cargo test"*`).

### AC-004 (traces to BC-7.03.073 postcondition 1)
Pass/fail status derivation follows the exact cascade from the bash source:
1. If `tool_response.exit_code == 0` (or `tool_response.returncode == 0`): STATUS = pass
2. If `tool_response.exit_code` is non-zero and non-null: STATUS = fail
3. Else if `tool_response.interrupted == true`: STATUS = fail
4. Else if `tool_response.interrupted == false`: STATUS = pass
5. Else: STATUS = unknown → exit 0 without writing state file
Bats test covers all 5 derivation paths.

### AC-005 (traces to BC-7.03.074 postcondition 1)
When STATUS is pass or fail: WASM plugin writes `.factory/regression-state.json` with
JSON structure `{"status": "<pass|fail>", "timestamp": "<ISO8601-UTC>", "command": "<cmd>"}`.
Timestamp format: `%Y-%m-%dT%H:%M:%SZ`. State file is atomically overwritten (not
appended) on each matching invocation. Exits 0.
When `.factory/` directory does not exist: exits 0 without writing (same as bash guard).

### AC-006 (traces to BC-7.03.075 postcondition 1)
When prior status (read from `.factory/regression-state.json`) was `"pass"` and new
STATUS is `"fail"`: WASM plugin calls `host::emit_event` with:
`type=hook.block hook=regression-gate matcher=Bash reason=regression_gate_pass_to_fail
severity=warn command=<cmd>`,
then writes to stderr:
`"regression-gate: suite transitioned pass → fail.\n  command: <cmd>\n  recorded: .factory/regression-state.json\n"`.
Exits 0. Does NOT block execution (exit 0, on_error=continue).

### AC-007 (traces to BC-7.03.071 postcondition 1 through BC-7.03.075 postcondition 1)
Bats parity test at `tests/integration/hooks/regression-gate.bats` covers:
(a) `cargo test` command, exit_code=0, no prior state: STATUS=pass, state written, no warning;
(b) `cargo test` command, interrupted=false, no prior state: STATUS=pass, state written, no warning;
(c) `cargo test` command, exit_code=1, prior state=pass: STATUS=fail, regression warning emitted, state written;
(d) `cargo test` command, interrupted=true, prior state=pass: STATUS=fail, regression warning emitted;
(e) `cargo test` command, neither exit_code nor interrupted present: STATUS=unknown, no state write;
(f) `git commit` command (non-test): exit 0 immediately, no state write;
(g) `.factory/` absent: exit 0, no state write;
(h) prior state=fail, new STATUS=fail: state overwritten, no regression warning (only pass-to-fail triggers);
(i) WASM warm invocation time compared to S-8.00 baseline; within 20% regression ceiling.
All bats tests pass.
