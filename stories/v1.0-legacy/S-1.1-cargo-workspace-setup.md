## Goal

Establish the Cargo workspace at the repo root so all Rust crates
(dispatcher, SDK, hook-plugins, sink drivers) share a coherent build
graph, dependency set, and CI pipeline. This is a pure-plumbing story —
no business logic — but it's the single blocker preventing Tier B.x from
fanning out in parallel.

## Acceptance criteria

- [ ] `Cargo.toml` at repo root declares a workspace with members
  `crates/factory-dispatcher`, `crates/hook-sdk`, and a stub plugin
  `crates/hook-plugins/capture-commit-activity`
- [ ] Each member crate has a minimal `Cargo.toml` + placeholder `lib.rs`
  or `main.rs` that compiles cleanly
- [ ] Workspace uses `resolver = "2"`
- [ ] Shared dependency versions pinned in `[workspace.dependencies]` —
  proposed minimum: `serde`, `serde_json`, `toml`, `anyhow`, `thiserror`,
  `tokio`, `tracing`, `tracing-subscriber`, `wasmtime`, `clap`
- [ ] `rust-toolchain.toml` pins a specific stable Rust version (e.g.,
  `1.83.0`) for reproducibility across developer machines and CI
- [ ] `.github/workflows/ci.yml` gains a `cargo-check` job that runs
  `cargo check --workspace --all-targets` + `cargo fmt --check` +
  `cargo clippy --workspace --all-targets -- -D warnings`
- [ ] Existing bats CI (1177 tests) continues to run unchanged
- [ ] `target/` is gitignored (verify)
- [ ] Workspace builds clean on darwin-arm64, linux-x64 at minimum (other
  platforms arrive in S-2.3)

## Contracts

**Workspace layout contract (consumed by S-1.2 through S-1.9):**

```
Cargo.toml                                  # workspace root
crates/
  factory-dispatcher/                       # the binary
  hook-sdk/                                 # the SDK
  hook-plugins/
    <per-plugin>/                           # each a separate crate
```

Stories that add crates:
- **S-1.2, S-1.4, S-1.5, S-1.6, S-1.7**: touch `factory-dispatcher`
- **S-1.3, S-2.5**: touch `hook-sdk`
- **S-2.1, S-3.1, S-3.2, S-3.3**: add / touch members under
  `crates/hook-plugins/`
- **S-1.8, S-1.9, S-4.1–S-4.6**: add new crates `crates/sink-*` (new
  members added to workspace as they're introduced)

**Rust version pin:** all crates target the same stable Rust version
declared in `rust-toolchain.toml`. Stories that need a newer feature
negotiate a version bump as a separate PR.

**Dependency version contract:** shared deps (serde, tokio, wasmtime, etc.)
have versions pinned in `[workspace.dependencies]`. Member crates
reference them with `workspace = true`. Changing a pinned version is a
separate PR requiring rebuild of all members.

## Notes

Rust version pin: propose `1.83.0` as of this writing (2026-04). Bump if
we discover a need for a specific feature only in a newer version.

`Cargo.lock` commit decision: binary-producing workspaces commit it;
libraries typically don't. We're the former (producing a factory-dispatcher
binary), so commit. This also ensures reproducibility for the committed
binaries CI produces.

`crate-type = ["cdylib"]` + `wasm32-wasip1` target for plugin crates is
non-negotiable — required by wasmtime to instantiate them as modules.

No `#[hook]` macro in the hook-sdk stub yet — S-1.3 adds that. This story
is purely structural plumbing.

The CI matrix starts with 2 platforms to prove the workspace compiles. The
full 5-platform matrix (darwin-arm64/x64, linux-x64/arm64, windows-x64)
lands in S-2.3 so we don't block this story on Windows CI setup complexity.
