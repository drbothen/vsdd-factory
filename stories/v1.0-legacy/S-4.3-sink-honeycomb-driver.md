# S-4.3: sink-honeycomb driver

**Epic:** [v1.0 Factory Plugin Kit](EPIC.md)
**Phase:** Phase 4 — Observability sinks
**Milestone:** `1.0.0-rc.1`
**Tier:** E
**Blocks:** S-4.7, S-4.8
**Blocked by:** S-1.8, S-4.1 (shared HTTP infrastructure)
**Estimated effort:** S

## Goal

Implement a Honeycomb-specific sink using their Events API. Similar
pattern to Datadog: send JSON events, honor rate limits, handle errors.
Honeycomb's model is slightly different (event-at-a-time structure,
dataset-based routing) which this story handles.

## Acceptance criteria

- [ ] `HoneycombSink` implements the `Sink` trait
- [ ] Config: `api_key`, `dataset`, `api_host` (default
  `https://api.honeycomb.io`)
- [ ] POSTs to `/1/batch/<dataset>` with the Honeycomb batch schema
- [ ] Handles rate limits via response headers
- [ ] Retries and circuit breaker from S-4.4
- [ ] Integration test against mock receiver

## Tasks

1. Create `crates/sink-honeycomb/`.
2. Implement payload translation:
   - Honeycomb expects `{ "data": {...}, "time": "...", "samplerate": 1 }`
     entries per event
3. POST batches to `/1/batch/<dataset>`
4. Auth via `X-Honeycomb-Team: <api-key>` header
5. Handle rate limiting
6. Mock receiver integration tests

## Artifacts

- `crates/sink-honeycomb/`
- `crates/factory-dispatcher/src/sinks/mod.rs` — register driver

## Contracts

**Config:**

```toml
[[sinks]]
name = "honeycomb-debugging"
type = "honeycomb"
enabled = false
api_key = "${HONEYCOMB_API_KEY}"
dataset = "vsdd-factory"
api_host = "https://api.honeycomb.io"       # optional
```

**Honeycomb batch payload:**

```json
[
  { "data": { "type": "commit.made", "commit_sha": "...", ... },
    "time": "2026-04-24T20:13:45Z", "samplerate": 1 },
  { "data": { ... }, "time": "...", "samplerate": 1 }
]
```

## Test plan

- Unit: event → Honeycomb entry conversion
- Integration: mock receiver verifies payload + auth header
- Integration: rate-limit + retry behavior

## Notes

Honeycomb's API is documented at
https://docs.honeycomb.io/api/events/. Payload format is stable.

The `samplerate` field supports Honeycomb's sampling; we always send `1`
(no sampling) for now. If factory event volume grows, sampling is a
future concern.

Honeycomb classic vs environments: API is slightly different. Target
Honeycomb Environments (the modern API) by default; add a
`classic: bool` config flag if operators need classic support later.
