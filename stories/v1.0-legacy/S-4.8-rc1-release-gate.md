## Goal

Gate story — cuts `1.0.0-rc.1` after Phase 3 + 4 complete and the beta
has been stable for at least 2 weeks. Validates rc.1 gate criteria 8–14
from the design doc's success criteria.

## Acceptance criteria

- [ ] **(8)** Native WASM ports of 3 hooks + emit_event host function
  all working
- [ ] **(9)** All 5 sink types implemented and integration-tested
- [ ] **(10)** Zero-disk mode demonstrable (from S-4.7 tests)
- [ ] **(11)** Hybrid mode demonstrable (from S-4.7 tests)
- [ ] **(12)** Circuit breaker validated (from S-4.7 tests)
- [ ] **(13)** Windows: at least one real user has run the plugin with
  `/vsdd-factory:factory-health` showing all 4 native-WASM hooks
  firing
- [ ] **(14)** 2 weeks of beta (post-beta.1 tag) with no critical bugs

Process gates:

- [ ] All S-3.x, S-4.x stories closed
- [ ] `1.0.0-rc.1` release notes drafted
- [ ] CHANGELOG entry written
- [ ] Version bumped via `bump-version.sh 1.0.0-rc.1 "..."`
- [ ] Release workflow produces pre-release with bundled binaries

## Contracts

**rc status semantics:**

- Breaking changes still allowed between rc.N and rc.N+1 (though
  discouraged)
- No breaking changes between final rc.N and `1.0.0` — rc is the
  stabilization window
- Marketplace still installs 0.79.4 by default; rc is explicit opt-in

## Test plan

- Manual walkthrough of every rc criterion with evidence
- CI green
- GH pre-release created correctly
- Dogfood: vsdd-factory factory running on rc for ≥ 1 week before 1.0
  gate story opens

## Notes

rc.1 is "almost stable" — the last chance to fix things before the
semver stability promise kicks in at 1.0.0.

If a critical bug surfaces in rc.1, iterate with `1.0.0-rc.2` —
same gate checklist, just diff from rc.1.

The 2-week beta stability window is a soft gate; if we find a showstopper
on day 13, extend. Don't promote to rc on a hard deadline.
