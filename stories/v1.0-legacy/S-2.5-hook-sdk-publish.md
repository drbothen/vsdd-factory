## Goal

Publish the `vsdd-hook-sdk` crate (and its proc-macro subcrate
`vsdd-hook-sdk-macros`) to crates.io at version 0.1.0. Establishes the
public Rust API surface third-party plugin authors (and our own plugin
crates) depend on. Per Q2 resolution: starts at 0.1.0 during the v1.0
prerelease cycle; stabilizes to 1.0.0 alongside the final v1.0.0 factory
release.

## Contracts

**SDK version cadence:**

- SDK version is INDEPENDENT of factory plugin version
- SDK moves forward only when its public API or host ABI changes
- Factory dispatcher pins a compatible SDK range:
  `vsdd-hook-sdk = ">= 0.1, < 1"` during prerelease; `">= 1, < 2"` after
  stabilization
- Breaking SDK changes require bump (0.2.0 in 0.x range, or major in 1.x+)

**Host ABI compatibility:**

```rust
// In vsdd-hook-sdk:
pub const HOST_ABI_VERSION: u32 = 1;

// Plugin build time check:
const _: () = assert!(HOST_ABI_VERSION == 1);
// Dispatcher refuses to load plugins declaring an incompatible ABI.
```

SDK 0.x can change HOST_ABI_VERSION freely between releases; SDK 1.x is
committed to ABI stability.

**Publishing order per release:**

1. `vsdd-hook-sdk-macros` first (proc-macro crates must publish before
   their consumers)
2. `vsdd-hook-sdk` second (depends on macros)
3. All downstream plugins rebuilt against the new version in the same
   release cycle

## Notes

Name reservation: if `vsdd-hook-sdk` is taken, try `factory-hook-sdk`,
`claude-factory-hook-sdk`. Don't squat other names preemptively.

Proc-macro publishing: the crates.io convention is to publish the proc-
macro crate and re-export from a main library crate. We're following
that (hook-sdk re-exports `#[hook]` from hook-sdk-macros).

Automated publishing has failure modes (token expiry, rate limits). Keep
the manual fallback documented.

License: match whatever vsdd-factory's repo license is. If dual-licensed
(MIT OR Apache-2.0) like most Rust crates, do the same.

Docs.rs builds: may need a `[package.metadata.docs.rs]` block if the
crate has platform-specific features or requires `wasm32-wasip1` target
to compile examples. Test with `cargo doc --all-features` locally before
publishing.
