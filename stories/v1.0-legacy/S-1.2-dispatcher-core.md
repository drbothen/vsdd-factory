# S-1.2: factory-dispatcher core (stdin, TOML load, routing)

**Epic:** [v1.0 Factory Plugin Kit](EPIC.md)
**Phase:** Phase 1 — Dispatcher foundation
**Milestone:** `1.0.0-beta.1`
**Tier:** B.x
**Blocks:** S-1.4, S-1.5, S-1.6, S-1.7, S-2.1, S-2.3, S-2.7
**Blocked by:** S-1.1
**Estimated effort:** L

## Goal

Implement the core skeleton of `factory-dispatcher`: read stdin JSON from
Claude Code, parse it, load `hooks-registry.toml` (cached + mtime-invalidated),
filter registered plugins by event + tool match, group them by priority tier,
and hand off to the execution machinery (S-1.5/S-1.6 fill in). This story
owns the dispatcher's I/O boundary, config loading, and routing decisions —
everything before a plugin actually runs.

## Acceptance criteria

- [ ] `factory-dispatcher` reads stdin JSON and parses Claude Code hook
  payload (event name, tool name, session_id, tool_input, tool_response)
- [ ] `dispatcher_trace_id` generated per invocation (UUID v4)
- [ ] `hooks-registry.toml` loaded via `toml` crate into strongly typed
  structs; schema_version field validated (must be 1)
- [ ] Registry cached in-process for the duration of the invocation;
  mtime-invalidation logic documented even if the dispatcher is a short-
  lived process (relevant for future long-running modes)
- [ ] Plugin filter: iterates registry, selects entries where
  `event == payload.event_name` AND (no `tool` field OR tool regex matches
  `payload.tool_name`)
- [ ] Matched plugins grouped by `priority` field; tiers emit in ascending
  priority order
- [ ] Dispatcher exits 0 by default; exit 2 only when an `on_error = "block"`
  plugin returned a block result (plumbing is stubbed here; real enforcement
  is in S-1.5/S-1.6)
- [ ] Structured error envelope on malformed stdin, missing registry, etc.
  — emit `internal.dispatcher_error` event and exit 0 (non-blocking)
- [ ] Covered by unit tests for every parsing/routing branch

## Tasks

1. Define Rust types for:
   - `HookPayload` (mirrors Claude Code's stdin JSON — use `serde_json::Value`
     for `tool_input`/`tool_response` since the schema varies per tool)
   - `RegistryEntry` (name, event, tool?, plugin_path, priority, enabled,
     timeout_ms?, on_error?, capabilities?)
   - `Registry` (schema_version, defaults, [[hooks]])
2. Implement `load_registry(path: &Path) -> Result<Registry>` with TOML
   parsing + schema_version validation.
3. Implement `match_plugins(registry: &Registry, payload: &HookPayload) -> Vec<&RegistryEntry>`:
   filter by event and tool regex, drop `enabled == false`.
4. Implement `group_by_priority(matched: Vec<&RegistryEntry>) -> Vec<Vec<&RegistryEntry>>`:
   sorted tiers.
5. Main entry point:
   - Parse stdin
   - Assign `dispatcher_trace_id`
   - Load registry from `${CLAUDE_PLUGIN_ROOT}/hooks-registry.toml`
   - Match + group
   - **Call into execution layer** (S-1.5/S-1.6; stub for now with a
     placeholder `execute_tiers(tiers) -> Vec<PluginResult>` that just
     prints tier info)
   - Aggregate results; decide exit code
6. Error handling: any recoverable error emits `internal.dispatcher_error`
   via the S-1.7 writer (stub the writer for now; S-1.7 implements it).
7. Tests: unit coverage for tool-regex matching, event filtering, priority
   grouping, malformed registry rejection, missing registry, disabled
   entries.

## Contracts

**`Registry` Rust type (consumed by S-1.4, S-1.5, S-1.6):**

```rust
pub struct Registry {
    pub schema_version: u32,         // must be 1 for v1.0
    pub defaults: RegistryDefaults,
    pub hooks: Vec<RegistryEntry>,
}

pub struct RegistryDefaults {
    pub timeout_ms: u32,             // default 5000
    pub on_error: OnError,           // default Continue
}

pub struct RegistryEntry {
    pub name: String,
    pub event: String,
    pub tool: Option<String>,        // regex; None = all tools
    pub plugin: PathBuf,
    pub priority: u32,               // default 500
    pub enabled: bool,               // default true
    pub timeout_ms: Option<u32>,
    pub fuel_cap: Option<u64>,       // default 10_000_000
    pub on_error: Option<OnError>,
    pub capabilities: Option<Capabilities>,
}

pub enum OnError { Continue, Block }
```

**Routing decision contract (consumed by S-1.5, S-1.6):**

The dispatcher hands the execution layer a `Vec<Vec<&RegistryEntry>>`
(tiers ordered ascending by priority, plugins within a tier in registry
order). Execution layer is responsible for:
- Running each tier sequentially
- Running plugins within a tier in parallel (S-1.6)
- Returning per-plugin results

  emits the right events and exits correctly
- Malformed-input tests: broken JSON, missing registry, schema_version=2,

## Notes

Use `serde` with `deny_unknown_fields` on registry types so typos in the
TOML fail loudly rather than silently. Exception: `capabilities` is an
opaque subtree we'll type in S-1.4.

The dispatcher is expected to be short-lived (one invocation per tool
call, process exits). Keep startup cost tight: lazy-load as little as
possible, avoid global state, minimize dependencies. Target startup < 5ms
on a warm page cache.

`dispatcher_trace_id` is a UUID v4 string; propagated to every emitted
event as an attribute. S-1.7 wires it into `emit_event()`.
