# S-5.1: SessionStart hook wiring

**Epic:** [v1.0 Factory Plugin Kit](EPIC.md)
**Phase:** Phase 5 — New Claude Code hook events
**Milestone:** `1.0.0`
**Tier:** G
**Blocks:** S-5.7
**Blocked by:** S-4.8
**Estimated effort:** S

## Goal

Wire the `SessionStart` event type through the dispatcher. Emits
`session.started` with session_id, runs a factory-health brief, checks
tool dependencies. First session-level event, unlocks the session-
duration tracking that `SessionEnd` (S-5.2) completes.

## Acceptance criteria

- [ ] Dispatcher accepts `SessionStart` in hooks.json and registry
- [ ] A `session-start-telemetry` WASM plugin emits `session.started`
  with session_id, factory_version, plugin_count, activated_platform,
  timestamp
- [ ] Plugin runs `factory-health --brief` (via `exec_subprocess` host
  fn) and captures its result into the event
- [ ] Plugin checks basic tool dependencies (`git`, `jq`, `yq`) and
  emits a `session.tool_deps_missing` event if any are absent
- [ ] Registered in `hooks-registry.toml` with `once = true` (mirrors
  hooks.json `"once": true` convention for session events)
- [ ] Works on all 5 platforms
- [ ] Integration test: session starts → event lands in file sink with
  expected fields

```json
{
  "type": "session.started",
  "session_id": "...",
  "factory_version": "1.0.0",
  "plugin_count": 31,
  "activated_platform": "darwin-arm64",
  "factory_health": "healthy" | "warnings" | "errors",
  "tool_deps": { "git": "...", "jq": "...", "yq": "..." }
}
```

## Notes

The `factory-health --brief` subprocess spawn requires
`exec_subprocess` capability; add to the plugin's registry entry.

`session_id` comes from Claude Code as an environment or payload
field — use the canonical source (dispatcher's `session_id()` host
function).
