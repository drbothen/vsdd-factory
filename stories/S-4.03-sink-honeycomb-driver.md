# S-4.03: sink-honeycomb driver

**Epic:** E-4 — Observability Sinks and RC Release
**Phase:** Phase 4 — Observability sinks
**Milestone:** `1.0.0-rc.1`
**Tier:** E
**Blocks:** S-4.07, S-4.08

**Estimated effort:** S

## Capability Anchor Justification

Anchoring to **CAP-023: "Ship advanced observability sinks (HTTP, Datadog, Honeycomb)"** per capabilities.md §CAP-023.

CAP-023 states: "Planned sink drivers (S-4.1/4.2/4.3) allow direct event forwarding without a local OTel collector." S-4.03 is explicitly cited as S-4.3 in CAP-023 as the Honeycomb sink. This story delivers one of the three named sink drivers in CAP-023's outcome statement.

## Narrative
- **As a** factory operator using Honeycomb
- **I want to** configure a Honeycomb sink with my API key and dataset
- **So that** factory events appear in Honeycomb as structured events for tracing and analysis

## Goal

Implement a Honeycomb-specific sink using their Events API. Similar
pattern to Datadog but Honeycomb's model uses event-at-a-time structure
with a mandatory `time` field and dataset routing via headers.

## Acceptance Criteria

- [ ] `crates/sink-honeycomb` implements the `Sink` trait
  (traces to BC-3.01.001 postcondition 1: Sink integrates with SinkRegistry base machinery)
- [ ] Sends events to Honeycomb Events API (`/1/events/<dataset>`)
  (traces to BC-3.01.001 postcondition 1: Sink integrates with SinkRegistry machinery; non-blocking Honeycomb submit is uncontracted — v1.1 BC candidate `BC-3.NN.NNN-cross-sink-non-blocking-submit`)
- [ ] Auth via `X-Honeycomb-Team` header from config
  (traces to BC-3.06.005 postcondition 1: SinkConfigCommon governs config parsing; API key is a required field)
- [ ] Each event includes `time` field in RFC3339 format
  (traces to BC-3.01.001 postcondition 1: events flow through registry to Honeycomb sink; `time` field injection is uncontracted — v1.1 BC candidate `BC-3.NN.NNN-honeycomb-time-field-rfc3339`)
- [ ] Dataset configurable per sink instance
  (traces to BC-3.06.005 postcondition 1: per-sink config fields parsed via SinkConfigCommon pattern)
- [ ] Integration test with mock Honeycomb endpoint
  (traces to BC-3.01.001 postcondition 1: end-to-end registry-level operations succeed)
