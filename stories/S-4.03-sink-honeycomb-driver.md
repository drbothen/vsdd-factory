# S-4.03: sink-honeycomb driver

**Epic:** E-4 — Observability Sinks and RC Release
**Phase:** Phase 4 — Observability sinks
**Milestone:** `1.0.0-rc.1`
**Tier:** E
**Blocks:** S-4.07, S-4.08
**Blocked by:** S-1.08, S-4.01 (shared HTTP infrastructure)
**Estimated effort:** S

## Narrative
- **As a** factory operator using Honeycomb
- **I want to** configure a Honeycomb sink with my API key and dataset
- **So that** factory events appear in Honeycomb as structured events for tracing and analysis

## Goal

Implement a Honeycomb-specific sink using their Events API. Similar
pattern to Datadog but Honeycomb's model uses event-at-a-time structure
with a mandatory `time` field and dataset routing via headers.

## Acceptance criteria

- [ ] `crates/sink-honeycomb` implements the `Sink` trait
- [ ] Sends events to Honeycomb Events API (`/1/events/<dataset>`)
- [ ] Auth via `X-Honeycomb-Team` header from config
- [ ] Each event includes `time` field in RFC3339 format
- [ ] Dataset configurable per sink instance
- [ ] Integration test with mock Honeycomb endpoint
