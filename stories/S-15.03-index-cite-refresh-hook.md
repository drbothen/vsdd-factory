# S-15.03 — ARCH-INDEX Cite-Refresh Hook + Lessons Retroactive-Sweep Verification

## Problem Statement

Prose codification of index-cite-refresh and lesson-retroactive-sweep discipline has failed
across 15+ fix-bursts. ARCH-INDEX self-codified the cite-refresh rule at v1.18 (2026-05-07)
and reinforced it at v1.19. Despite this, the rule was systematically ignored across fix-bursts
4 through 18; pass-20 found ARCH-INDEX cite was 10 versions stale (BC-INDEX v1.33 vs current
v1.43).

Three consecutive HIGH passes (P18-001 sibling-hook bug, P19-001 codified-not-applied,
P20-002 cite-stale-15-fix-bursts) demonstrate that prose codification of cite-refresh discipline
is structurally insufficient. Mechanical enforcement is required.

## Goal

Implement WASM hook plugins that enforce these disciplines mechanically, preventing recurrence
of the codified-not-enforced class of failures.

## Scope

1. `validate-index-cite-refresh` hook — checks ARCH-INDEX body cites against BC-INDEX/VP-INDEX/
   STORY-INDEX frontmatter versions; blocks Edit/Write if cites are stale by >0 versions.

2. `validate-lesson-retroactive-sweep` hook (optional) — checks that any new entry to lessons.md
   was preceded by a corpus-wide grep for the codified pattern; blocks Write to lessons.md without
   an associated grep+migration commit.

## Out of Scope

- Other types of cite drift (e.g., between non-index docs).
- Automated cite refresh (hook blocks only; human fixes the cite).
