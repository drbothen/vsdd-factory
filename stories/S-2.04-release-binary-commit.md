# S-2.04: Binary commit automation in Release workflow

**Epic:** E-2 — Legacy Adapter and Beta Release
**Phase:** Phase 2 — Legacy bash adapter
**Milestone:** `1.0.0-beta.1`
**Tier:** C
**Blocks:** S-2.08
**Blocked by:** S-2.03 (needs the matrix artifacts)
**Estimated effort:** M

## Narrative
- **As a** release engineer
- **I want to** push a tag and have the release workflow automatically commit the per-platform binaries to the repo
- **So that** users installing via `/plugin install` get the correct pre-built binary for their platform

## Goal

When a release tag is pushed, the Release workflow downloads the CI
matrix's platform-specific `factory-dispatcher` binaries and WASM plugin
artifacts, commits them to `plugins/vsdd-factory/hooks/dispatcher/bin/<platform>/`
in the repo, and attaches them to the GitHub Release.

## Acceptance criteria

- [ ] Release workflow downloads artifacts from the CI matrix run that produced the tag's commit
- [ ] Binaries committed to `plugins/vsdd-factory/hooks/dispatcher/bin/<platform>/factory-dispatcher[.exe]`
- [ ] WASM plugins committed to `plugins/vsdd-factory/hooks/plugins/*.wasm`
- [ ] Chore commit message follows convention: `chore: commit release binaries for vX.Y.Z`
- [ ] Binaries attached to the GitHub Release as downloadable assets
- [ ] Workflow fails if any platform binary is missing from CI artifacts
