# S-2.03: Cross-platform CI matrix build targets

**Epic:** E-2 — Legacy Adapter and Beta Release
**Phase:** Phase 2 — Legacy bash adapter
**Milestone:** `1.0.0-beta.1`
**Tier:** C
**Blocks:** S-2.04, S-2.08
**Blocked by:** S-1.01, S-1.02 (needs a buildable dispatcher)
**Estimated effort:** M

## Narrative
- **As a** CI/release engineer
- **I want to** have `factory-dispatcher` and all WASM plugins built on all 5 target platforms in CI
- **So that** every push to main produces verified cross-platform artifacts

## Goal

Extend the CI workflow to build `factory-dispatcher` and all WASM plugins
on all 5 target platforms: darwin-arm64, darwin-x64, linux-x64, linux-arm64,
windows-x64. Every push to main must produce a green build across the full matrix.

## Acceptance criteria

- [ ] CI workflow matrix covers all 5 platforms
- [ ] `factory-dispatcher` binary built for each platform
- [ ] All WASM plugins built for `wasm32-wasip1` (platform-independent)
- [ ] Build artifacts uploaded as GitHub Actions artifacts
- [ ] Windows build uses cross-compilation (not Windows runners) or native Windows runner
- [ ] `ci/platforms.yaml` defines the platform matrix; drift check script validates it
- [ ] Existing bats suite continues to run unchanged on linux-x64

## Capability Anchor Justification

Anchoring to CAP-007: "Deploy and activate the plugin on any supported platform" per capabilities.md §CAP-007.
Justification: S-2.03 implements the 5-platform CI build matrix (darwin-arm64, darwin-x64, linux-x64, linux-arm64, windows-x64). BC-9.01.004 contracts that this matrix is pinned in ci/platforms.yaml and that check-platforms-drift.py gates divergence. This is the direct implementation vehicle for "deploy on any supported platform" — the CI matrix produces the platform-specific binaries that the activate skill then deploys.
