## Capability Anchor Justification

Anchoring to **CAP-003: "Stream observability events to multiple configurable sinks"** per capabilities.md §CAP-003.

CAP-003 states: "The dispatcher fans out every internal event to all enabled sink drivers (file, OTel gRPC; HTTP/Datadog/Honeycomb planned for rc.1). Sinks are independently configured via `observability-config.toml`." Per-sink routing filters and tag enrichment are the mechanisms by which the "independently configured" portion of CAP-003 is fully realized — without routing filters, all sinks receive all events (no per-sink control), which violates the "independently configured" promise. S-4.06 completes the wiring that makes per-sink independent configuration observable.

## Narrative
- **As a** factory operator with multiple sinks
- **I want to** configure routing filters and static tags per sink
- **So that** audit events go to an audit file sink and metrics to a metrics sink, with additional context tags added automatically

## Goal

Implement the routing-filter and tag-enrichment features every sink
supports. Routing filters let operators route specific event types to
specific sinks. Tag enrichment adds static key/value tags to every event
passing through a sink.

## Acceptance Criteria

- [ ] `RoutingFilter` fully applied in sink dispatch path (not just parsed)
  (traces to BC-3.01.004 postcondition 1: `RoutingFilter` empty = pass-through, allow non-empty = whitelist, deny applied after allow; the "applied in dispatch path" wiring itself is uncontracted — v1.1 BC candidate `BC-3.NN.NNN-router-filter-wired-in-dispatch`. See v1.1 BC Candidates table.)
- [ ] Static tags merged into every event before sink `submit()`
  (traces to BC-3.02.010 postcondition 1: configured tags `env=prod,team=factory` land on every event passing through the sink; tags applied without overwriting producer fields per BC-3.02.011 postcondition 1)
- [ ] Tag enrichment does not overwrite producer-set fields
  (traces to BC-3.02.011 postcondition 1: tag with `key="type"` does NOT clobber producer's type field — invariant preserved after wiring)
- [ ] Routing filter supports: include_event_types, exclude_event_types, include_plugins
  (traces to BC-3.01.004 postcondition 1: allow non-empty = whitelist; deny applied after allow; BC-3.06.001 postcondition 1: empty allow + empty deny → every event passes)
- [ ] Case-sensitive matching in allow-list
  (traces to BC-3.06.006 postcondition 1: allow-list compares case-sensitively — `Commit.Made` does not match `commit.made`)
- [ ] Integration test: two sinks with different filters; verify events routed correctly
  (traces to BC-3.02.009 postcondition 1: `allow=["commit.made"]` passes 2 of 3 events through; BC-3.04.002 postcondition 1: Router is the stable extension point that S-4.x wires routing behavior into without touching call sites)
- [ ] Default (empty filter) passes all events through to sink
  (traces to BC-3.06.001 postcondition 1: empty allow + empty deny → every event passes; BC-3.01.004 postcondition 1: empty RoutingFilter = pass-through)
