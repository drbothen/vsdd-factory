# S-3.01: Port capture-commit-activity to WASM

**Epic:** E-3 — WASM Port: High-Value Hooks
**Phase:** Phase 3 — High-value port to native WASM
**Milestone:** `1.0.0-rc.1`
**Tier:** E
**Blocks:** S-4.07, S-4.08
**Blocked by:** S-2.08 (beta.1 shipped), S-3.04 (emit_event host fn)
**Estimated effort:** M

## Narrative
- **As a** factory operator
- **I want to** have `capture-commit-activity` running as native WASM instead of legacy bash
- **So that** git commit events are captured reliably with typed, structured data

## Goal

Rewrite `capture-commit-activity` from bash to native WASM using the
`vsdd-hook-sdk`. First of four priority ports. Parses `git commit`
invocations from PostToolUse/Bash payloads, extracts sha/branch from
the git subprocess, and emits `commit.made` events.

## Acceptance criteria

- [ ] `capture-commit-activity.wasm` replaces the legacy bash hook
- [ ] Parses `git commit` invocations from PostToolUse/Bash `tool_input.command`
- [ ] Invokes `git log -1 --format=%H` via `exec_subprocess()` to get commit sha
- [ ] Emits `commit.made` event with fields: sha, branch, message, author, timestamp
- [ ] Handles non-commit bash commands gracefully (no-op)
- [ ] `hooks-registry.toml` entry updated from legacy-bash to native WASM
- [ ] Existing bats tests for commit capture still pass
