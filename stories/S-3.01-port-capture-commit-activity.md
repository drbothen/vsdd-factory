# S-3.01: Port capture-commit-activity to WASM

**Epic:** E-3 — WASM Port: High-Value Hooks
**Phase:** Phase 3 — High-value port to native WASM

## Capability Anchor Justification

Primary: CAP-013 ("Capture post-execution activity (PostToolUse hooks)") per capabilities.md §CAP-013. This story replaces the legacy bash capture-commit-activity hook with a native WASM plugin that captures commit metadata and emits `commit.made` events — directly implementing the PostToolUse capture capability.

FR anchor: FR-014 ("Capture-commit-activity plugin (PostToolUse event capture)") per PRD §2.4. FR-014 is the sole SS-04 FR for this plugin. Note: BC-4.03.001 is the pre-S-3.01 stub placeholder; this story replaces the stub with full implementation, which will require v1.1 BCs (see below).

VP candidates are listed under v1.1 BC candidates below.

## Narrative
- **As a** factory operator
- **I want to** have `capture-commit-activity` running as native WASM instead of legacy bash
- **So that** git commit events are captured reliably with typed, structured data

## Goal

Rewrite `capture-commit-activity` from bash to native WASM using the
`vsdd-hook-sdk`. First of four priority ports. Parses `git commit`
invocations from PostToolUse/Bash payloads, extracts sha/branch from
the git subprocess, and emits `commit.made` events.
