# S-0.01: bump-version.sh prerelease support

**Epic:** E-0 — Infrastructure Prep
**Phase:** Phase 0 — Infrastructure prep
**Milestone:** `1.0.0-beta.1`
**Tier:** A
**Blocks:** S-2.08, S-4.08, S-5.07 (release gate stories)
**Blocked by:** none
**Estimated effort:** S

## Narrative
- **As a** release engineer
- **I want to** run `bump-version.sh` with prerelease version strings like `1.0.0-beta.1`
- **So that** every v1.0 prerelease can be tagged and shipped without manual workarounds

## Goal

`scripts/bump-version.sh` currently accepts semver format `N.N.N` (validated
at line 49 via `^[0-9]+\.[0-9]+\.[0-9]+$`). Prerelease tags like
`1.0.0-beta.1` will be rejected, blocking every v1.0 prerelease. Extend the
validator to accept semver prerelease suffixes while keeping the stable-form
behavior identical.

## Acceptance criteria

- [ ] `scripts/bump-version.sh 1.0.0` still works (no regression)
- [ ] `scripts/bump-version.sh 1.0.0-beta.1` passes validation and writes
  the prerelease version to `plugin.json` and `marketplace.json`
- [ ] `scripts/bump-version.sh 1.0.0-rc.2` works identically
- [ ] `scripts/bump-version.sh 1.0.0-alpha` works (no numeric suffix allowed
  per strict semver, but our use case is `beta.N`/`rc.N`)
- [ ] `scripts/bump-version.sh 1.0` still fails with a clear error
- [ ] `scripts/bump-version.sh 1.0.0-` fails with a clear error
- [ ] CHANGELOG idempotent-stub detection (v0.79.1) still works against
  prerelease headings like `## 1.0.0-beta.1 — ...`

## Tasks

1. Update the version regex at `scripts/bump-version.sh:49` to accept
   optional prerelease per semver 2.0:
   `^[0-9]+\.[0-9]+\.[0-9]+(-[0-9A-Za-z.-]+)?$`
2. Update the usage message to mention prerelease examples.
3. Update the CHANGELOG heading detection regex (line 116) so that
   `## 1.0.0-beta.1` is detected as a pre-existing heading. (Current regex
   already uses the full version string, so likely just verify.)
4. Add unit-ish tests in `plugins/vsdd-factory/tests/bump-version.bats` (or
   create if missing) covering stable + prerelease + malformed cases.
5. Dry-run against a scratch branch and verify output.

## Artifacts

- `scripts/bump-version.sh` — regex update, usage string update
- `plugins/vsdd-factory/tests/bump-version.bats` — new or extended tests

## Contracts

- **Version string format accepted:** semver 2.0 core (`N.N.N`) with optional
  prerelease suffix matching `-[0-9A-Za-z.-]+` (e.g., `beta.1`, `rc.2`,
  `alpha`, `beta.1.dev3`)
- **Plugin + marketplace JSON**: both receive the full version string
  verbatim, including prerelease suffix. Release workflow is expected to
  treat prerelease versions as "pre-release" on GitHub Releases.
- **CHANGELOG heading detection**: unchanged — `^## <version>(space|$)`
  regex works for both stable and prerelease since version is interpolated.

## Test plan

- `bats plugins/vsdd-factory/tests/bump-version.bats` — new tests cover
  every acceptance criterion.
- Manual dry-run: create throwaway branch, run
  `scripts/bump-version.sh 1.0.0-beta.1`, verify both JSON files updated
  correctly and CHANGELOG stub (if no prior entry) injected properly.

## Notes

Semver 2.0 prerelease spec: https://semver.org/#spec-item-9

The script's idempotent CHANGELOG guard shipped in v0.79.1; it uses
`^## $NEW_VERSION([[:space:]]|$)` which should work for prerelease versions
unchanged, but verify in tests.

Related release-workflow work is S-0.02 (Release workflow itself handling
prerelease tags as GH prereleases).
