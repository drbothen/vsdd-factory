# S-13.01: Path Governance Bundle — Registry, WASM Hook, Skill Updates, relocate-artifact

**Epic:** E-13 — Artifact Integrity — Path Discipline Single Source of Truth
**Delivery order:** FIRST in cycle (C → A → B). Hard constraint: S-13.01 must be
fully CONVERGED before S-12.01 or S-12.02 are dispatched.
**Depends on:** (none)
**Blocks:** S-12.01, S-12.02 (both depend on `.factory/` path discipline being live)
**Priority:** P0

## Narrative

- **As a** vsdd-factory agent (any role: architect, product-owner, story-writer, etc.)
  writing artifacts to `.factory/`
- **I want** a registry-backed single source of truth for canonical artifact paths,
  enforced by a WASM hook and consulted by every creation skill
- **So that** artifact misplacement is detected and blocked at write time, not
  discovered post-hoc during a consistency audit

## Acceptance Criteria

### AC-001 (traces to BC-4.11.001 precondition 2 + VP-069 proptest)

`plugins/vsdd-factory/config/artifact-path-registry.yaml` exists and is readable. The
`load_registry` function in `validate-artifact-path/src/lib.rs` never panics on any
byte sequence input. VP-069 proptest harness passes (minimum 200 trials, arbitrary
byte sequences).

**Falsifiable test:** `cargo test -p validate-artifact-path` includes the proptest suite
and all trials pass. The registry YAML file exists and `grep artifact_type` returns ≥6
entries (one per artifact class defined in F1 Section 4).

### AC-002 (traces to BC-4.11.001 invariant 2 + VP-070 kani)

`matches_canonical(path, registry)` is pure and deterministic. For the same `(path, registry)`
pair the function always returns the same `PathDecision`. VP-070 kani harness passes
(unwind 16, all four proof functions verified): determinism, non-`.factory/` always
returns `Allow`, empty path returns `Allow`, advisory-only registry never produces `Block`.

**Falsifiable test:** `cargo kani -p validate-artifact-path` completes without counterexamples.

### AC-003 (traces to BC-4.11.001 postcondition 6 — unregistered path blocked)

Hook emits `HookResult::block_with_fix` with `code: "ARTIFACT_PATH_UNREGISTERED"` when
`tool_input.file_path` targets `.factory/` and matches no registry entry. Block message
uses `hook: "validate-artifact-path"`, non-empty `reason` and `recommendation` strings
per the canonical Why/Fix/Code pattern (HOST_ABI.md).

**Falsifiable test:** Rust unit test: inject payload with `file_path:
".factory/feature-deltas/F1-delta.md"` (no registry match); assert
`HookResult::block_with_fix` with `code: "ARTIFACT_PATH_UNREGISTERED"`.

### AC-004 (traces to BC-4.11.001 postcondition 7 — non-.factory early exit)

Hook returns `HookResult::Continue` immediately for any path outside `.factory/`.
No registry lookup is performed for non-`.factory/` paths.

**Falsifiable test:** Rust unit test: inject payload with `file_path: "src/lib.rs"`;
assert `HookResult::Continue` returned and read_file callback NOT invoked.

### AC-005 (traces to BC-4.11.001 postcondition 3/4/5 — per-entry enforcement_level)

Hook respects `enforcement_level` per registry entry:
- `block` entry: path matches → `HookResult::Continue` (write proceeds — the path is valid).
- `warn` entry: path matches → `hook.warn` event + stderr + `HookResult::Continue`.
- `advisory` entry: path matches → `host::log_debug` + `HookResult::Continue` (no stderr).
- No match → `HookResult::block_with_fix` with `ARTIFACT_PATH_UNREGISTERED`.

**Falsifiable test:** Four Rust unit tests (one per branch), each with injectable
read_file and emit_event callbacks. Assert correct block/continue/event emission per branch.

### AC-006 (traces to BC-4.11.001 edge case EC-001, EC-002 — graceful degrade)

When registry is absent (read_file returns `Err`), hook returns `HookResult::Continue`
and logs via `host::log_debug`. When registry YAML is malformed, hook returns
`HookResult::Continue` and logs via `host::log_error`. Neither case emits a block.

**Falsifiable test:** Two Rust unit tests: (a) read_file callback returns `Err`; (b)
read_file callback returns malformed YAML string. Both assert `HookResult::Continue`.

### AC-007 (traces to BC-6.22.001 postcondition 1–5 — dry-run mode)

`relocate-artifact` skill (dry-run, default mode) walks `.factory/`, classifies each
`.md` file by frontmatter `document_type`, looks up canonical pattern in registry,
and emits a Markdown violation table. Emits `"X violations found. Re-run with --apply
to execute."` or `"0 violations found. Registry is clean."` No filesystem changes occur
in dry-run mode.

**Falsifiable test:** bats integration test: fixture directory with one known misplaced
BC file. Dry-run output contains a table row with current and proposed paths. No `git mv`
is executed.

### AC-008 (traces to BC-6.22.001 postcondition 6–9 — apply mode)

`relocate-artifact --apply` executes `git mv <old> <canonical>` for each violation,
updates cross-references in `.md` files, appends move summary to the active cycle's
`decision-log.md`, and reports `"0 violations remaining. Registry is clean.
validate-artifact-path hook may now be registered."` after all moves.

**Falsifiable test:** bats integration test: fixture git repo with misplaced files.
After `--apply`: `git status` shows renames; decision-log contains `D-NNN (auto-relocation)` entry; dry-run shows 0 violations.

### AC-009 (traces to BC-6.22.001 invariant 3 — detect-before-apply atomicity)

`relocate-artifact --apply` detects ALL violations before executing any `git mv`. If
detection fails for any artifact, the entire apply aborts with a non-zero exit and no
moves are performed.

**Falsifiable test:** bats integration test: inject a fixture where the second artifact
has an unresolvable cross-reference. Assert: no `git mv` was executed for the first
artifact either; exit code is non-zero.

### AC-010 (traces to BC-4.11.001 invariant 7 + BC-6.22.001 invariant 7 — delivery sequencing gate)

`validate-artifact-path.wasm` MUST NOT be registered in `hooks-registry.toml` until
`relocate-artifact` has been run and reports zero violations. This is enforced as a
delivery ordering constraint: task T5 (run relocate) precedes task T6 (register hook)
in the story's task list, and the PR description must include the dry-run output
confirming `"0 violations found"`.

**Falsifiable test:** bats integration test reads `hooks-registry.toml` and asserts
`validate-artifact-path` appears AFTER a cycle's dry-run confirms 0 violations (fixture:
presence of `"0 violations found"` in the cycle's `decision-log.md` predates the hook
registration commit timestamp).

Key design decisions codified in F1 and F2 that F4 must honor:

1. **Immediate block mode:** Per OQ5 resolution in BC-4.11.001, the hook ships in
   immediate enforcement mode. The `enforcement_level` field in the registry controls
   per-entry behavior (block/warn/advisory), but unregistered paths are always blocked.
   There is no global "warn-phase rollout" for the hook itself.
2. **injectable-callback pattern:** `fn hook_logic(...)` takes all host I/O as closures.
   Mirrors `regression-gate/src/lib.rs` and `handoff-validator/src/lib.rs` exactly.
3. **Single source of truth invariant:** Neither the hook source, skills, nor the
   relocate skill embed a duplicate path list. All three read the registry at runtime.
   VP-072 bats harness enforces this mechanically.
4. **YAML parser decision (OQ-1):** Spike `serde_yaml` first; if binary > 500 KB,
   switch to a hand-written YAML-subset parser. Document the decision in the PR.

## Architecture Compliance Rules

Extracted from BC-4.11.001, BC-6.22.001, and ADR-016. These are BUILD-TIME invariants:

1. **No hardcoded path lists:** If `lib.rs` imports a constant array of `.factory/`
   path patterns, the build is in violation. All path lookups go through the registry
   read callback.
2. **No bare `HookResult::block()` calls:** All block messages use `block_with_fix`.
   `clippy` should catch this with a custom lint or test assertion.
3. **`HOST_ABI_VERSION = 1`:** No new host functions are introduced. The hook uses
   only `host::read_file`, `host::log_*`, and `host::emit_event` (already in ABI v1).
4. **`relocate-artifact` skill: git mv only:** Direct file copy + delete is prohibited.
   `git mv` is the only move mechanism.
5. **Hook registration after relocation:** The PR for T-6 must not be merged before
   the T-5 dry-run output (0 violations) is recorded in the cycle decision-log.
6. **Forbidden dependencies — `validate-artifact-path` crate:** This crate MUST NOT
   depend on `factory-dispatcher`, `validate-per-story-adversary-convergence`, or any
   other crate in the workspace beyond `vsdd-hook-sdk` and a YAML/JSON parser. If this
   crate gains a dependency on `factory-dispatcher`, the build MUST fail.
