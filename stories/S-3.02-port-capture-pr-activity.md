**Epic:** E-3 — WASM Port: High-Value Hooks
**Phase:** Phase 3 — High-value port to native WASM
**Milestone:** `1.0.0-rc.1`
**Tier:** E

## Capability Anchor Justification

Primary: CAP-013 ("Capture post-execution activity (PostToolUse hooks)") per capabilities.md §CAP-013. This story ports the PR activity capture hook to native WASM, expanding the PostToolUse capture capability from commit events to PR lifecycle events (create/merge/close).

FR anchor: FR-014 ("Capture-commit-activity plugin (PostToolUse event capture)") per PRD §2.4, extended to PR-activity. No dedicated FR exists for the PR-activity WASM port. This story extends FR-014 scope to cover `capture-pr-activity`; a new FR-046 ("capture-pr-activity plugin extension of FR-014") is proposed in the v1.1 BC candidates section (F-003 fix: FR-045 is reserved for S-5 lifecycle events).

Note: BC-2.01.002 is the active SS-02 SDK anchor cited in Architecture Compliance Rules; SS-04 plugin BCs are v1.1 candidates listed below.

## Narrative
- **As a** factory operator
- **I want to** have `capture-pr-activity` running as native WASM
- **So that** PR events (create, merge, close) are captured with typed, structured data

## Goal

Rewrite `capture-pr-activity` from bash to native WASM. More complex
than commit-activity because it handles multiple `gh pr` subcommands
(`create`, `merge`, `close`) with different payload shapes and emits
distinct event types per subcommand.
