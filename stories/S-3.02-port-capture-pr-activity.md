# S-3.02: Port capture-pr-activity to WASM

**Epic:** E-3 — WASM Port: High-Value Hooks
**Phase:** Phase 3 — High-value port to native WASM
**Milestone:** `1.0.0-rc.1`
**Tier:** E
**Blocks:** S-4.07, S-4.08
**Blocked by:** S-2.08, S-3.04
**Estimated effort:** M

## Narrative
- **As a** factory operator
- **I want to** have `capture-pr-activity` running as native WASM
- **So that** PR events (create, merge, close) are captured with typed, structured data

## Goal

Rewrite `capture-pr-activity` from bash to native WASM. More complex
than commit-activity because it handles multiple `gh pr` subcommands
(`create`, `merge`, `close`) with different payload shapes and emits
distinct event types per subcommand.

## Acceptance criteria

- [ ] `capture-pr-activity.wasm` compiled and registered
- [ ] Handles `gh pr create`, `gh pr merge`, `gh pr close` subcommands
- [ ] Emits `pr.created`, `pr.merged`, `pr.closed` events with PR number, title, URL, branch fields
- [ ] Handles non-PR bash commands gracefully (no-op)
- [ ] `hooks-registry.toml` entry updated to native WASM
- [ ] Existing bats tests for PR capture still pass

| Component | Module | Pure/Effectful |

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | `gh pr create` fails | Emit `pr.create_failed`; return Continue |
