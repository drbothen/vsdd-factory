# S-1.03: hook-sdk crate (macro, types, bindings)

**Epic:** E-1 — Dispatcher Foundation
**Phase:** Phase 1 — Dispatcher foundation
**Milestone:** `1.0.0-beta.1`
**Tier:** B.x
**Blocks:** S-2.01, S-3.01, S-3.02, S-3.03, and all future plugin authoring
**Blocked by:** S-1.01
**Estimated effort:** L

## Narrative
- **As a** plugin author
- **I want to** write `#[hook] fn on_hook(payload: HookPayload) -> HookResult` and have the SDK handle the ABI
- **So that** I never need to touch raw WASI FFI or manage stdin/stdout manually

## Goal

Design and implement the `vsdd-hook-sdk` crate — the library every WASM
plugin depends on to write hooks ergonomically. Provides the `#[hook]`
proc-macro, the `HookPayload` / `HookResult` / `Event` types, and the
host-function import declarations.

## Acceptance criteria

- [ ] `vsdd-hook-sdk` compiles as a library with `wasm32-wasip1` target
- [ ] `#[hook]` proc-macro generates the `extern "C" fn _start()` that the dispatcher invokes
- [ ] The macro-generated entry point reads stdin, deserializes into `HookPayload`, catches panics, invokes user's function, serializes `HookResult`
- [ ] `HookPayload` type matches the Claude Code stdin schema
- [ ] `HookResult` type with variants: `Continue`, `Block(reason: String)`, `Error(message: String)`
- [ ] Host function imports declared via `extern "C"` for: `emit_event`, `log`, `read_file`, `exec_subprocess`, `session_id`, `dispatcher_trace_id`, `plugin_root`, `plugin_version`, `cwd`, `env`
- [ ] `HOST_ABI_VERSION` constant matches the dispatcher's (S-1.04)
- [ ] `#[hook]` works on a trivial example (returns `Continue` always)
- [ ] Unit tests cover macro expansion, payload deserialization, result serialization, panic boundary

## Contracts

**`#[hook]` macro user-facing contract:**

```rust
use vsdd_hook_sdk::{hook, HookPayload, HookResult};

#[hook]
pub fn on_hook(payload: HookPayload) -> HookResult {
    HookResult::Continue
}
```

**`HOST_ABI_VERSION` constant:**

```rust
pub const HOST_ABI_VERSION: u32 = 1;
```

**`HookPayload` schema:**

```rust
pub struct HookPayload {
    pub event_name: String,
    pub tool_name: String,
    pub session_id: String,
    pub dispatcher_trace_id: String,
    pub tool_input: serde_json::Value,
    pub tool_response: Option<serde_json::Value>,
}
```

## Test plan

- Unit: macro expansion produces expected code (check with `trybuild` or `macrotest`)
- Unit: `HookPayload` deserializes from representative JSON fixtures
- Unit: `HookResult` serializes correctly per variant
- Integration: compile `examples/hello-hook` to `wasm32-wasip1`, run under wasmtime
- Panic boundary: a plugin that panics must exit cleanly

## Notes

The `#[hook]` macro is the entire ergonomic story. Proc-macro crates must be
standalone library crates with `proc-macro = true`.

Panic handling: use `std::panic::catch_unwind` inside the macro-generated `_start`.
