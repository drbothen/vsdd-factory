**Complexity note (Low — 3 pts):** session-learning is the simplest Stop hook.
It performs a pure append-only write to `.factory/sidecar-learning.md` with an

if not; (b) create `sidecar-learning.md` with header if absent; (c) append one line.
No subprocess execution needed in the WASM port. binary_allow can be empty (or `[]`)
after migration — contrast with the legacy adapter which listed `["bash"]` only.

## Narrative

- **As a** vsdd-factory operator running Claude Code sessions
- **I want** session-learning to execute as a native WASM plugin instead of through the legacy-bash-adapter
- **So that** session-end markers are appended to `.factory/sidecar-learning.md` on all
  platforms (including Windows without git-bash), with lower latency, no adapter
  indirection, and no external binary dependency (`bash` is no longer needed)

## Goal

Port `plugins/vsdd-factory/hooks/session-learning.sh` to a native Rust WASM crate at
`crates/hook-plugins/session-learning/`. The hook fires on the Stop event (session end),
checks for `.factory/` directory existence (exits 0 if absent), creates
`.factory/sidecar-learning.md` with a header if it does not exist, and appends a single
line: `- Session ended at <ISO8601-UTC> (awaiting /session-review)`.

Behavior parity is the sole requirement (E-8 D-2): the Rust implementation must reproduce
every postcondition in BC-7.03.076, BC-7.03.077, and BC-7.03.078. No behavior change is
permitted during the port. After the port, the `.sh` file and the hooks.json command entry
for this hook are deleted, and the registry entry updated to reference the native `.wasm`.

**Key simplification:** session-learning does NOT call `bin/emit-event` — the bash source
uses only `date` and file I/O. The WASM port replaces `date` with Rust's `chrono` or
`std::time` and file writes with the `host::write_file` host fn (or equivalent Rust std
path if the WASM module can access the CWD filesystem). No `host::emit_event` calls are
needed. This makes session-learning the lowest-complexity Tier 1 port.

## Acceptance Criteria

### AC-001 (traces to BC-7.03.076 postcondition 1)
WASM crate `crates/hook-plugins/session-learning/` exists with `Cargo.toml` targeting
`wasm32-wasi`, implements the `vsdd_hook_sdk` hook interface, and builds successfully
via `cargo build --target wasm32-wasi`. Registry entry in `hooks-registry.toml` is
updated: `plugin = "hook-plugins/session-learning.wasm"`, `script_path` removed,
`shell_bypass_acknowledged` removed. Event binding preserved: `event = "Stop"`,
`priority = 910`, `on_error = "continue"`. The `binary_allow` list in
`[hooks.capabilities.exec_subprocess]` is empty or the `exec_subprocess` section is
removed entirely (session-learning requires no subprocess invocations post-port).

### AC-002 (traces to BC-7.03.076 invariant 1)
`hooks.json` command entry for `session-learning` is deleted from all platform-specific
`hooks.json.*` files and the template. `plugins/vsdd-factory/hooks/session-learning.sh`
is deleted from the repository.

### AC-003 (traces to BC-7.03.077 postcondition 1)
When `.factory/` directory exists and `sidecar-learning.md` does not exist: WASM plugin
creates `.factory/sidecar-learning.md` with the header block:
```
# Sidecar Learning

Session-end markers for the VSDD factory. Run /session-review to synthesize.

```
Then appends: `- Session ended at <ISO8601-UTC> (awaiting /session-review)\n`
(where `<ISO8601-UTC>` matches format `%Y-%m-%dT%H:%M:%SZ`, e.g.
`2026-04-30T14:25:00Z`). Exits 0.

### AC-004 (traces to BC-7.03.078 postcondition 1)
When `.factory/` directory does not exist: WASM plugin exits 0 immediately without
creating any files or writing to stderr. No error output. This guard must be the first
check in the plugin body (before any file I/O attempts).

### AC-005 (traces to BC-7.03.077 postcondition 1 + BC-7.03.076 postcondition 1)
When `.factory/` exists and `sidecar-learning.md` already has prior content: WASM
plugin appends exactly one new line per invocation (does NOT overwrite or re-add the
header). Multiple consecutive Stop firings each add exactly one marker line. Bats parity
test at `tests/integration/hooks/session-learning.bats` verifies: (a) first invocation
creates file + header + one marker; (b) second invocation appends one more marker, no
duplicate header; (c) invocation without `.factory/` exits 0, no file created; (d) WASM
warm invocation time compared to S-8.00 baseline; result within 20% regression ceiling
per E-8 epic AC-7. All bats tests pass.

### AC-006 (traces to BC-7.03.076 invariant 2)
`bin/emit-event` is NOT called by session-learning (never was — bash source uses only
`date` and bash file-append). Verify: no reference to `bin/emit-event` or
`host::emit_event` in `crates/hook-plugins/session-learning/`. The `bin/emit-event`
binary is NOT removed (E-8 D-10 — deferred to S-8.28).

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | `.factory/` directory exists but is not writable | WASM plugin attempts write; if `host::write_file` returns error, exit 0 (non-blocking per on_error=continue; safe to fail per bash precedent — bash `>>` would fail silently) |
| EC-002 | `sidecar-learning.md` exists but contains only a partial header (no trailing blank line) | WASM plugin appends the marker line without re-adding the header; file integrity is preserved |
| EC-003 | Stop event fires during rapid consecutive sessions (< 1s apart) | Each invocation appends a distinct timestamp line; no deduplication required; ISO-8601 precision to seconds is sufficient |
| EC-004 | `sidecar-learning.md` is very large (many previous sessions) | Append-only; no read of full file needed before appending; WASM implementation must not buffer full file into memory |
| EC-005 | WASM hook receives Stop JSON envelope from dispatcher (Stop event may carry session metadata) | session-learning does NOT parse the stdin envelope — it ignores stdin entirely (same as bash: no `cat` or JSON parsing in source). WASM port should read and discard stdin or skip reading entirely |
