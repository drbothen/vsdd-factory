# S-5.06: Semver commitment documentation

**Epic:** E-5 — New Hook Events and 1.0.0 Release
**Phase:** Phase 5 — New Claude Code hook events
**Milestone:** `1.0.0`
**Tier:** G
**Blocks:** S-5.07
**Blocked by:** S-4.08
**Estimated effort:** S

## Narrative
- **As a** plugin author or operator
- **I want to** know exactly what is stable in v1.0 and what the breaking change policy is
- **So that** I can build against vsdd-factory without fear of surprise breakage

## Goal

Write the public semver commitment doc at `docs/guide/semver-commitment.md`.
States exactly what's stable after 1.0, what isn't, and how we handle
breaking changes.

## Acceptance criteria

- [ ] `docs/guide/semver-commitment.md` created with full content
- [ ] Section: What's stable (hook-sdk ABI, registry schema, hooks.json format, event type namespaces)
- [ ] Section: What's NOT stable (internal JSONL format, dispatcher invocation args)
- [ ] Section: Breaking change policy (major version bump, migration guide required)
- [ ] Section: Plugin backward compat policy (host ABI version guarantees)
- [ ] Cross-linked from `docs/guide/index.md`
