Anchoring to CAP-014 ("Decompose product specs into verified behavioral contracts") per capabilities.md §CAP-014. BC-8.26.006 (story-completeness check 5: user-facing-docs deliverable) is the SS-08 rule that directly governs this docs deliverable.

## Narrative
- **As a** plugin author or operator
- **I want to** know exactly what is stable in v1.0 and what the breaking change policy is
- **So that** I can build against vsdd-factory without fear of surprise breakage

## Goal

Write the public semver commitment doc at `docs/guide/semver-commitment.md`.
States exactly what's stable after 1.0, what isn't, and how we handle
breaking changes.
