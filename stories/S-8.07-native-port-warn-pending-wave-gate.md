## Capability Anchor Justification

**Primary anchor:** CAP-022 ("Port hook plugins from bash to native WASM") per
`domain-spec/capabilities.md` §CAP-022. S-8.07 ports `warn-pending-wave-gate.sh` to
native WASM, eliminating legacy-bash-adapter indirection for this session-end safety net.

**Stretch-Anchor Disclosure:** S-8.07 anchors to CAP-022 but spans two subsystems:
- **SS-07 Hook Bash Layer** owns `warn-pending-wave-gate.sh` and its BC-7.03.091/092
  behavioral contracts. SS-07 is the source-of-truth for the hook's behavior spec.
- **SS-01 Hook Dispatcher Core** owns the hooks-registry.toml routing and the host fn
  surface (`vsdd_hook_sdk::host::*`) used by the native crate.

Both subsystem anchors verified against ARCH-INDEX Subsystem Registry:
- SS-01 Hook Dispatcher Core: canonical name confirmed.
- SS-07 Hook Bash Layer: canonical name confirmed.
(Per POLICY 5 anchor justification requirement + POLICY 6 architecture-is-source-of-truth.)

**Complexity note (Low — 3 pts):** warn-pending-wave-gate is a read-only advisory hook.
It: (a) checks for `.factory/wave-state.yaml`, exits 0 if absent; (b) checks python3
availability, exits 0 if absent (bash version); (c) parses wave-state.yaml to find
waves with `gate_status: pending`; (d) if any pending waves found, emits
`hook.block severity=warn` via `bin/emit-event` and writes a WAVE GATE REMINDER to
stderr; (e) always exits 0.

The primary porting consideration: the bash source uses `python3` for YAML parsing.
The WASM port must use a pure Rust YAML library (e.g., `serde_yaml`) instead of a
subprocess. This is a straightforward substitution — the YAML structure is simple
(`waves.{name}.gate_status`). No new host fns are required (read_file + emit_event).

**Dependency anchor:** S-8.07 depends on S-8.00. S-8.07 can be implemented in
parallel with S-8.01..S-8.06 within W-15; ordering within the tier is informational.
S-8.07 depends_on=[S-8.00] because S-8.00 confirms BC-7.03.091/092 are Spec-Current
before implementation begins.

## Narrative

- **As a** vsdd-factory operator ending Claude Code sessions
- **I want** warn-pending-wave-gate to execute as a native WASM plugin instead of through the legacy-bash-adapter
- **So that** the session-end wave gate reminder runs on all platforms (including Windows
  without python3), with lower latency, no adapter indirection, and no python3 dependency

## Goal

Port `plugins/vsdd-factory/hooks/warn-pending-wave-gate.sh` to a native Rust WASM crate
at `crates/hook-plugins/warn-pending-wave-gate/`. The hook fires on the Stop event,
reads `.factory/wave-state.yaml` (exits 0 if absent), parses all waves looking for
`gate_status: pending`, and if any are found: calls `host::emit_event` with
`type=hook.block hook=warn-pending-wave-gate matcher=Stop reason=pending_wave_gate_at_session_end severity=warn pending_waves=<comma-joined>`,
then writes a WAVE GATE REMINDER block to stderr listing each pending wave name and
invocation hints. Always exits 0.

Behavior parity is the sole requirement (E-8 D-2). The key change vs bash: `python3`
YAML parsing is replaced with `serde_yaml` (or equivalent pure Rust). The `_emit()`
function calls are replaced with `host::emit_event`. All output format, field names,
and stderr message text must be identical to the bash version.

## Acceptance Criteria

### AC-001 (traces to BC-7.03.091 postcondition 1)
WASM crate `crates/hook-plugins/warn-pending-wave-gate/` exists with `Cargo.toml`
targeting `wasm32-wasi`, implements the `vsdd_hook_sdk` hook interface, and builds
successfully via `cargo build --target wasm32-wasi`. Registry entry in
`hooks-registry.toml` is updated: `plugin = "hook-plugins/warn-pending-wave-gate.wasm"`,
`script_path` removed, `shell_bypass_acknowledged` removed. Event binding preserved:
`event = "Stop"`, `priority = 920`, `on_error = "continue"`.

### AC-002 (traces to BC-7.03.091 invariant 1)
`hooks.json` command entry for `warn-pending-wave-gate` is deleted from all
platform-specific `hooks.json.*` files and the template.
`plugins/vsdd-factory/hooks/warn-pending-wave-gate.sh` is deleted from the repository.

### AC-003 (traces to BC-7.03.092 postcondition 1)
When `.factory/wave-state.yaml` exists and at least one wave has `gate_status: pending`:
WASM plugin calls `host::emit_event` with:
```
type=hook.block hook=warn-pending-wave-gate matcher=Stop
reason=pending_wave_gate_at_session_end severity=warn
pending_waves=<comma-joined wave names>
```
Then writes to stderr:
```
\nWAVE GATE REMINDER:\n
  - <wave1> gate is pending. Run the gate before starting the next wave.\n
  [one line per pending wave]
\n
  Invoke /vsdd-factory:wave-gate or update .factory/wave-state.yaml\n
  with gate_status: passed (after running checks) or deferred (with rationale).\n
```
Exits 0.

### AC-004 (traces to BC-7.03.091 postcondition 2)
When `.factory/wave-state.yaml` is absent, or when no wave has `gate_status: pending`,
or when the YAML file is empty/malformed: WASM plugin exits 0 with no output and no
emit_event call. All early-exit paths must be silent (no stderr, no event).

### AC-005 (traces to BC-7.03.092 postcondition 1 + BC-7.03.091 postcondition 1)
Bats parity test at `tests/integration/hooks/warn-pending-wave-gate.bats` covers:
(a) wave-state.yaml with one pending wave → exit 0 + expected stderr REMINDER text
    + comma-joined pending_waves event field;
(b) wave-state.yaml with two pending waves → exit 0 + both wave names in REMINDER;
(c) wave-state.yaml with no pending waves (all passed) → exit 0 + no stderr;
(d) wave-state.yaml absent → exit 0 + no output;
(e) malformed YAML → exit 0 + no output (graceful parse error handling);
(f) WASM warm invocation time compared to S-8.00 baseline; within 20% regression ceiling.
All bats tests pass.

### AC-006 (traces to BC-7.03.091 invariant 2)
The python3 subprocess call in the bash source is replaced with pure Rust YAML parsing
(serde_yaml or equivalent). Verify: no reference to `python3` or subprocess invocation
in `crates/hook-plugins/warn-pending-wave-gate/`. The `binary_allow` list in
`[hooks.capabilities.exec_subprocess]` may retain or remove `["bash"]`; if python3 is
no longer needed, the exec_subprocess section can be removed or minimized. Do NOT
retain `binary_allow = ["bash"]` if no subprocess is required post-port.

### AC-007 (traces to BC-7.03.092 postcondition 1)
`bin/emit-event` calls from the bash `_emit()` function are replaced with
`host::emit_event` Rust calls in the WASM crate. Verify: no reference to
`bin/emit-event` remains in `crates/hook-plugins/warn-pending-wave-gate/`.
`bin/emit-event` binary is NOT removed (E-8 D-10 — deferred to S-8.28).

| EC-006 | host::emit_event call fails (host fn unavailable) | Continue to stderr output; exit 0. Event emission is best-effort (same as bash `\|\| true` fallback) |
