# S-15.14: validate-dispatch-advance WASM hook

**Epic:** E-12 — Context Resolvers + Platform Stories (brownfield-backfill wave, S-15.03 PRIORITY-A M2)
**Delivery order:** M2 Wave-4 (final of four M2 WASM hook stories; completes M2, unblocks M3)
**Depends on:** S-15.07 (merged), S-15.11 (merged), S-15.09 (merged) — parity sibling crates as
  structural reference; zero formal blocking imports; serial per architect wave plan Q3
**Blocks:** none formal (M3 stories unblock once M2 is complete, not from this story specifically)
**Priority:** P1 (closes D-440(a)+D-441(a)+D-442(a)+D-443(a)+D-439(b)+D-441(b)+D-451(c))

## Narrative

- **As a** state-manager or agent writing a STATE.md dispatch-advance update in a fix burst
- **I want to** receive an immediate PostToolUse block when the `current_step:` frontmatter field
  I just wrote contains forbidden meta-commentary patterns, is missing any of the 4 required
  index version citations, has a trajectory-tail with the wrong component count, or has a stale
  D-chain cite — and when an INDEX.md adversary-pass row I wrote has the wrong column count
- **So that** the recurring class of dispatch-advance structural violations discovered by the
  adversary in every F5 engine-discipline pass (forbidden meta-commentary labels in
  `current_step:`, missing index cites, trajectory-tail cardinality errors, stale D-chain cites,
  and malformed INDEX.md adversary-pass rows) is caught mechanically at write time rather than
  discovered by the adversary N bursts later

## Why This Exists

The F5 engine-discipline cycle (passes 59–74+) generated a persistent class of adversary findings
around the `current_step:` frontmatter field in STATE.md. D-440(a)+D-441(a)+D-442(a) codified
the forbidden meta-commentary prohibition. D-439(b) established the 4-index version cite
requirement. D-451(c) established trajectory-tail LENGTH=4 in `current_step:`. D-443(a)
established D-chain cite currency. D-441(b) established the 5-column schema for INDEX.md
adversary-pass rows (within the `## Adversarial Reviews` h2 section).

Manual adherence never held — every cycle produced at least one `current_step:` structural
finding. This hook makes the entire dispatch-advance structural-correctness failure class
impossible at PostToolUse time.

## Goal

Deliver a production-grade WASM hook (`validate-dispatch-advance`) that PostToolUse-blocks any
Edit/Write to `STATE.md` whose `current_step:` frontmatter field contains:
- Forbidden meta-commentary patterns (`META-LEVEL-\d+ WATCH`, `self-app TEST`, `expected verdict`)
- Missing 4-index version citations (BC-INDEX v, VP-INDEX v, STORY-INDEX v, ARCH-INDEX v)
- Trajectory-tail with component count other than exactly 4 (`→(\d+)` matches)
- Stale D-chain cite (no `D-\d+` reference in `current_step:` at all, or max extracted D-NNN
  integer below the highest D-NNN visible in STATE.md body)

And PostToolUse-blocks any Edit/Write to `INDEX.md` containing adversary-pass rows (under the
`## Adversarial Reviews` h2 section, in a 5-col-header section) with column count other than 5
(pipe count other than 6).

## Behavioral Contract

### BC-5.39.006: validate-dispatch-advance WASM hook MUST block on forbidden meta-commentary in current_step, missing 4-index version citations, trajectory-tail cardinality violations, stale D-chain cites in STATE.md, and non-5-column adversary-pass rows in the INDEX.md ## Adversarial Reviews section

**Subsystem:** SS-05 Pipeline Orchestration (Engine Governance capability class within Pipeline
Orchestration subsystem — dispatch-advance structural validation automation sub-capability)
**Capability:** E-12 Engine Governance
**BC ID:** BC-5.39.006
**File:** `.factory/specs/behavioral-contracts/ss-05/BC-5.39.006.md`

#### Preconditions (summary — authoritative form in BC-5.39.006.md)

1. A PostToolUse Edit/Write event has fired on a file whose `file_name` path component is exactly
   `STATE.md` (path-component-strict matching: `Path::new(file_path).file_name() == Some("STATE.md")`)
   OR on a file whose `file_name` path component is exactly `INDEX.md` (same guard:
   `Path::new(file_path).file_name() == Some("INDEX.md")`).
2. The dispatcher has invoked the `validate-dispatch-advance` WASM plugin with the write payload.
3. The file content is read via `host::read_file` (filesystem-authoritative; NOT payload field).
4. `host::read_file` available with `max_bytes = 524288` (512 KiB; matching BC-5.39.005 cap) and
   `timeout_ms = 2000` per call. Registry-level timeout: `timeout_ms = 5000`.

#### Postconditions (summary — authoritative form in BC-5.39.006.md)

**STATE.md arm (file_name == "STATE.md"):**

1. If ALL hold — no forbidden meta-commentary in `current_step:`, all 4 index version patterns
   present, trajectory-tail LENGTH=4, D-chain cite not stale — emit `HookResult::Continue`.
2. If `current_step:` matches forbidden meta-commentary regex
   `META-LEVEL-\d+ WATCH|self-app TEST|expected verdict`, emit `HookResult::BlockWithFix` naming
   the offending pattern, citing D-440(a)+D-441(a)+D-442(a).
3. If any of the 4 index version patterns is absent from `current_step:`, emit
   `HookResult::BlockWithFix` naming each missing cite, citing D-439(b).
4. If trajectory-tail in `current_step:` does not contain exactly 4 `→(\d+)` matches, emit
   `HookResult::BlockWithFix` naming actual count, required count (4), citing D-451(c).
5. If D-chain cite in `current_step:` is stale (terminal integer below latest D-NNN in STATE.md),
   emit `HookResult::BlockWithFix` naming the stale cite, citing D-443(a).
6. Multiple violations in one write produce a single `HookResult::BlockWithFix` enumerating all.
7. If `host::read_file` returns an error for STATE.md, emit `HookResult::Continue` + log_warn —
   fail-open.

**INDEX.md arm (file_name == "INDEX.md"):**

8. If every in-scope adversary-pass row in the `## Adversarial Reviews` h2 section is a 5-column
   table row (6 pipe chars), OR if the section uses a 4-column header (grandfathered historical
   schema), emit `HookResult::Continue`.
9. If any in-scope adversary-pass row in a 5-column-header `## Adversarial Reviews` section has
   column count other than 5 (pipe count other than 6), emit `HookResult::BlockWithFix` naming
   the row (by h2 context or line position), actual column count, required count (5), citing
   D-441(b)/D-442(b).
10. If `host::read_file` returns an error for INDEX.md, emit `HookResult::Continue` + log_warn —
    fail-open.

#### Invariants (summary — authoritative form in BC-5.39.006.md)

1. Hook NEVER writes to any file. Read-only post-write validator.
2. Hook fires PostToolUse only — never prevents a write; signals AFTER write completes.
3. Path-component-strict matching on BOTH trigger paths (both guards use `Path::file_name()`,
   NOT `ends_with`).
4. Forbidden meta-commentary regex anchored to `current_step:` frontmatter field only — NOT
   applied to full STATE.md body content.
5. All 4 index version patterns: literal strings `BC-INDEX v`, `VP-INDEX v`, `STORY-INDEX v`,
   `ARCH-INDEX v` (each followed by version token). All 4 must be present.
6. Trajectory-tail detection: regex `→(\d+)` applied globally to `current_step:` value; match
   count must equal exactly 4.
7. D-chain currency: fail-open design — extract ALL `D-(\d+)` integers from `current_step:`
   value via global regex; take max_cited; if no `D-\d+` reference present at all, that is a
   violation; if max_cited < max_in_file (highest D-NNN visible anywhere in STATE.md), that is
   stale; fail-open if max_cited >= max_in_file (prevents false-positive blocks on in-progress
   writes). The literal prefix `D-382..` is NOT required.
8. Adversary-pass row column detection: applies ONLY to rows under `## Adversarial Reviews` h2
   heading (exact heading text). Within that section: if header row pipe count is 5 (4-column
   schema), all rows grandfathered — skip column validation. If header row pipe count is 6
   (5-column schema), enforce 6-pipe (5-column) compliance on all non-separator data rows.
   Pipe count formula: 1 leading + 4 internal separators + 1 trailing = 6 pipes for 5 columns.
   Separator rows (`| --- |` pattern) excluded.
9. All `host::read_file` calls are fail-open.
10. All byte-index slice expressions with possible multi-byte UTF-8 input (em-dash, en-dash, NBSP
    in STATE.md narrative text) MUST use `is_char_boundary()` guards before slicing.

  ```
  fn extract_current_step(content: &str) -> Option<&str>
  ```
  Extract the `current_step:` frontmatter value. YAML frontmatter is bounded by `---` delimiters.
  Find the line starting with `current_step:` within the frontmatter block; return the value
  (single-quoted or unquoted). Guard byte-index slice expressions with `is_char_boundary()` where
  multi-byte UTF-8 input is possible.

  ```
  fn check_forbidden_meta_commentary(current_step_value: &str) -> Option<Violation>
  ```
  Regex: `META-LEVEL-\d+ WATCH|self-app TEST|expected verdict`. If matched: return
  Some(Violation) with `cited_raw` set to the matched substring, `decision_refs` citing
  `D-440(a)+D-441(a)+D-442(a)`.

  ```
  fn check_index_version_cites(current_step_value: &str) -> Option<Violation>
  ```

  ```
  fn check_trajectory_tail_length(current_step_value: &str) -> Option<Violation>
  ```

  != 4: return Some(Violation) naming actual count and required count 4, `decision_refs` citing

  ```
  fn check_d_chain_currency(content: &str, current_step_value: &str) -> Option<Violation>
  ```
  Extract ALL `D-(\d+)` integers from `current_step_value` via global regex `D-(\d+)`. Take
  max_cited = maximum integer found. If no match at all: return Some(Violation) naming absent cite,
  `decision_refs` citing `D-443(a)`. If max_cited found: scan full STATE.md `content` for all
  `D-(\d+)` matches; find max_in_file = maximum integer found anywhere in file. If max_cited <
  max_in_file: return Some(Violation) naming `max_cited` as stale vs latest `max_in_file`. If
  max_cited >= max_in_file: return None (current; fail-open per invariant 7). The literal prefix
  `D-382..` is NOT required — prose forms such as `D-chain cite D-476` are valid.

  ```
  fn validate_state_md(content: &str) -> Vec<Violation>
  ```
  Orchestrates all 4 STATE.md checks: extract `current_step_value`, run all 4 checks, accumulate
  non-None results into Vec. Return Vec (may be empty for pass).

  ```
  fn validate_index_md(content: &str) -> Vec<Violation>
  ```
  Locate the `## Adversarial Reviews` h2 section (exact heading text). Extract lines between that
  heading and the next h2 (`^## `). Within that section: find the first non-separator `|` row
  (header row). If its pipe count is 5 (4-column schema), treat all rows as grandfathered-historical
  and return empty Vec. If its pipe count is 6 (5-column schema), for each subsequent non-separator
  data row line whose trimmed form begins and ends with `|`: count pipe characters. If count != 6
  (not 5 columns; formula: 1 leading + 4 internal separators + 1 trailing = 6 pipes for 5 columns):
  add Violation naming line text (truncated to 120 chars), actual column count (pipe_count - 1),
  required 5, `decision_refs` citing `D-441(b)/D-442(b)`. Skip separator rows (`\|\s*---` pattern).
  Return accumulated violations.
