## Capability Anchor Justification

Primary: CAP-002 ("Hook Claude Code tool calls with sandboxed WASM plugins") per capabilities.md §CAP-002. This story wires the `SessionStart` event type through the dispatcher to the `session-start.wasm` plugin, which emits `session.started` telemetry and triggers factory-health checks — directly exercising the sandboxed WASM plugin hook capability.

FR anchor: FR-046 ("New Claude Code lifecycle hook events: SessionStart/SessionEnd/WorktreeCreate/WorktreeRemove/PostToolUseFailure") per PRD §2.4. FR-046 is the canonical FR for all S-5 lifecycle event wiring stories. Resolves DRIFT-006.

Behavioral contracts: BC-4.04.001 (session.started event emission + payload schema), BC-4.04.002 (factory-health subprocess + fail-open), BC-4.04.003 (idempotency on duplicate SessionStart), BC-4.04.004 (hooks.json.template registration with once:true).

Verification: VP-065 (Session-Start Plugin Surface Invariant — integration test across all four BCs).

Cross-subsystem references (informational only, not this story's primary anchors): BC-1.02.005 (dispatcher SessionStart envelope parsing, SS-01); BC-1.01.001 (registry schema, SS-01).

## Narrative
- **As a** factory operator
- **I want to** receive a session.started event at the beginning of each Claude Code session
- **So that** I can trigger factory-health checks and track session-level telemetry

## Goal

Wire the `SessionStart` event type through the dispatcher. Emits
`session.started` with session_id, runs a factory-health brief,
checks tool dependencies. First session-level event.

## Acceptance criteria

- [ ] Dispatcher handles `SessionStart` event type from Claude Code
  - _trace: BC-1.02.005 (SS-01 envelope parsing tolerates missing tool_name on lifecycle events) + BC-4.04.004 (hooks.json.template SessionStart registration routes to session-start.wasm)_
- [ ] `session.started` event emitted with session_id, factory_version, plugin_count, activated_platform, factory_health, tool_deps, timestamp
  - _trace: BC-4.04.001 (session.started payload schema + emit_event postcondition)_
- [ ] Factory-health brief triggered on session start; plugin still emits session.started if factory-health fails
  - _trace: BC-4.04.002 (exec_subprocess factory-health --brief + fail-open EC-001)_
- [ ] `hooks.json.template` updated with SessionStart handler using once:true
  - _trace: BC-4.04.004 (hooks.json.template SessionStart registration with once:true)_
- [ ] Integration test: send SessionStart payload; verify session.started in file sink; verify idempotency on duplicate SessionStart
  - _trace: VP-065 (Session-Start Plugin Surface Invariant — integration test across BC-4.04.001–004)_
