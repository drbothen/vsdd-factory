# S-5.01: SessionStart hook wiring

**Epic:** E-5 — New Hook Events and 1.0.0 Release

## Capability Anchor Justification

Primary: CAP-002 ("Hook Claude Code tool calls with sandboxed WASM plugins") per capabilities.md §CAP-002. This story wires the `SessionStart` event type through the dispatcher, extending the hook routing capability to session lifecycle events. The session-start plugin emits `session.started` and triggers factory-health checks.

FR anchor: FR-007 ("Legacy hook routing compatibility + dispatcher main") per PRD §2.1, extended to cover new event types. No dedicated FR exists for SessionStart/SessionEnd/WorktreeCreate/PostToolUseFailure wiring — this is tracked as DRIFT-006 in PRD §11. Extends FR-007 scope; a new FR-045 or FR-046 for "New Claude Code lifecycle hook events (SS-04)" is proposed below.

Cross-subsystem note: BC-1.02.005 ("SessionStart parses with tool_name defaulting to ''") at SS-01 is the closest existing BC for dispatcher-side SessionStart envelope handling. BC-1.01.NNN (registry schema, once:true) applies to the hooks.json template update. These are cross-subsystem references — the story's SS-04 plugin behavior has no existing BCs; v1.1 candidates are pre-emptively logged below.

## Narrative
- **As a** factory operator
- **I want to** receive a session.started event at the beginning of each Claude Code session
- **So that** I can trigger factory-health checks and track session-level telemetry

## Goal

Wire the `SessionStart` event type through the dispatcher. Emits
`session.started` with session_id, runs a factory-health brief,
checks tool dependencies. First session-level event.

## Acceptance criteria

- [ ] Dispatcher handles `SessionStart` event type from Claude Code
  - _trace: BC-1.02.005 postcondition 1 (SessionStart parses cleanly via HookPayload::from_bytes) at SS-01 scope; dispatcher routing of new event type extends FR-007_ [process-gap: no SS-04 BC contracts the dispatcher wiring for SessionStart → session-start plugin — v1.1 BC candidate]
- [ ] `session.started` event emitted with session_id, plugin_version, platform
  - _trace: CAP-002 ("WASM plugins run in wasmtime … a plugin can … allow it with sub-10ms overhead")_ [process-gap: no SS-04 BC contracts the session.started event schema — v1.1 BC candidate]
- [ ] Factory-health brief triggered on session start (invokes `/vsdd-factory:factory-health`)
  - _trace: story EC-001 (factory-health check fails → log warning; session.started still emitted)_ [process-gap: no SS-04 BC contracts the factory-health invocation on session start — v1.1 BC candidate]
- [ ] `hooks.json` template updated with SessionStart handler
  - _trace: story architecture compliance (BC-1.01 registry schema, once:true at SS-01 scope)_ [process-gap: no SS-04 BC contracts hooks.json template inclusion for SessionStart — v1.1 BC candidate]

  - _trace: BC-1.02.005 (SessionStart parses; integration test validates the end-to-end path)_
