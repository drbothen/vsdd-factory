## Capability Anchor Justification

Primary: CAP-002 ("Hook Claude Code tool calls with sandboxed WASM plugins") per capabilities.md §CAP-002. This story wires the `SessionStart` event type through the dispatcher to the `session-start-telemetry.wasm` plugin, which emits `session.started` telemetry and triggers factory-health checks — directly exercising the sandboxed WASM plugin hook capability.

FR anchor: FR-046 ("New Claude Code lifecycle hook events: SessionStart/SessionEnd/WorktreeCreate/WorktreeRemove/PostToolUseFailure") per PRD §2.4. FR-046 is the canonical FR for all S-5 lifecycle event wiring stories. Resolves DRIFT-006.

Behavioral contracts: BC-4.04.001 (session.started event emission + payload schema; activated_platform read via existing read_file host fn + path_allow per BC-4.04.005), BC-4.04.002 (factory-health subprocess + fail-open + 5000ms timeout), BC-4.04.003 (idempotency on duplicate SessionStart — delegated to Claude Code Layer 1 once:true per BC-4.04.004 invariant 1; plugin is unconditionally stateless), BC-4.04.004 (hooks.json.template registration routing to dispatcher binary with once:true and async:true per ADR-011 Layer 1), BC-4.04.005 (hooks-registry.toml registration with [hooks.capabilities.read_file] table, [hooks.capabilities.exec_subprocess] table, and timeout_ms = 8000 per ADR-011 Layer 2).

S-5.01 anchors purely to SS-04 plugin ecosystem. Cross-subsystem dependencies (read_file host fn for activated_platform; Claude Code Layer 1 once-discipline) use existing canonical patterns — no new SS-01 BCs introduced. Pass-4 architect retired BC-1.10.001 (over-engineered new host fn — replaced by existing read_file host fn capability declaration in BC-4.04.005) and BC-1.10.002 (over-engineered dispatcher dedup — replaced by Layer 1 once-discipline in BC-4.04.004 invariant 1).

Verification: VP-065 (Session-Start Plugin Surface Invariant — integration test across all five BCs: BC-4.04.001–005).

Informational: BC-1.02.005 (dispatcher SessionStart envelope parsing, SS-01) is a precondition for all five SS-04 BCs but does not anchor S-5.01's behavior — the dispatcher already handles SessionStart envelopes correctly per Decision C (CAP-002 widened to cover lifecycle events).

Pass-5 verified the dual-routing-tables architecture (Layer 1 hooks.json.template harness routing + Layer 2 hooks-registry.toml dispatcher routing) with strict timeout hierarchy 5000ms (subprocess) < 8000ms (dispatcher) < 10000ms (harness).

Pass-7 split the emit_event surface contract: BC-1.05.012 was widened to cover both filter (drop reserved) AND enrichment (host-set 4 HostContext fields) halves; ADR-011 documents the 4+4+6 field provenance.

## Narrative
- **As a** factory operator
- **I want to** receive a session.started event at the beginning of each Claude Code session
- **So that** I can trigger factory-health checks and track session-level telemetry

## Goal

Wire the `SessionStart` event type through the dispatcher. Emits `session.started` with
session_id, runs a factory-health brief, checks tool dependencies. First of five lifecycle
events in PRD FR-046 (SessionStart, SessionEnd, WorktreeCreate, WorktreeRemove,
PostToolUseFailure); blocks v1.0.0 release gate (S-5.07) per epic E-5 sequence.

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | Factory-health check fails, times out (5000ms), or returns CAPABILITY_DENIED | Log warning; session.started still emitted with factory_health = "unknown" |
| EC-002 | Second SessionStart in same session — Layer 1 once:true bypassed (e.g., manual SessionStart injection during testing) | Plugin emits multiple session.started events; this is operator-observable and acceptable at the plugin layer. Idempotency in normal operation is guaranteed at Layer 1 (BC-4.04.004 invariant 1) |
| EC-003 | read_file host fn fails for activated_platform read (file missing, parse error, key absent, or capability denied) | Plugin emits session.started with activated_platform = "unknown" per BC-4.04.001 EC-002 (read_file failure modes); does not abort |
