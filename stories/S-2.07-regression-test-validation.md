# S-2.07: Regression test suite validation

**Epic:** E-2 — Legacy Adapter and Beta Release
**Phase:** Phase 2 — Legacy bash adapter
**Milestone:** `1.0.0-beta.1`
**Tier:** C
**Blocks:** S-2.08

**Estimated effort:** M

## Narrative
- **As a** release engineer preparing 1.0.0-beta.1
- **I want to** validate that the full existing bats suite (1177+ tests) passes under the new dispatcher
- **So that** beta.1 ships with confirmed backward compatibility

## Goal

Validate that the full existing bats suite (1177+ tests) continues to
pass under the new dispatcher with every bash hook routed through
`legacy-bash-adapter`. This is the single most important regression
gate before beta.1.

## Acceptance criteria

- [ ] All 1177+ existing bats tests pass with the new dispatcher active
- [ ] `regression-v1.0.bats` (11 tests) covers end-to-end dispatcher flow
- [ ] Zero regressions introduced by the dispatcher chain vs. direct bash invocation
- [ ] Test runner CI job clearly labeled "regression-v1.0"
- [ ] Any test failures get root-cause analysis before blocking S-2.08
