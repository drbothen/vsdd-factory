# S-2.07: Regression test suite validation

**Epic:** E-2 — Legacy Adapter and Beta Release

## Capability Anchor Justification

Primary: CAP-002 ("Hook Claude Code tool calls with sandboxed WASM plugins") per capabilities.md §CAP-002.
Justification: S-2.07 is the end-to-end regression gate that validates every bash hook (30+) continues to fire via the WASM dispatcher's legacy-bash-adapter routing, which is exactly the behavioral guarantee CAP-002 defines. The 1177+ bats tests exercise the full dispatcher pipeline — BC-1.07.001 (all hooks fire) and BC-1.08.001/002 (exit code semantics) are the primary contracts under test. BC-1.07.002 (commit.made events) validates the dispatcher's real-world capture path.

FR anchor: FR-007 ("Legacy hook routing compatibility and dispatcher main") per PRD §2.1. FR-007 owns BC-1.07.001–BC-1.08.006 (10 BCs), and S-2.07 is the primary integration-validation story for this FR group.

## Behavioral Contracts

| BC ID | Title | Scope Reason |
|-------|-------|--------------|
| BC-1.07.001 | All 30+ existing bash hooks fire via legacy-bash-adapter on Linux/macOS | Primary regression invariant — AC-1 (1177+ bats pass) directly validates this postcondition |
| BC-1.07.002 | `commit.made` events fire reliably on real Claude Code git commit | Regression gate verifies real-harness PostToolUse capture still fires correctly |
| BC-1.08.001 | dispatcher exits 0 on registry/payload/engine errors (non-blocking) | Regression suite must not introduce new blocking failure modes on dispatcher errors |
| BC-1.08.002 | dispatcher exit code is 2 iff at least one block_intent recorded | Exit code contract validated via regression — any deviation is a regression |
