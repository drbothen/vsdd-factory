# S-1.08: sink-file driver

**Epic:** E-1 — Dispatcher Foundation
**Phase:** Phase 1 — Dispatcher foundation (feeds into Phase 4 sink system)
**Milestone:** `1.0.0-beta.1`
**Tier:** B.x
**Blocks:** S-2.07, S-4.07
**Blocked by:** S-1.01, S-1.07
**Estimated effort:** M

## Narrative
- **As a** factory operator
- **I want to** have events appended to JSONL files on disk automatically
- **So that** I can inspect the event stream without any external tooling

## Goal

Implement the `file` sink type — JSONL append to a configurable path.
This is the minimum-viable sink required for beta.1 and preserves the
current `.factory/logs/events-*.jsonl` behavior that existing tools depend on.

## Acceptance criteria

- [ ] `crates/sink-file` crate implements the `Sink` trait from `crates/sink-core`
- [ ] Events appended as newline-delimited JSON to the configured file path
- [ ] Daily rotation supported: file renamed to `<base>.<date>.jsonl` at midnight UTC
- [ ] File created on first write if missing; parent dirs auto-created
- [ ] `submit(event)` returns immediately (non-blocking); writes happen on dedicated thread
- [ ] `flush()` drains the queue and waits for all writes to complete
- [ ] Integration test: submit 1000 events, flush, verify all written to disk

## Architecture Mapping

| Component | Module | Pure/Effectful |

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | Disk full during write | Record `SinkFailure`, emit `internal.sink_error` |
| EC-002 | File path directory missing | Auto-create with `fs::create_dir_all` |
| EC-003 | Concurrent submits from multiple plugins | Thread-safe queue; no interleaved JSON |
