# S-2.08: 1.0.0-beta.1 release gate

**Epic:** E-2 — Legacy Adapter and Beta Release
**Phase:** Phase 2 — Legacy bash adapter (milestone boundary)
**Milestone:** `1.0.0-beta.1`
**Tier:** D
**Blocks:** S-3.01, S-3.02, S-3.03, S-3.04, S-4.01..S-4.07 (everything waiting on beta shakedown)
**Blocked by:** all S-0.x, S-1.x, S-2.01..S-2.07
**Estimated effort:** S

## Narrative
- **As a** release engineer
- **I want to** cut the `1.0.0-beta.1` tag after all gate criteria are met
- **So that** early adopters can begin shakedown while Tier E work starts in parallel

## Goal

Gate story — no new code. Verifies every beta.1 entry criterion is met,
then cuts the `1.0.0-beta.1` tag and ships to the marketplace.

## Acceptance criteria

- [ ] **(1)** `factory-dispatcher` builds clean on all 5 CI platforms
- [ ] **(2)** `vsdd-hook-sdk` 0.1.0 dry-run passes (real publish in S-2.05)
- [ ] **(3)** All 30+ existing hooks fire via `legacy-bash-adapter` on Linux/macOS. Full bats suite passes.
- [ ] **(4)** `commit.made` events fire reliably in a real Claude Code session
- [ ] **(5)** Dogfood: vsdd-factory's own repo uses the new dispatcher for at least one release cycle
- [ ] **(6)** Windows smoke test: dispatcher starts, `dispatcher.started` event appears in internal log
- [ ] **(7)** `/vsdd-factory:activate` correctly detects platform and writes the right `hooks.json`
- [ ] All stories S-0.x, S-1.x, S-2.01..S-2.07 closed
- [ ] Release notes drafted at `.github/release-notes/1.0.0-beta.1.md`
- [ ] CHANGELOG entry written
- [ ] `bump-version.sh 1.0.0-beta.1` runs successfully
- [ ] Release workflow produces a GH pre-release with bundled binaries
