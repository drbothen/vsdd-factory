# S-0.04: hooks.json.template + CI generation

**Epic:** E-0 — Infrastructure Prep
**Phase:** Phase 0 — Infrastructure prep
**Milestone:** `1.0.0-beta.1`
**Tier:** A (contract-frozen; binaries reference comes from S-1.x)
**Blocks:** S-2.03, S-2.06, S-2.08
**Blocked by:** none
**Estimated effort:** M

## Narrative
- **As a** CI/release engineer
- **I want to** maintain a single `hooks.json.template` that generates 5 platform variants
- **So that** platform-specific dispatcher binary paths never drift from the canonical template

## Goal

Establish the source-of-truth `hooks.json.template` format and a CI step
that renders it into 5 platform-specific `hooks.json.<platform>` variants.
The template uses placeholders that the generator substitutes with the
correct platform-specific dispatcher-binary path for each variant. CI
commits the generated variants as part of the release chore commit and
verifies there's no drift between template and variants on every push.

## Acceptance criteria

- [ ] `plugins/vsdd-factory/hooks/hooks.json.template` exists, committed, documented
- [ ] Placeholder syntax is explicit and minimal: `{{PLATFORM}}` and `{{EXE_SUFFIX}}`
- [ ] Generator script (`scripts/generate-hooks-json.sh`) reads the template and
  produces 5 variants at `plugins/vsdd-factory/hooks/hooks.json.<platform>`
- [ ] Each generated variant has the correct dispatcher binary path:
  `${CLAUDE_PLUGIN_ROOT}/hooks/dispatcher/bin/<platform>/factory-dispatcher[.exe]`
- [ ] `hooks.json` (the canonical one) is in `.gitignore`
- [ ] CI job "generate-hooks-json-variants" runs on every push to main; fails on drift
- [ ] Release workflow regenerates variants as part of the release chore commit
- [ ] Generator is covered by bats tests

## Tasks

1. Design the template file. Single template structure; only the dispatcher binary path varies per platform.
2. Define the placeholder set: `{{PLATFORM}}` and `{{EXE_SUFFIX}}`.
3. Write `scripts/generate-hooks-json.sh` — POSIX bash, iterates the 5 platforms, sed-substitutes placeholders.
4. Add `.gitignore` entry for `plugins/vsdd-factory/hooks/hooks.json`.
5. Commit initial versions of `hooks.json.template` and the 5 `hooks.json.<platform>` files.
6. Add CI job that runs the generator and diffs against committed output.
7. Add bats test: temp-dir-based run with a fixture template, assertions on each generated file.

## Contracts

**Template placeholder contract:**

```
{{PLATFORM}}      → "darwin-arm64" | "darwin-x64" | "linux-x64" |
                    "linux-arm64" | "windows-x64"
{{EXE_SUFFIX}}    → "" for Unix, ".exe" for Windows
```

Non-blocking events (PostToolUse, Stop, SubagentStop, SessionStart,
SessionEnd) get `"async": true`. Blocking events (PreToolUse,
PermissionRequest) stay sync so `exit 2` still blocks.

## Test plan

- **Generator tests**: fixture template + run generator + diff each expected variant.
- **CI drift check**: simulate a hand-edit of a variant, verify CI fails.
- **Template schema validity**: each generated variant must pass JSON parsing.

## Notes

The 7 hook events in the template (PreToolUse, PostToolUse, PermissionRequest,
Stop, SubagentStop, SessionStart, SessionEnd) cover every event for v1.0.

`"once": true` on SessionStart/End prevents re-entry if same session re-evaluates.

Template rendering is simple sed substitution; no Jinja/Mustache dep.
