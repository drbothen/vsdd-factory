# S-4.01: sink-http driver

**Epic:** E-4 — Observability Sinks and RC Release
**Phase:** Phase 4 — Observability sinks
**Milestone:** `1.0.0-rc.1`
**Tier:** E
**Blocks:** S-4.02, S-4.03, S-4.04, S-4.07, S-4.08
**Blocked by:** S-1.08 (Sink trait)
**Estimated effort:** M

## Narrative
- **As a** factory operator with a custom observability backend
- **I want to** configure a generic HTTP endpoint to receive events as JSON POST batches
- **So that** I can send factory telemetry to any HTTP-capable backend without a vendor-specific sink

## Goal

Implement a generic `http` sink that batches events and POSTs them to a
user-configured endpoint. This is the "bring your own backend" option and
the base HTTP infrastructure that Datadog and Honeycomb sinks build on.

## Acceptance criteria

- [ ] `crates/sink-http` implements the `Sink` trait
- [ ] Events batched and POSTed as JSON array to configured URL
- [ ] Configurable: endpoint URL, batch size, flush interval, auth header
- [ ] Handles HTTP errors: retry on 5xx, drop on 4xx
- [ ] `flush()` sends current batch synchronously
- [ ] Integration test with a mock HTTP server verifies batch delivery
