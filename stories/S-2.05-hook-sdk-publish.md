# S-2.05: hook-sdk publish to crates.io (0.1.0)

**Epic:** E-2 — Legacy Adapter and Beta Release
**Phase:** Phase 2 — Legacy bash adapter
**Milestone:** `1.0.0-beta.1`
**Tier:** C
**Blocks:** S-2.08
**Blocked by:** S-1.03 (the SDK crate itself)
**Estimated effort:** S

> **Status note:** As of v1.0.0-beta.4, `cargo publish --dry-run` passes cleanly but the
> real `cargo publish` to crates.io has been deferred. The crate is publish-ready;
> the actual publishing step is outstanding.

## Narrative
- **As a** third-party plugin author
- **I want to** add `vsdd-hook-sdk` as a dependency from crates.io
- **So that** I can write WASM plugins without cloning the vsdd-factory repository

## Goal

Publish the `vsdd-hook-sdk` crate (and its proc-macro subcrate
`vsdd-hook-sdk-macros`) to crates.io at version 0.1.0. Establishes the
public Rust API surface third-party plugin authors depend on.

## Acceptance Criteria

### AC-001: Dry-run publish passes for both crates
- [ ] `cargo publish --dry-run` passes for `vsdd-hook-sdk` and `vsdd-hook-sdk-macros` without errors.
- [process-gap] No behavioral contract currently scopes the publish-pipeline outcome; candidate BC-2.06.001 (`hook-sdk-published-on-crates-io`) to be authored in v1.1.

### AC-002: Required crate metadata is present
- [ ] Crate metadata fields `description`, `license`, `repository`, `keywords`, `categories` are all set in both `Cargo.toml` files.
- [process-gap] No behavioral contract covers crate metadata completeness; packaging concern without in-process behavioral semantics.

### AC-003: README with quickstart example at crate root
- [ ] `README.md` is present at `crates/hook-sdk/README.md` with a quickstart example showing the `#[hook]` usage.
- [process-gap] No behavioral contract covers documentation publication; packaging concern without in-process behavioral semantics.

### AC-004: `vsdd-hook-sdk` version 0.1.0 published
- [ ] `vsdd-hook-sdk` is published to crates.io at version `0.1.0` and resolvable via `cargo search vsdd-hook-sdk`.
- [process-gap] No behavioral contract currently in scope; candidate BC-2.06.001 (`hook-sdk-published-on-crates-io`) to be authored in v1.1.

### AC-005: `vsdd-hook-sdk-macros` version 0.1.0 published before main crate
- [ ] `vsdd-hook-sdk-macros` is published to crates.io at version `0.1.0` before `vsdd-hook-sdk` (dependency ordering required by crates.io).
- [process-gap] No behavioral contract currently in scope; candidate BC-2.06.002 (`hook-sdk-macros-published-before-main`) to be authored in v1.1.

### AC-006: Release workflow includes `publish-sdk` job
- [ ] `.github/workflows/Release.yml` has a `publish-sdk` job gated on the `CRATES_IO_TOKEN` secret that triggers on tag push.
- [process-gap] No behavioral contract covers CI workflow shape; candidate BC-2.06.002 ordering constraint applies indirectly.

**No behavioral contracts apply to S-2.05.** The `behavioral_contracts: []` array is empty by design, not by omission.
