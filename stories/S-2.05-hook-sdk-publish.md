# S-2.05: hook-sdk publish to crates.io (0.1.0)

**Epic:** E-2 — Legacy Adapter and Beta Release
**Phase:** Phase 2 — Legacy bash adapter
**Milestone:** `1.0.0-beta.1`
**Tier:** C
**Blocks:** S-2.08
**Blocked by:** S-1.03 (the SDK crate itself)
**Estimated effort:** S

> **Status note:** As of v1.0.0-beta.4, `cargo publish --dry-run` passes cleanly but the
> real `cargo publish` to crates.io has been deferred. The crate is publish-ready;
> the actual publishing step is outstanding.

## Narrative
- **As a** third-party plugin author
- **I want to** add `vsdd-hook-sdk` as a dependency from crates.io
- **So that** I can write WASM plugins without cloning the vsdd-factory repository

## Goal

Publish the `vsdd-hook-sdk` crate (and its proc-macro subcrate
`vsdd-hook-sdk-macros`) to crates.io at version 0.1.0. Establishes the
public Rust API surface third-party plugin authors depend on.

## Acceptance criteria

- [ ] `cargo publish --dry-run` passes for `vsdd-hook-sdk` and `vsdd-hook-sdk-macros`
- [ ] Crate metadata: `description`, `license`, `repository`, `keywords`, `categories` fields all set
- [ ] `README.md` at crate root with quickstart example
- [ ] `vsdd-hook-sdk` published at `0.1.0`
- [ ] `vsdd-hook-sdk-macros` published at `0.1.0`
- [ ] Release workflow has a `publish-sdk` job that publishes on tag push
