# S-5.02: SessionEnd hook wiring

**Epic:** E-5 — New Hook Events and 1.0.0 Release
**Phase:** Phase 5 — New Claude Code hook events
**Milestone:** `1.0.0`
**Tier:** G
**Blocks:** S-5.07
**Blocked by:** S-4.08
**Estimated effort:** S

## Narrative
- **As a** factory operator
- **I want to** receive a session.ended event when each Claude Code session finishes
- **So that** I can compute session duration, tool-call count, and session-level cost tracking

## Goal

Wire `SessionEnd` through the dispatcher. Emits `session.ended` with
duration, tool-call count, and cost. Pairs with `SessionStart` (S-5.01)
to give operators session-level telemetry.

## Acceptance criteria

- [ ] Dispatcher handles `SessionEnd` event type from Claude Code
- [ ] `session.ended` event emitted with session_id, duration_ms, tool_call_count
- [ ] Duration computed from `SessionStart` timestamp (if available)
- [ ] `hooks.json` template updated with SessionEnd handler
- [ ] Integration test: send SessionEnd payload; verify `session.ended` in file sink
