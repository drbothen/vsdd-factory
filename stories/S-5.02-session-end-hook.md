## Capability Anchor Justification

Primary: CAP-002 ("Hook Claude Code tool calls with sandboxed WASM plugins") per capabilities.md §CAP-002. This story wires the `SessionEnd` event through the dispatcher and implements the session-end plugin that emits `session.ended` with session telemetry (duration, tool-call count).

FR anchor: FR-007 ("Legacy hook routing compatibility + dispatcher main") per PRD §2.1, extended. No dedicated FR exists for SessionEnd wiring — same DRIFT-006 gap as S-5.01. Both S-5.01 and S-5.02 implement the proposed FR-045 ("New Claude Code lifecycle hook events").

Cross-subsystem note: BC-1.02.005 (SessionStart parses without tool_name) applies to SessionEnd parsing by the same envelope invariant. SS-04 plugin BCs are v1.1 candidates.

## Verification Properties

## Narrative
- **As a** factory operator
- **I want to** receive a session.ended event when each Claude Code session finishes
- **So that** I can compute session duration, tool-call count, and session-level cost tracking

## Goal

Wire `SessionEnd` through the dispatcher. Emits `session.ended` with
duration, tool-call count, and cost. Pairs with `SessionStart` (S-5.01)
to give operators session-level telemetry.
